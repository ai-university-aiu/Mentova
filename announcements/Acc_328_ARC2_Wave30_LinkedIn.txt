This task involves frame blocks: rectangular 3x3 squares or cross shapes,
each with a center color and an arm color. A snake path of 5-valued cells
threads through the grid and is left unchanged.

The frame_absorb rule builds an absorption graph: if block X's center color
equals block Y's arm color, then X can absorb Y. Absorbing means Y's cells
are erased and X's center cell is repainted with Y's original center color.

The solver processes this graph in a strict priority order:
- Monochrome blocks (center = arm color) are processed first.
- Then chain-start blocks (blocks with no incoming absorption edge) are processed.
- Any remaining blocks in a cycle are erased.

dead-end block, and a two-step absorption chain -- all in pure Prolog with
no machine learning and no pretraining.
