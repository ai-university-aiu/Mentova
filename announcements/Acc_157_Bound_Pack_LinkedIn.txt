PrologAI can now compute tight bounding boxes, crop objects, trim borders,
pad grids, and place sub-grids at computed positions.

Work Package 137 - the bound pack - adds 14 bd_* predicates. 42/42 tests.

Core pipeline:

bd_bbox(Grid, Color, R0, C0, R1, C1)  % tight bounding box of Color
bd_crop_color(Grid, Color, Sub)         % crop to that box in one call
bd_trim(Grid, Bg, Trimmed)              % remove all-Bg borders
bd_pad(Grid, N, Bg, Padded)            % add N Bg cells on all sides
bd_place(Canvas, Patch, R0, C0, Bg, New) % place sub-grid at (R0,C0)

Box queries:

bd_bbox_h(R0, R1, H)      % H = R1 - R0 + 1
bd_bbox_w(C0, C1, W)      % W = C1 - C0 + 1
bd_center(Grid, Color, CR, CC)  % integer center (floor division)

Region filling:

bd_fill_bbox(Grid, R0, C0, R1, C1, Color, New)  % paint a rectangle

Box algebra:

bd_bbox_contains(R0,C0,R1,C1, R,C)  % is (R,C) inside?
bd_bbox_overlap(R0A,C0A,R1A,C1A, R0B,C0B,R1B,C1B)  % do boxes share any cell?
bd_bbox_union(R0A,C0A,R1A,C1A, R0B,C0B,R1B,C1B, R0,C0,R1,C1)  % enclosing box
bd_expand(R0,C0,R1,C1, N, ER0,EC0,ER1,EC1)  % grow by N cells on all sides

The key insight: bd_trim + bd_pad implement the extract-and-center pipeline.
bd_place assembles result grids from independently computed parts.
bd_slice_row_ uses inclusive bounds (Len = C1-C0+1) with length/2 + append/3.
