PrologAI now has a scenepair pack - 14 ps_* predicates that analyze a Before
and After scene pair holistically, computing observables for rule hypothesis ranking.

Instead of blindly trying every transformation rule, a solver can first ask
scenepair what kind of change most likely occurred. The pack answers: did the
object count change? did the color set change? was this a single recolor, a
recolor-all, a uniform shift, or a shift-to-origin?

Key design: object identity uses color-and-form matching (normalized to origin
position + sorted cells), not positional index. This makes ps_added_objs and
ps_removed_objs robust to scene reordering.

ps_color_delta computes Old-New pairs from position-matched pairs, with duplicate
suppression. ps_is_recolor uses set subtraction on the distinct-color lists.
ps_is_shift derives a candidate (DR,DC) from the first matched object, then
verifies the entire scene with msort before cutting.
