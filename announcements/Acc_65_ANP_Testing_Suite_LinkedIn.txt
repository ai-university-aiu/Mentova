WHAT ANP MEANS:

Unlike MCP, A2A, and ACP — which all assume the client already knows the agent's endpoint — ANP enables decentralized discovery without a central registry.

Any agent that knows Mentova's hostname can:
1. Resolve did:web:ccai2 to discover Mentova's capabilities and key.
2. Query /.well-known/agent-descriptions to find supported protocols.
3. Choose which protocol to use (MCP, A2A, ACP, or ANP) via meta-protocol negotiation.
4. Exchange signed messages with verified sender identity.

REJECTION OF TAMPERED MESSAGES IS STRUCTURAL:

The security guarantee is architectural: tampered messages are rejected at the signature verification step, before any content enters any scope. This is not a policy check. It is a structural boundary.
