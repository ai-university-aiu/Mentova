Abstract

PrologAI gains a dedicated numerical sequence operations pack as Work
Package 145 (Layer 124).

The pack provides 14 predicates, all carrying the nr_* prefix, for
analyzing and transforming one-dimensional lists of numbers.

These predicates complement the cross pack (Layer 121, cx_*) which
extracts rows, columns, and diagonals from two-dimensional grids.

The numseq pack then processes those extracted slices: computing
pairwise differences, cumulative sums, running statistics, element-wise
binary operations, scalar transforms, sorted and arithmetic sequence
tests, and period detection.

All 42 acceptance tests pass.


1. Motivation

Grid-solving tasks frequently require analyzing the numerical content
of a single row, column, or diagonal after it has been extracted.

The cross pack isolates the slice; the numseq pack provides the
vocabulary to reason about it.

Pairwise differences reveal arithmetic progressions.

Cumulative sums reveal counting patterns.

Running maximum and minimum track the extreme value seen so far.

Element-wise zip operations compare two slices position by position.

Period detection finds repeating structural patterns.


2. Predicates

The pack exports 14 predicates.

nr_diff(+List, -Diffs) computes consecutive pairwise differences.

Each element of Diffs is the difference between the next and current
elements of List.

For [1, 3, 6, 10] the result is [2, 3, 4].

nr_cumsum(+List, -Sums) computes prefix cumulative sums.

Each element of Sums is the sum of all List elements up to and
including that position.

For [1, 2, 3, 4] the result is [1, 3, 6, 10].

nr_running_max(+List, -Maxs) computes the running maximum.

Each element of Maxs is the maximum of all List elements seen so far.

For [3, 1, 4, 1, 5, 9, 2] the result is [3, 3, 4, 4, 5, 9, 9].

nr_running_min(+List, -Mins) computes the running minimum symmetrically.

For [5, 3, 4, 1, 2] the result is [5, 3, 3, 1, 1].

Five predicates perform element-wise binary operations on two
equal-length lists.

nr_zip_add(+A, +B, -C) produces C[I] = A[I] + B[I].

nr_zip_sub(+A, +B, -C) produces C[I] = A[I] - B[I].

nr_zip_mul(+A, +B, -C) produces C[I] = A[I] times B[I].

nr_zip_max(+A, +B, -C) produces C[I] = max(A[I], B[I]).

nr_zip_min(+A, +B, -C) produces C[I] = min(A[I], B[I]).

Two scalar predicates transform every element uniformly.

nr_scale(+List, +K, -Scaled) multiplies every element by K.

nr_offset(+List, +D, -Shifted) adds D to every element.

Two test predicates analyze sequence properties.

nr_is_sorted(+List) succeeds iff the list is non-decreasing.

nr_is_arith(+List) succeeds iff the list has a constant pairwise
difference (an arithmetic sequence).

One structural predicate detects repetition.

nr_period(+List, +MaxP, -P) finds the shortest repeating period P
within the range 1 to MaxP.

For [1, 2, 1, 2, 1, 2] with MaxP = 3 the result is P = 2.

For [5, 5, 5, 5] with MaxP = 4 the result is P = 1.


3. Implementation

The five zip predicates use maplist/3 from library(apply) with named
per-element helper predicates.

Named helpers (nr_add_, nr_sub_, nr_mul_, nr_max_, nr_min_) are
preferred over lambda syntax to ensure compatibility across all
SWI-Prolog versions.

The scalar predicates use the same maplist/3 pattern with helpers
(nr_scale_, nr_offset_) that capture the scalar from the outer call.

The running statistics predicates (nr_running_max, nr_running_min,
nr_cumsum) use recursive accumulators implemented as private helper
predicates with an underscore suffix.

nr_is_sorted uses three-clause recursion with the A =< B guard.

nr_is_arith computes the common difference D from the first pair and
verifies D2 =:= D throughout the remainder via a recursive helper.

nr_period uses between/3 to try candidate periods in ascending order,
cuts on the first that satisfies Len mod P =:= 0 and passes the
element-equality check, which uses forall/2 with nth0/3.

The predicates length/2, between/3, and forall/2 are SWI-Prolog
built-ins and are not imported from any library.

The predicates nth0/3 and numlist/3 are imported from library(lists).


4. Testing

42 acceptance tests cover all 14 predicates at 3 tests per predicate.

Tests cover the canonical case, an edge case (empty or single-element
input), and a boundary or failure case (sorted check failing, period
detection on a constant sequence, negative offsets).

All 42 pass under SWI-Prolog.

6. Position in the Cognitive Architecture

The numseq pack is Layer 124 in the PrologAI cognitive stack.

It sits above the cross pack (Layer 121) which supplies raw 1D slices.

Together, the two packs form a pipeline: extract a slice, then analyze
its numerical content.

This pipeline is a foundation for reasoning about arithmetic progressions,
geometric patterns, counting sequences, and periodic structures in
two-dimensional grids.
