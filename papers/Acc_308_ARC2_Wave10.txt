ABSTRACT

Wave 10 of the ARC-AGI-2 benchmark climb adds the apex_shadow rule to the
glass-box named-rule library, solving task 409aa875 and advancing the score
from 9/120 (7.50%) to 10/120 (8.33%).  The rule discovers that every
non-background shape in a 16x16 grid projects a shadow via the geometry of its
two arm vectors.  A unified normalised-arm-vector formula covers all four
chevron orientations and all four L-shape orientations with no special cases.
Two projections to the same background cell produce a collision marker (value 1)
instead of a shadow (value 9).  A non-background projection target triggers
recolouring of the entire connected component to 9.


1. TASK DESCRIPTION

Task 409aa875 presents 16x16 grids (background colour 7) containing:

  - Chevron shapes: a size-1 isolated cell (apex) whose two non-background
    8-connected neighbours (wings) define its orientation.
  - L-shaped components: three 4-connected cells forming an L, with the corner
    cell (the unique cell with two 4-connected intra-component neighbours)
    serving as the projection centre.

The output is identical to the input except that each active shape casts a
shadow of value 9 at a specific target position.  When two shadows fall on the
same background cell the result is 1 (collision).  When a shadow falls on an
existing non-background cell the entire 4-connected component containing that
cell is recoloured to 9.


2. THE ARM-VECTOR RULE

For every component:

  Centre: apex cell (size-1 isolated cell with exactly 2 non-bg 8-neighbors)
          OR corner cell (3-cell L-shape, the cell with 2 intra-component
          4-connected neighbors).

  Arm1, Arm2: unit-step direction vectors from centre to the two related cells.

  arm sum: sR = arm1R + arm2R,  sC = arm1C + arm2C.

  Projection displacement: dR = -5 * sR / max(|sR|, |sC|)
                           dC = -5 * sC / max(|sR|, |sC|)

  Target: (centre_R + dR, centre_C + dC).

The L-infinity normalisation (dividing by max of |sR|, |sC|) ensures exactly 5
steps in the dominant direction, covering all orientations:

  Shape type                         arm sum    dR  dC  direction
  -----------------------------------------------------------------
  Standard chevron (wings R+1,C+-1)  (+2, 0)   -5   0  straight up
  Inverted chevron (wings R-1,C+-1)  (-2, 0)   +5   0  straight down
  Right-column chevron (wings R+-1,C+1) (0,+2)  0  -5  straight left
  Left-column chevron (wings R+-1,C-1)  (0,-2)  0  +5  straight right
  L arms up+right from corner        (-1,+1)   +5  -5  down-left
  L arms up+left from corner         (-1,-1)   +5  +5  down-right
  L arms down+right from corner      (+1,+1)   -5  -5  up-left
  L arms down+left from corner       (+1,-1)   -5  +5  up-right

5. KEY BUG FIX AND LESSON

The original implementation used a global "leftmost non-background column" L
as the column offset for every shape: ΔC = -L.  This coincidentally matched
the three training pairs (which all contain only downward-pointing chevrons and
one standard L-orientation, giving ΔC = 0 or -5 matching leftmost = 0 or 5).
The test case introduced inverted and diagonal chevrons and a mirrored L, whose
correct ΔC values (+0, -5, +5) do not depend on the global leftmost column.

The fix replaces the global offset with the per-shape arm-vector formula.  The
single unified formula handles all eight orientation cases automatically.

Lesson: training pairs alone cannot validate a formula when all training shapes
share the same orientation.  The test case is the proof of concept.


6. PROLOG IMPLEMENTATION

Key new predicates in Mentova/src/mentova/games/arc_benchmark_2.pl:

  as_comp_proj_/7: dispatches by component size (1 = apex, 3 = L-corner) and
    calls as_arm_proj_ with the derived arm vectors.

  as_arm_proj_/10: computes SR=arm1R+arm2R, SC=arm1C+arm2C, M=max(|SR|,|SC|),
    then dR=-5*SR//M, dC=-5*SC//M, and verifies the target is in-bounds.

  as_8nbr_wings_/5: returns the list of DR-DC direction vectors to all non-bg
    8-neighbors of a given cell.

  as_bg_nonbg_split_/5: partitions a target list into background-cell targets
    and non-background-cell targets.

  as_count_hits_/2: takes a sorted list and produces a key-count list by
    grouping identical elements.

Retained helpers: as_components_, as_bfs4_comp_, as_corner_, as_shadow9_.
