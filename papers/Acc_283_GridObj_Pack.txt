Abstract

PrologAI gains a fourteenth grid perception pack dedicated to connected component
analysis, flood fill, and object manipulation.

The gridobj pack (Work Package 263, Layer 242, prefix gob_*) provides 14 predicates
for identifying, querying, and transforming discrete colored regions in symbolic grids
using 4-connectivity (up, down, left, right - no diagonals).

Background

ARC-AGI-2 tasks frequently require the system to perceive a grid not as a flat array
of color values but as a collection of discrete objects - contiguous regions of
uniform color that can be counted, measured, moved, or filled.

Prior PrologAI grid packs addressed color operations, resizing, tiling, and spatial
analysis but none offered first-class connected component primitives.

The gridobj pack closes that gap, providing the full object lifecycle from discovery
through manipulation.

The Pack

Module: gridobj. File: packs/gridobj/prolog/gridobj.pl.
Prefix: gob_. Layer: 242. Work Package: 263. PrologAI PR: 353.

Predicates:

gob_object_cells(+Grid, +R, +C, -Cells)
Returns the list of r(R,C) positions in the 4-connected component seeded at (R,C).
Uses BFS with duplicate prevention in both the visited set and the frontier tail.

gob_object_color(+Grid, +R, +C, -Color)
Returns Grid[R][C] - the color shared by all cells in the component.

gob_object_size(+Grid, +R, +C, -Size)
Returns the count of cells in the 4-connected component at (R,C).

gob_object_bbox(+Grid, +R, +C, -BBox)
Returns r0(R0,C0,R1,C1): the tight bounding box of the component.

gob_object_mask(+Grid, +R, +C, +Bg, -Mask)
Returns a grid of the same dimensions as Grid, with component cells shown in their
color and all other cells replaced by Bg.

gob_extract_object(+Grid, +R, +C, +Bg, -Extracted)
Returns the bounding-box subgrid of the component, with non-component cells
inside the bounding box filled with Bg.

gob_all_objects(+Grid, +Bg, -Objects)
Returns a list of ob(Color,Cells,BBox) for every non-Bg connected component,
in scan order (top-to-bottom, left-to-right by seed cell).

gob_object_count(+Grid, +Bg, -Count)
Returns the number of distinct non-Bg connected components.

gob_largest_object(+Grid, +Bg, -Cells)
Returns the cell list of the connected component with the most cells.
Fails if Grid has no non-Bg cells.

gob_smallest_object(+Grid, +Bg, -Cells)
Returns the cell list of the connected component with the fewest cells.
Fails if Grid has no non-Bg cells.

gob_flood_fill(+Grid, +R, +C, +NewColor, -Result)
Replaces all cells of the connected component at (R,C) with NewColor.
Cells of other colors, including Bg, are unchanged.

gob_fill_enclosed(+Grid, +Bg, +FgColor, -Result)
Fills all Bg regions fully enclosed by non-Bg cells with FgColor.
A region is enclosed if it has no 4-connected path to the grid border.
Uses reverse BFS: seeds from all border Bg cells, then fills anything unreachable.

gob_remove_object(+Grid, +R, +C, +Bg, -Result)
Replaces all cells of the connected component at (R,C) with Bg.
Other cells are unchanged.

gob_move_object(+Grid, +R, +C, +NewR, +NewC, +Bg, -Result)
Moves the connected component at (R,C) so its bounding-box top-left is at
(NewR,NewC). Removes the original cells, then places the component at the
new position. Cells that fall outside the grid bounds after moving are clipped.

BFS Implementation

The core BFS (gob_bfs_/7) uses a frontier list and a visited set.

Starting from the seed cell, each BFS step expands the first frontier cell to its
4 neighbors, filtering by: (1) in-bounds, (2) matching color, (3) not in visited,
(4) not already in the frontier tail.

Checking both visited and the tail prevents any cell from being enqueued twice,
which would cause duplicate entries in the result even when two frontier cells
share a neighbor.

gob_fill_enclosed uses a symmetric BFS (gob_bg_bfs_/7) seeded from all border
Bg cells simultaneously, collecting all Bg reachable from the border.
The complement - Bg cells not in that reachable set - are the enclosed regions.
