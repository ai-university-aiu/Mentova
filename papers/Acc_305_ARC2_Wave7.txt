Wave 7 of the ARC-AGI-2 benchmark climb adds the staircase_lift named glass-box rule,
solving task 4c3d4a41 and bringing the running total to 7 out of 120 tasks (5.83%).

The staircase_lift rule decodes a staircase pattern encoded in the left half of an
8x20 grid and uses it to precisely reposition vertical color bars in the right half.
Each staircase column height H controls a single bar: the bar ends at row (5 - H) in
the output, with its length clipped to fit the available space.


TASK ANALYSIS: 4c3d4a41

Grid dimensions: 8 rows x 20 columns (same for input and output).
Training pairs: 2.
Test case: 1.

Left half structure (columns 0-8):
  - Row 5 (floor): columns 1-7 filled with 5.
  - Staircase bars at odd columns 1, 3, 5, 7, each extending upward from row 5.
  - The height H[i] of bar i = count of consecutive 5s from row 5 upward
    in column (2i+1).

Right half structure (columns 9-19):
  - Outer rectangular frame: row 0, row 7, column 9 (partial), column 19.
  - Four vertical color bars at odd columns 11, 13, 15, 17.
  - Even columns 10, 12, 14, 16, 18 are zero-valued separators.

Output rule for each bar i (i = 0, 1, 2, 3):
  MaxEnd[i] = 5 - H[i]           (last row the bar can occupy)
  OutHeight[i] = min(L[i], MaxEnd[i])  (bar height, clipped to available space)
  OutEnd[i] = MaxEnd[i]
  OutStart[i] = OutEnd[i] - OutHeight[i] + 1
  Color[i] occupies rows OutStart[i] through OutEnd[i] in column (11 + 2i).
  Rows (OutEnd[i]+1) through 4 in that column become 5 (staircase fill).
  Row 5 (inner bottom bar, columns 11-17): all 5.
  Left half: all 0 in the output.

KEY INSIGHT

The staircase height H encodes the BOTTOM ROW of the output bar region
via the formula MaxEnd = 5 - H. Taller staircases leave less room, forcing
shorter or higher bars; shorter staircases leave more room, allowing longer bars.
The input bar length sets an upper bound on output height, and the staircase
sets a positional constraint. These two constraints together uniquely determine
every output bar placement.
