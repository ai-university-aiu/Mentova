Abstract

Mentova's cognitive substrate gains its 78th work package: the induction pack (id_* predicate family, Layer 59). This pack enables systematic inductive reasoning over grid pairs - the exact operation required when analyzing examples to form a transformation hypothesis. Given an input grid and its corresponding output grid, the induction pack determines what changed, how colors were remapped, whether the output is a pure color substitution of the input, and whether the grid scaled up by an integer factor. These primitives turn raw before-and-after observations into structured hypotheses.

Introduction

Inductive reasoning from examples is the core challenge in visual analogy tasks. A system that can observe multiple input-output pairs and distill a consistent rule is engaging in the same kind of learning that underlies human concept acquisition. The induction pack provides the foundational measurement tools: before higher-level reasoning can form a hypothesis, it must be able to answer basic factual questions about what happened between input and output.

The pack complements the compare pack (cp_*, Layer 57), which provides cell-level diff operations, by adding hypothesis-level operations: is this a recolor? is this a scale? what colors are new? The two packs together give Mentova a complete observational toolkit for grid-pair analysis.

Predicates

The induction pack exports 14 predicates under the id_* prefix.

Dimension analysis:
id_grid_dims/3 returns the number of rows and columns in any grid.

Color set analysis:
id_input_colors/2 returns a sorted list of distinct colors in the input grid.
id_output_colors/2 returns the same for the output grid.
id_new_colors/3 returns colors present in output but absent from input.
id_lost_colors/3 returns colors present in input but absent from output.

Cell-level change analysis:
id_changed_cells/3 returns the list of r(R,C) cells where input and output values differ.
id_unchanged_cells/3 returns the cells where they are the same.

Color map and recolor detection:
id_color_map/3 infers a sorted list of Old-New color substitution pairs from all changed cells. If a single input color consistently maps to one output color everywhere it appears, the map is consistent.
id_is_recolor/2 succeeds when the output is a pure, consistent color substitution of the input: every cell with the same input color gets the same output color, and unchanged cells keep their value.

Uniform output detection:
id_uniform_output/2 succeeds when every cell in the output has the same color, and returns that color.
id_output_color/2 returns the single uniform output color; fails if the output is not uniform.

Scale detection:
id_size_ratio/3 computes the ratio of output dimensions to input dimensions, returning integers when division is exact.
id_is_scale/2 succeeds when the output is a uniform integer scale of the input: both the row ratio and the column ratio are the same positive integer.
id_scale_factor/3 returns that integer scale factor.

Implementation

The pack is implemented in pure SWI-Prolog (packs/induction/prolog/induction.pl). Color set operations delegate to library(lists) subtract/3 for set difference. Change detection uses include/3 with a locally defined predicate that fetches and compares cell values via nth0/3. The recolor consistency check uses negation-as-failure: the map is consistent if no cell violates it. Scale detection uses integer modulo and integer division to avoid floating-point comparison.

All 37 PLUnit acceptance tests pass. The pack depends only on the grid pack for its r(R,C) cell convention.
