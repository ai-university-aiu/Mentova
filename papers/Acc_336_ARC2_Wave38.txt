Acc_336: ARC-AGI-2 Wave 38 - fill_enclosed Rule - Task 8b7bacbf

D. R. Dison, Founder of AIU, Creator and Owner of PrologAI and Mentova, ORCID 0009-0001-9246-5758, linkedin.com/in/d-r-dison/

Date: 2026-06-30


ABSTRACT

Wave 38 of the PrologAI ARC-AGI-2 benchmark climb adds the fill_enclosed named
rule, which solves task 8b7bacbf. The rule identifies enclosed background (BG)
pockets in a grid by detecting connected BG components whose every in-grid
4-connected neighbor belongs to the frame material (FrameSet). Frame values are
derived from values adjacent to the VAdj indicator cells, with a pure-frame
structural test distinguishing true alternative frame colors from planted
contamination tokens. Enclosed pockets with a clean VAdj indicator adjacent to
their frame are filled with the singleton marker M; pockets without a clean
indicator are left unchanged as decoys. After Wave 38, PrologAI solves 38 of 120
ARC-AGI-2 tasks (31.67%) using pure Prolog named rules with no machine learning.


1. TASK DESCRIPTION

Task 8b7bacbf presents input grids containing a background color BG (most
frequent value), a singleton marker M (appearing exactly once), rectangular frame
structures of one or more frame colors, and small BG clusters (pockets) enclosed
by those frames. Some pockets are targets (to be filled with M) and others are
decoys (left unchanged). Outside the frames, VAdj-colored indicator cells sit
adjacent to target-pocket frames, and contamination tokens (VDec cells) may be
present to mislead naive indicator detection.

The task requires five capabilities:

(a) Pocket detection: correctly identifying which BG clusters are enclosed by
frame material, handling boundary-flush pockets, disconnected backgrounds, and
outer-wall structures that must NOT be treated as frame material.

(b) FrameSet derivation: computing which non-BG non-M non-VAdj values constitute
frame material, distinguishing them from outer walls and contamination tokens.

(c) Indicator detection: finding VAdj-colored cells adjacent to the frame of each
pocket and checking for contamination via a chain-based BFS test.

(d) Multi-frame classification: in grids with multiple frame colors (P1), pockets
surrounded exclusively by the dominant frame color and shaped as rectangles are
decoys; others are targets.

(e) Edge-case handling: when no clean indicator exists for any pocket, all pockets
are targets (as in P4 where the path cell is far from any frame).


2. ALGORITHM

Step 1. BG = most frequent value in the grid. Ms = values appearing exactly once
(excluding BG). VAdjs[i] = most common non-BG non-M 4-connected neighbor of Ms[i].

Step 2. Find ALL BG connected components via list-based 4-connected BFS. The BFS
passes a growing visited set so each BG cell is processed exactly once across all
component searches.

Step 3. FrameSet derivation:
- FSAdj = non-BG/M/VAdj values that appear as 4-connected neighbors of any VAdj
  cell in the grid.
- If FSAdj is empty (VAdj cells surrounded only by BG), fall back to all
  non-BG/M/VAdj values (handles grids where frame and path are separated).
- FDom = most frequent value in FSAdj (or fallback set).
- FOthers = non-FDom values in FSAdj that ALONE form a complete enclosure around
  at least one BG component (the pure-frame structural test). Values that appear
  in FSAdj but fail the pure-frame test are VDecContam (contamination tokens).
- FrameSet = {FDom} union FOthers.
- VDec = VDecContam union any value in the grid not in BG/Ms/VAdjs/FrameSet.

Step 4. Pocket test: a BG component C is a pocket iff FrameSet is non-empty AND
every in-grid 4-connected non-cluster neighbor of every cell in C belongs to FrameSet.

Step 5. Classification:
- Multi-frame (FOthers non-empty): pockets that are NOT rectangular or NOT
  exclusively FDom-framed are targets for Ms[0]; rectangular FDom-only pockets
  are decoys.
- Single-frame (FOthers empty): a pocket is a target for Ms[i] if any frame
  neighbor of the pocket has a 4-connected VAdjs[i]-valued cell that is NOT
  contaminated. A VAdj cell is contaminated if any cell in its 4-connected
  VAdj-chain is adjacent to a VDec cell.
- Exception: if no pocket has any clean indicator, all pockets are targets.

Step 6. Fill each target pocket's cells with its assigned M value.


3. BUG SEQUENCE AND FIXES

Bug 1: Boundary-seed BFS incorrectly missed boundary-flush pockets.
The initial implementation seeded BFS from ALL boundary BG cells. P3 has a 2x2
pocket at rows 1-2, cols 18-19 that touches the right grid boundary. Seeding from
boundary cells directly included these pocket cells, marking them reachable and
failing to detect the pocket. Fix: component-based detection with the FrameSet
neighbor test correctly identifies this pocket because its in-grid neighbors are
all 2-cells (FrameSet), regardless of its boundary position.

Bug 2: Single-corner BFS created a false enclosed region in P4.
BFS from corner (0,0) only reached the left background region. P4's 1-cell
diagonal path separates the background into left and right regions. The right
15-cell region was falsely classified as a pocket. Fix: same component approach;
the right region has 1-cells (VAdj) as neighbors, and VAdj is not in FrameSet,
so the region fails the pocket test.

Bug 3: Outer-wall value 1 in P3 was included in FrameSet.
P3 has a large 1-cell outer wall that forms the boundary of a large interior
region. Value 1 is not BG, M, or VAdj (VAdj=3 in P3), so the naive FrameSet
computation included it. FrameSet={1,2} caused the large 52-cell interior BG
region to pass the pocket test. Fix: compute FrameSet as values adjacent to
VAdj cells. In P3, VAdj=3 and 3-cells are only adjacent to 2-cells (not 1-cells).
FrameSet={2}, correctly excluding the outer wall.

Bug 4: Empty FSAdj in P4 gave FrameSet=[].
In P4 the 1-cells (VAdj) are surrounded by BG cells and not adjacent to any
frame cells. FSAdj was empty, FrameSet=[], and no pockets were found. Fix:
fallback to all non-BG/M/VAdj values when FSAdj is empty. In P4: FrameSet={2}.

Bug 5: Test case value 0 (VDec) was included in FOthers.
Value 0 (VDec, appearing at 2 cells) is adjacent to a 3-cell (VAdj) in the test
case, so it entered FSAdj. The naive approach placed it in FOthers, activating
multi-frame classification logic incorrectly. Fix: the pure-frame structural test
requires a value V to ALONE enclose some BG component. Value 0 appears at 2 cells
and cannot form any enclosure; it becomes VDecContam. Values 3 and 5 in P1 each
form an 8-cell frame around a 2x2 pocket; they correctly enter FOthers.


4. IMPLEMENTATION

The implementation in Mentova/src/mentova/games/arc_benchmark_2.pl uses
list-based BFS throughout (no dynamic facts), consistent with the existing file
style. The critical predicates are:

arc2_fec_all_comps_/5: finds all BG components via arc2_fec_find_comps_/7 and
arc2_fec_bfs_comp_/9, accumulating a shared visited set across all component BFS
runs so each BG cell is processed exactly once.

arc2_fec_frame_info_/11: derives FSAdj, FDom, FOthers (via arc2_fec_pure_frame_/5),
VDecContam, FrameSet, with the empty-FSAdj fallback.

arc2_fec_is_pocket_/5: checks the FrameSet-neighbor condition for each cluster.

arc2_fec_clean_ind_/8: finds a clean VAdj indicator by checking frame neighbors,
then doing VA-chain BFS and testing for VDec adjacency.

arc2_fec_classify_all_/10: dispatches to multi-frame or single-frame logic.


5. RESULTS

All 4 training pairs and the test case of task 8b7bacbf pass.
Runtime: approximately 0.64 seconds for all 5 cases.
Score: 37/120 = 30.83% advancing to 38/120 = 31.67%.
