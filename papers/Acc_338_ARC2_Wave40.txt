ABSTRACT

Wave 40 of the PrologAI ARC-AGI-2 benchmark climb adds the grid_tile named
rule, which solves task eee78d87. The rule reads the 3x3 bounding box of
non-background colored cells from a 6x6 input grid, decodes four tiling
constants from the nine cell positions of that box, tiles a 16x16 output grid
using a period-3 repeating pattern governed by those constants, and then
highlights the central 6x6 region of the output by replacing all
background-zero positions with 9. After Wave 40, PrologAI solves 38 of 120
ARC-AGI-2 tasks (31.67%) using pure Prolog named rules with no machine
learning.


1. TASK DESCRIPTION

Task eee78d87 presents 6x6 input grids with background value 7. Each input
contains a small colored shape whose non-background cells fit entirely within a
3x3 bounding box. The output is always 16x16. Every output exhibits a
period-3 repeating tile pattern constructed from two values (0 and 7), and the
central 6x6 region of the output (rows 5 through 10, columns 5 through 10)
has all 0-background positions replaced by 9.

The input shape encodes the tile: the nine positions of the 3x3 bounding box
map to four tiling constants that fully determine the output background.


2. PATTERN ANALYSIS

2.1 Tiling Structure

The 16x16 output uses a period-3 grid with dividers at every column and row
index divisible by 3 (positions 0, 3, 6, 9, 12, 15). This partitions the grid
into four position types depending on (r mod 3, c mod 3):

  corner   : r mod 3 = 0 and c mod 3 = 0  (divider-row intersects divider-col)
  h_edge   : r mod 3 = 0 and c mod 3 != 0 (divider-row, non-divider col)
  v_edge   : r mod 3 != 0 and c mod 3 = 0 (non-divider row, divider-col)
  block    : r mod 3 != 0 and c mod 3 != 0 (interior 2x2 cell content)

Each position type takes a value of either 0 or 7 uniformly across the entire
16x16 background.

2.2 Decoding Tiling Constants from the Input Shape

The 3x3 bounding box of the colored cells has nine positions indexed (dr, dc)
with dr, dc in {0, 1, 2}. Each position is either background (7) or colored
(non-7). The four constants are derived as:

  block   = 7  if all four corners  (0,0), (0,2), (2,0), (2,2)  are background
  v_edge  = 7  if top-bottom midpoints  (0,1) and (2,1)  are both background
  h_edge  = 7  if left-right midpoints  (1,0) and (1,2)  are both background
  corner  = 7  if center  (1,1)  is background

Otherwise each constant is 0.

The naming reflects the connectivity axis: the top-bottom midpoints lie on the
vertical axis of the 3x3 box and determine the vertical-divider stripe (v_edge),
while the left-right midpoints determine the horizontal-divider stripe (h_edge).

2.3 Active Region

The central 6x6 region rows 5 through 10, cols 5 through 10, is the active
region. Within this region every cell whose background tiling value is 0 is
changed to 9. Cells with background value 7 remain 7. Outside the active region
the background tiling is used as-is.

2.4 Training Example Verification

Training pair 1: cross of 3s at rows 3-5, cols 3-5.
  Bounding box corners all 7 -> block=7. Midpoints all colored -> h_edge=v_edge=0.
  Center colored -> corner=0. Tiling: 7 only at (r%3!=0, c%3!=0).

Training pair 2: H-shape of 1s at rows 1-3, cols 1-3.
  Corners colored -> block=0. Top+bottom mids both 7 -> v_edge=7.
  Left+right mids colored -> h_edge=0. Center colored -> corner=0.
  Tiling: 7 only at (r%3!=0, c%3=0).

Training pair 3: X-shape of 6s at rows 1-3, cols 1-3.
  Corners colored -> block=0. Top+bottom mids both 7 -> v_edge=7.
  Left+right mids both 7 -> h_edge=7. Center colored -> corner=0.
  Tiling: 7 at (r%3=0, c%3!=0) or (r%3!=0, c%3=0).

Test: 4-corners shape of 2s at rows 0-2, cols 2-4.
  Corners colored -> block=0. Top+bottom mids both 7 -> v_edge=7.
  Left+right mids both 7 -> h_edge=7. Center is 7 -> corner=7.
  Tiling: 7 wherever r%3=0 or c%3=0.

All three training pairs and the test case verified PASS.


3. IMPLEMENTATION

3.1 Predicate Summary

  arc2_transform(grid_tile, +Grid, -Out)
      Entry point. Calls arc2_gt_bbox_, arc2_gt_tile_, arc2_gt_build_.

  arc2_gt_bbox_(+Grid, -Rmin, -Cmin)
      Finds bounding box top-left of non-7 cells via findall over all rows/cols.

  arc2_gt_tile_(+Grid, +R0, +C0, -Block, -HEdge, -VEdge, -Corner)
      Reads nine sub-grid values; maps positional structure to tiling constants.

  arc2_gt_gv_(+Grid, +R, +C, -V)
      Extracts value at (R,C) using nth0/3 twice.

  arc2_gt_build_(+Block, +HEdge, +VEdge, +Corner, -Out)
      Builds 16x16 list-of-lists by mapping arc2_gt_cell_ over all (r,c).

  arc2_gt_cell_(+R, +C, +Block, +HEdge, +VEdge, +Corner, -V)
      Applies period-3 tiling and active-region 0->9 replacement.

3.2 Key Design Notes

The bounding box approach works because every training and test input has exactly
one 3x3 colored region with all non-background cells within that box. The
findall + min_list approach is robust and does not require knowing the grid
dimensions in advance.

The period-3 classification by (r mod 3, c mod 3) is the critical structural
insight. The output background is entirely determined by four scalar constants,
and these constants map directly and independently to four sub-positions of the
input bounding box.

The active region (rows 5-10, cols 5-10) is a fixed central 6x6 of the 16x16
output. This region is independent of where in the 6x6 input the colored cells
appear: even though different training pairs place colored cells at rows 3-5 and
rows 1-3, the active region in the output is always the same center block.
