TASK DESCRIPTION

Task d35bdbdc presents grids containing:
- A snake path of 5-valued cells that persists unchanged in the output.
- A set of frame blocks: either 3x3 squares (8 border cells + 1 center cell)
  or cross shapes (4 arm cells + 1 center cell).
- Each frame block has a center color CV and an arm color AV.
  For monochrome blocks all cells share one color, so CV = AV.

The transformation rule: build an absorption graph where an edge X -> Y
means X.CV = Y.AV (X can absorb Y). Process the graph to compute which
blocks absorb others (gaining a new center color) and which are erased.

FRAME_ABSORB RULE

Step 1 - Identify frame blocks.
Collect all non-background (0), non-snake (5) cell positions and partition
them into 4-connected components via BFS. For each component:
- If the component has exactly two distinct colors, the singleton color is CV
  (the center) and the majority color is AV (the arms). The center cell is
  the unique cell whose grid value equals CV.
- If the component is monochrome (one color), CV = AV, and the center cell
  is the cell with the highest 4-connectivity degree within the component.

Step 2 - Compute the absorption action sequence.
The solver processes blocks in this priority order:

Priority 1 (monochrome first): If any block Y has CV = AV, check whether a
block X exists such that X.CV = Y.AV. If so, X absorbs Y: X's center cell
is repainted to Y's center color (Y.CV), and Y's cells are all erased. If
no predecessor X exists, Y is a dead end and is erased outright.

Priority 2 (chain starts): After all monochrome blocks are handled, find a
block X that has no predecessor in the remaining pool (no block Z with
Z.CV = X.AV). If X has a successor Y (X.CV = Y.AV), X absorbs Y as above.
If X has no successor, X is a dead end and is erased.

Cycle fallback: If no monochrome block and no start block can be found
(all blocks form a cycle with no predecessor), erase all remaining blocks.

Step 3 - Execute actions.
For each absorb(X, Y) action: erase all cells of Y (set to 0); repaint
X's center cell with Y's original center color (Y.CV).
For each erase(B) action: set all cells of B to 0.
The snake (5-valued) cells are never touched by any action.

IMPLEMENTATION

The rule is implemented as a set of arc2_fa_* helper predicates in
arc_benchmark_2.pl (WP-288, Layer 263). The key predicates are:

arc2_transform(frame_absorb, Grid, Out):
  Entry point. Collects non-0, non-5 positions; partitions into components;
  identifies blocks; solves absorptions; applies actions via foldl.

arc2_fa_comps_(Positions, Components):
  BFS partition of positions into 4-connected components.

arc2_fa_block_(Grid, Cells, block(CR,CC,CV,AV,Cells)):
  Classifies one component as a frame block. Two-color: singleton = CV.
  Monochrome: max-degree cell = center.

arc2_fa_solve_(Blocks, Actions):
  Recursive solver implementing the three-priority action sequence.

arc2_fa_exec_(Action, G0, G1):
  Applies one absorb or erase action to the grid.
