ABSTRACT

We report the completion of Track A of the Mentova Demonstration and Proof-of-Concept
Plan: the Transparent Reasoning Assistant. Track A is the no-hardware practical track
specified in Volume 6, Part 6 of the PrologAI SPARC series. Mentova answers ten expert
biomedical questions over two real ontologies - the Gene Ontology (GO) and the Disease
Ontology (DO) - each loaded into its own scope, with cross-scope queries linking them.
Every answer includes a human-readable justification chain built entirely from named
node_facts. The Gene Ontology multi-scope showpiece passes: deep taxonomic deduction
across four-hop is_a chains, gene function and location queries, cross-scope gene-disease
explanation, shared-gene discovery between diseases, and reverse function-to-disease
lookup - all glass-box, all auditable, requiring no robot, no neural network, and no
opaque weights.


INTRODUCTION

The 48-rung reasoning ladder (Acc_01 through Acc_48) proved that Mentova can reason
across the full human-cognitive-science taxonomy in isolation. Track A proves that
Mentova is practically useful: it can answer real expert questions over real ontologies
and show every step of its reasoning.

Volume 6, Part 6 specifies two practical tracks. Track A requires no hardware: it is a
glass-box helper that poses a question, returns the answer, and shows the readable chain
of node_facts and justifications that produced it. The Gene Ontology multi-scope
showpiece is the stated expert-mode demonstration for this track.

This accomplishment record documents the Track A implementation and its ten demonstration
queries, each of which passes the stated criterion: the answer arrives with a
human-readable justification that names the specific node_facts used.


BACKGROUND AND RELATED WORK

The Gene Ontology (GO) is a structured vocabulary and annotation database that describes
gene functions across three sub-ontologies: Biological Process (BP), Molecular Function
(MF), and Cellular Component (CC). It is maintained by the Gene Ontology Consortium
and uses four principal relations: is_a (taxonomic subsumption), part_of (mereological
containment), regulates (regulatory influence), and has_part (inverse mereology).

The Disease Ontology (DO) is a structured vocabulary for human diseases, maintained by
the University of Maryland. It uses is_a hierarchies to classify diseases and captures
gene-disease associations curated from literature.

Both ontologies are deeply hierarchical: GO contains over 43,000 terms; DO contains over
10,000 terms. Expert reasoning over them requires transitive closure over is_a chains -
exactly the kind of symbolic reasoning that PrologAI's Lattice knowledge store and
backward-chaining deduction engine are built for.

The PrologAI scope system (PR 5) enables loading each ontology into a named compartment
so that GO terms, DO terms, and the Small-World Commonsense knowledge base coexist
without namespace collisions. Every fact is a node_fact; the scope system enforces
nine-zone Lattice compartmentalization with full provenance.

This work is directly specified in Volume 6, Part 4 (the Gene Ontology multi-scope
showpiece as the expert mode of Track A) and Part 6 (Track A pass criterion).


METHOD

The Track A implementation adds four new files to the Mentova repository:

  knowledge/gene_ontology.pl
    A curated toy subset of GO covering all three sub-ontologies. Forty-five GO terms
    are registered via go_term/2 facts. The is_a hierarchy is encoded as go_is_a/2
    facts reaching up to four hops from leaf to root. Part-of containment is encoded
    as go_part_of/2. Regulatory relations are go_regulates/2. Gene annotations for five
    genes (BRCA1, TP53, EGFR, PTEN, KRAS) are encoded as go_annotation/3 facts with
    evidence codes (experimental or inferred). Derived predicates go_is_a_chain/3 and
    go_ancestor/2 compute transitive closure.

  knowledge/disease_ontology.pl
    A curated toy subset of DO covering cancers, neurological disorders, and metabolic
    diseases. Twenty-two disease terms are registered via disease_term/2 facts. The
    is_a hierarchy is encoded as disease_is_a/2 facts reaching up to five hops from
    leaf to root. Gene-disease associations for the same five genes are encoded as
    disease_gene/3 facts with evidence codes (curated or inferred). Symptom facts are
    disease_symptom/2. Derived predicates disease_is_a_chain/3 and disease_ancestor/2
    compute transitive closure.

  src/mentova/track_a.pl
    The Track A reasoning module. Exports mentova_track_a/3, the glass-box query
    predicate. Query families:
      GO scope:     gene_function/1, gene_location/1, gene_process/1,
                    go_classify/2, go_ancestors/1
      DO scope:     gene_diseases/1, disease_genes/1, disease_classify/2,
                    disease_symptoms/1
      Cross-scope:  explain_disease/1, shared_genes/2, function_to_disease/1
    Every clause returns both a Result and a Justification term built from named
    node_facts. No inference step is hidden; the justification names every fact used.

  demos/track_a_demo.pl
    The demonstration driver. Loads Mentova, boots it, and runs all ten queries,
    printing both the answer and its justification for each.

The mentova_query/3 predicate in the Mentova bootstrap was extended with a track_a
clause that delegates to mentova_track_a/3, so Track A queries are accessible through
the same top-level interface as all 48 reasoning rungs.

RESULTS

Pass criterion met: all ten demonstration queries return answers with human-readable
justification chains built from named node_facts.

Key results by query type:

  GO Scope - gene function (Q1):
    BRCA1 is annotated with go_0003677 (dna_binding, experimental evidence).
    The inferred chain shows dna_binding is_a molecular_function in two hops.
    Direct annotation returned as the primary result; inferred ancestor included
    in the justification for full provenance.

  GO Scope - gene location (Q2):
    BRCA1 localises to nucleoplasm (go_0005654, experimental evidence).
    The justification names the specific GO cellular component term and its
    evidence code.

  GO Scope - deep taxonomy (Q3):
    DNA repair (go_0006281) is confirmed as a descendant of biological_process
    (go_0008150) via a four-hop is_a chain:
    go_0006281 (dna_repair)
      -> go_0006259 (dna_metabolic_process)
      -> go_0008152 (metabolic_process)
      -> go_0008150 (biological_process)
    Each step is a named go_is_a/2 node_fact. The chain is printed in full.

  GO Scope - molecular function taxonomy (Q4):
    protein_tyrosine_kinase_activity (go_0004713) is confirmed as a descendant
    of molecular_function (go_0003674) via a five-hop chain:
    go_0004713 -> go_0004672 -> go_0016301 -> go_0003824 -> go_0003674
    This is the longest chain in the GO molecular function subset and demonstrates
    that the transitive closure is computed correctly without a depth limit.

  DO Scope - gene-disease associations (Q5):
    TP53 is associated with four diseases: cancer, lung_cancer, colorectal_cancer,
    and breast_cancer. All four associations are curated (manually reviewed). The
    justification lists every disease_gene/3 node_fact used.

  DO Scope - disease taxonomy (Q6):
    Glioblastoma (doid_3068) is confirmed as a kind of disease (doid_4) via a
    five-hop chain:
    doid_3068 (glioblastoma)
      -> doid_1319 (brain_cancer)
      -> doid_162 (cancer)
      -> doid_14566 (disease_of_cellular_proliferation)
      -> doid_4 (disease)
    The longest chain in the DO subset; computed correctly by disease_is_a_chain/3.

  DO Scope - disease gene lookup (Q7):
    Breast cancer (doid_1612) is associated with three genes: BRCA1, TP53, and PTEN,
    all with curated evidence. The justification lists every disease_gene/3 node_fact.

  Cross-scope - disease explanation (Q8):
    The explain_disease query for breast cancer crosses both scopes: it first looks up
    disease_gene/3 facts in the DO scope, then looks up go_annotation/3 facts in the
    GO scope for each gene. Result: BRCA1 contributes dna_binding; TP53 contributes
    dna_binding_transcription_factor_activity; PTEN contributes
    protein_tyrosine_phosphatase_activity. The justification names every fact used
    in both scopes.

  Cross-scope - shared genes (Q9):
    TP53 is the only gene shared between breast cancer (doid_1612) and lung cancer
    (doid_1324). The query computes set intersection across two disease_gene/3 lookups.
    The justification names both diseases and the shared gene.

  Cross-scope - function to disease (Q10):
    Signal transduction (go_0007165) is annotated on EGFR and KRAS. EGFR is
    associated with lung_cancer, glioblastoma, and colorectal_cancer; KRAS is
    associated with pancreatic_cancer, colorectal_cancer, and lung_cancer.
    The query returns four unique diseases: colorectal_cancer, glioblastoma,
    lung_cancer, pancreatic_cancer. The justification traces every link across
    both scopes.


SIGNIFICANCE

Track A turns the 48-rung ladder from a demonstration into a tool.

The ladder proved that Mentova can reason. Track A proves that Mentova can be
useful: it answers real expert questions over real ontologies, with the chain of
node_facts that produced every answer shown in full.

Three properties distinguish this from black-box biomedical question-answering systems:

First, auditability. Every answer names the specific facts used: go_annotation/3,
disease_gene/3, go_is_a/2, disease_is_a/2. A domain expert can read the justification,
check each fact against the source ontology, and verify or dispute the conclusion.

Second, scope isolation. The Gene Ontology and Disease Ontology live in separate
scopes. A cross-scope query explicitly bridges them; a single-scope query cannot
inadvertently mix terms. This is a structural guarantee, not a convention.

Third, honest provenance. Every node_fact carries an evidence code: experimental,
inferred, curated, or inferred. The justification propagates these codes to the answer,
so the user knows exactly how reliable each step is.

No black-box model demonstrates all three properties together. This is what the
glass-box principle delivers.

Track A is also the foundation for the Gene Ontology multi-scope showpiece specified in
Volume 6, Part 4: loading GO plus a second biomedical ontology (here, DO) into separate
scopes as an expert-mode demonstration that exercises the scope system and deep taxonomic
deduction. Both criteria are met.


HONEST LIMITATIONS

The knowledge bases are deliberately toy-sized. GO contains over 43,000 terms; this
subset contains 45. DO contains over 10,000 terms; this subset contains 22. The curated
gene annotations cover five genes. A production deployment would load the full OBO-format
ontologies programmatically.

The transitive closure predicates (go_is_a_chain/3, disease_is_a_chain/3) use
depth-first search without a visited-node check. For the toy subsets used here, no
cycles exist. For larger ontologies, a cycle-detection mechanism would be required.

Cross-scope queries currently require the user to specify the scope explicitly in the
query term. A routing layer that infers the correct scope from the query content would
make the interface more natural; that is planned as a Track A extension.

The evidence codes in this demonstration (experimental, curated, inferred) are
hand-assigned for the toy subset. A full implementation would parse GO evidence codes
(IDA, IMP, IPI, ISS, IEA, etc.) from the source OBO annotation files.


REPRODUCIBILITY

Requirements: SWI-Prolog 9.0.4 or later.

  % Clone the Mentova repository to your local machine.
  git clone https://github.com/ai-university-aiu/Mentova.git
  % Change into the Mentova directory.
  cd Mentova
  % Load the Track A demonstration script, run it, and halt SWI-Prolog.
  swipl -l demos/track_a_demo.pl -g "run_track_a" -t halt

Individual Track A queries can be run interactively:

  % Load the Track A module into SWI-Prolog interactively.
  swipl -l src/mentova/track_a.pl
  % Query Mentova's Track A for the molecular functions of BRCA1.
  ?- mentova_track_a(gene_function(brca1), Result, Justification).
  % Query Mentova's Track A for the diseases associated with TP53.
  ?- mentova_track_a(gene_diseases(tp53), Result, Justification).
  % Query Mentova's Track A to explain breast cancer via gene functions.
  ?- mentova_track_a(explain_disease(doid_1612), Result, Justification).

Or through the top-level mentova_query/3 interface:

  % Load the Mentova bootstrap into SWI-Prolog interactively.
  swipl -l src/mentova/mentova.pl
  % Boot Mentova then query Track A for a GO classification.
  ?- mentova_boot, mentova_query(track_a, go_classify(go_0006281, go_0008150), R).
