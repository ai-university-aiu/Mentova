ABSTRACT

We report Rung 33 of Mentova's 48-rung reasoning ladder: modal reasoning.
Mentova evaluates whether propositions are necessarily true (true in all accessible
worlds), possibly true (true in some accessible world), or contingent (possible
but not necessary). Fire being hot is necessary; rain is contingent.
Pass criterion met: necessarily-true proposition correctly distinguished from
merely-possible and contingent propositions.


INTRODUCTION

Modal logic (Kripke, 1963) formalises necessity and possibility using possible
worlds and accessibility relations. Rung 33 equips Mentova with a four-world
Kripke model and the standard modal operators.


BACKGROUND

Modal logic underpins temporal reasoning, belief revision, and deontic logic.
The system implemented here is S5-like: actual world has access to all worlds.


METHOD

A new module, src/mentova/modal.pl, implements:

    mentova_modal(+Query, -Result, -Justification)

Worlds: actual, w1, w2, w3. Facts assigned to worlds. Operators: necessarily/1,
possibly/1, contingent/1, classify/1.


DEMONSTRATION

    % Query Mentova for modal reasoning about 'necessarily(fire_is_hot)'.
    ?- mentova_query(modal, necessarily(fire_is_hot), R).
    R = answer(true, just(modal(necessarily(fire_is_hot), worlds_checked(all_accessible), result(true))))

    % Query Mentova for modal reasoning about 'classify(it_rains)'.
    ?- mentova_query(modal, classify(it_rains), R).
    R = answer(contingent, just(modal(classify(it_rains), classification(contingent))))


RESULTS

fire_is_hot: necessary (true in all 4 worlds). it_rains: contingent (true in 2 of 4).
Pass criterion met.


SIGNIFICANCE

Modal reasoning is essential for natural language understanding, temporal planning,
and belief reasoning. Necessity and possibility are fundamental epistemic categories.


HONEST LIMITATIONS

The possible-worlds model is static and small. Full modal reasoning requires
dynamic world generation and modal axiom systems.


REPRODUCIBILITY

Requirements: SWI-Prolog 9.0.4 or later.

  git clone https://github.com/ai-university-aiu/Mentova.git
  cd Mentova
  % Load the Mentova bootstrap into SWI-Prolog, execute the query, and halt.
  swipl -l src/mentova/mentova.pl \
        % Supply a goal or termination instruction to the SWI-Prolog command.
        -g "mentova_boot, mentova_query(modal, classify(it_rains), R), write(R), nl" \
        % Supply a goal or termination instruction to the SWI-Prolog command.
        -t halt
