ABSTRACT

Mentova's thirty-third accomplished reasoning rung is modal reasoning: correctly
distinguishing propositions that are necessarily true (true in all accessible
worlds), possibly true (true in at least one world), and contingent (true in
some but not all). Demonstrated: fire_is_hot is necessary, birds_fly is
necessary, tweety_flies is contingent (true in w1 and w2 but not in actual or
w3), and water_is_wet is necessary. All results include the worlds checked.


INTRODUCTION

Modal logic extends classical logic with operators for necessity (□) and
possibility (◇). A proposition is necessary if true in all accessible worlds,
possible if true in at least one, contingent if neither necessary nor impossible.

Pass criterion (Rung 33): "necessity and possibility correctly distinguished."


METHOD

The modal module (src/mentova/modal.pl) defines four worlds (actual, w1, w2, w3)
and a set of `holds(Fact, World)` facts. Three predicates:

    necessarily(P) — true if holds(P, W) for every W
    possibly(P)    — true if holds(P, W) for at least one W
    classify(P)    → necessary | contingent | impossible

mentova_modal/3 exposes these via mentova_query(modal, ...).


THE DEMONSTRATION

QUERY 1 — Is fire_is_hot necessary?
    necessarily(fire_is_hot) holds in actual, w1, w2, w3 → necessary: true
    Justification: worlds_checked(all_accessible), result(true)

QUERY 2 — Is tweety_flies possible?
    holds(tweety_flies, w1), holds(tweety_flies, w2) → witness world found
    possibly(tweety_flies): true, witness_world(w1)

QUERY 3 — Classify birds_fly
    holds(birds_fly, actual), holds(birds_fly, w1), holds(birds_fly, w2),
    holds(birds_fly, w3) → classify → necessary

QUERY 4 — Classify tweety_flies
    Holds in w1, w2 but not in actual or w3 → classify → contingent


RESULTS

Pass criterion met: necessary propositions (fire_is_hot, birds_fly) are
distinguished from contingent ones (tweety_flies). The justification names
the worlds checked and, for possibility queries, the witness world.


SIGNIFICANCE

Modal reasoning lets Mentova express the difference between "this always
holds" and "this might hold." This is essential for planning under uncertainty
(an action that possibly achieves the goal vs. one that necessarily does).


HONEST LIMITATIONS

Accessibility relation is currently universal (every world accesses every
other). S5 axioms hold by construction. Non-trivial accessibility structures
(e.g., temporal or epistemic Kripke frames) are not yet modelled.
