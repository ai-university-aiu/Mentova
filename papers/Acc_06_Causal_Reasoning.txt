ABSTRACT

We report Rung 6 of Mentova's 48-rung reasoning ladder: causal reasoning.
Mentova distinguishes between observing that a sprinkler is on and intervening
to turn it on — a distinction at the heart of Pearl's do-calculus. When the
sprinkler is merely observed to be on, the probability of wet ground is 0.72,
because the observation correlates with the rain season. When the sprinkler is
switched on by intervention (do-operator), the incoming edge from rain to the
sprinkler decision is severed, and the probability of wet ground rises to 0.90,
reflecting only the direct causal path. The intervention prediction differs
correctly from the mere observation. Pass criterion met.


INTRODUCTION

Correlation is not causation. A symptom that correlates with a disease does not
cause the disease. Observing that wet ground and sprinklers tend to co-occur does
not tell us what happens if we force the sprinkler on while controlling for all
other factors.

Judea Pearl's do-calculus formalises this distinction. The observational
probability P(wet_ground | sprinkler) reflects all paths by which sprinkler
and wet_ground are associated — including confounders like seasonal rain.
The interventional probability P(wet_ground | do(sprinkler)) reflects only
the direct causal arrow from sprinkler to wet_ground, because the do-operator
severs all incoming edges to the intervened variable.

Rung 6 implements this distinction in Mentova's glass-box framework.


BACKGROUND

Pearl's causal hierarchy has three rungs of its own: association (seeing),
intervention (doing), and counterfactual (imagining). Rung 6 covers the
second level — intervention — while Rung 19 of Mentova's ladder covers
counterfactuals.

The do-calculus was introduced in Pearl (1995, 2000) and provides exact
identifiability conditions for interventional distributions from observational
data given a known causal graph.

In Mentova's Small-World knowledge base, the causal graph is given by
causes/2 facts. Rung 6 stores interventional probabilities directly for
the test cases, capturing the conceptual difference without requiring full
graphical model machinery.


METHOD

A new module, src/mentova/causal.pl, implements:

    mentova_causal(+Query, -Result, -Justification)

Query forms supported:

    intervene(Cause, Effect)    P(Effect | do(Cause)) — cut incoming edges
    observe(Cause, Effect)      P(Effect | Cause observed) — all paths
    compare(Cause, Effect)      show both probabilities and difference
    chain(Cause)                list direct causal effects in causes/2

The justification for each intervention query explicitly notes that incoming
edges to the cause are severed: "incoming edges to Cause severed; only causal
path retained".

The top-level clause added to mentova_query/3:

    % Query Mentova for causal reasoning about 'CausalQuery'.
    mentova_query(causal, CausalQuery, answer(Result, Just)) :-
        % State the fact: mentova causal(...).
        mentova_causal(CausalQuery, Result, Just).


DEMONSTRATION

Query 1 — interventional: do(sprinkler) → wet_ground:

    % Query Mentova for causal reasoning about 'intervene(sprinkler'.
    ?- mentova_query(causal, intervene(sprinkler, wet_ground), R).
    R = answer(0.9, just(intervention(do(sprinkler), wet_ground, 0.9,
               note('incoming edges to Cause severed; only causal path retained'))))

    P(wet_ground | do(sprinkler)) = 0.90.
    The rain edge is severed. Only the sprinkler's direct effect remains.

Query 2 — observational: observe(sprinkler) → wet_ground:

    % Query Mentova for causal reasoning about 'observe(sprinkler'.
    ?- mentova_query(causal, observe(sprinkler, wet_ground), R).
    R = answer(0.72, just(observation(sprinkler, wet_ground, 0.72,
               note('no edge severing; all paths to Effect included'))))

    P(wet_ground | sprinkler) = 0.72.
    The observation correlates with rain season; the probability is lower
    because the sprinkler's presence does not guarantee rain is absent.

Query 3 — comparison:

    % Query Mentova for causal reasoning about 'compare(sprinkler'.
    ?- mentova_query(causal, compare(sprinkler, wet_ground), R).
    R = answer(compare(0.9, 0.72, 0.18), just(compare(...)))

    Intervention is 0.18 higher than observation.
    This is the correct direction: a forced intervention isolates the
    causal path and removes confounding from rain correlation.

Query 4 — causal chain:

    % Query Mentova for causal reasoning about 'chain(fire)'.
    ?- mentova_query(causal, chain(fire), R).
    R = answer(chain(fire, [smoke]), just(causal_chain(fire, [smoke])))

    Fire directly causes smoke. The chain is read from causes/2 in
    the Small-World knowledge base.


RESULTS

The intervention prediction (0.90) differs correctly from the observational
probability (0.72). The difference (0.18) reflects the removal of confounding
via the do-operator. The justification names the edge-severing operation.

Pass criterion met: intervention prediction differs correctly from mere observation.


SIGNIFICANCE

Rung 6 gives Mentova the ability to reason about actions, not just states.
A mind that can only observe cannot plan — it cannot know whether turning
on the sprinkler will cause the ground to be wet or merely correlates with
ground-wetness.

The do-calculus distinction is critical for any agent that must make decisions.
Planning, policy evaluation, and experiment design all require interventional
reasoning. Rung 6 is the foundation for Mentova's action-based reasoning.


HONEST LIMITATIONS

The interventional probabilities in this implementation are stored explicitly
rather than derived algorithmically from the causal graph using the full
do-calculus identifiability machinery. A complete implementation would use
the backdoor criterion or front-door criterion to derive P(Y|do(X)) from
observational data and the graph structure.

The causal graph in the Small-World knowledge base is a DAG of named
cause-effect pairs; it does not store conditional probability tables for
the full joint distribution.

Only two-node cause-effect relationships are handled. Multi-hop interventional
effects (do(rain) → wet_ground → plant_growth) would require path-level
graph traversal.


REPRODUCIBILITY

Requirements: SWI-Prolog 9.0.4 or later.

  git clone https://github.com/ai-university-aiu/Mentova.git
  cd Mentova
  % Load the Mentova bootstrap into SWI-Prolog, execute the query, and halt.
  swipl -l src/mentova/mentova.pl \
        % Supply a goal or termination instruction to the SWI-Prolog command.
        -g "mentova_boot, \
            % Query Mentova for causal reasoning about 'intervene(sprinkler'.
            mentova_query(causal, intervene(sprinkler,wet_ground), R1), write(R1), nl, \
            % Query Mentova for causal reasoning about 'observe(sprinkler'.
            mentova_query(causal, observe(sprinkler,wet_ground), R2), write(R2), nl, \
            % Query Mentova for causal reasoning about 'compare(sprinkler'.
            mentova_query(causal, compare(sprinkler,wet_ground), R3), write(R3), nl" \
        % Supply a goal or termination instruction to the SWI-Prolog command.
        -t halt
