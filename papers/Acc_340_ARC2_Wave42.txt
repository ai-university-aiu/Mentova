ABSTRACT

Wave 42 of the PrologAI ARC-AGI-2 benchmark climb adds the section_sort named
rule, which solves task 78332cb0. The rule locates all-6 divider rows and columns
that partition a grid into rectangular sections, counts the non-background cells
in each section, and reassembles the sections in ascending order of cell count.
The output direction (vertical stack or horizontal row) is the opposite of the
input direction; when the input is 2-dimensional, the output direction is
determined by comparing total section bounding-box heights against widths. After
Wave 42, PrologAI solves 40 of 120 ARC-AGI-2 tasks (33.33%) using pure Prolog
named rules with no machine learning.


1. TASK DESCRIPTION

Task 78332cb0 presents grids divided by rows or columns of the value 6 into
rectangular sections. Each section contains a small figure drawn in one or more
non-background, non-6 colors. The background value is 7 in all observed inputs.

The output is a reassembly of the same sections. The sections are sorted by their
non-background cell count in ascending order (fewest cells first). The output
direction alternates relative to the input: a vertical stack of sections becomes
a horizontal row, a horizontal row becomes a vertical stack. A 2-dimensional
arrangement (both row-dividers and column-dividers present) is sorted ascending
with ties broken by reverse reading order, and the output direction is chosen by
comparing the sum of section bounding-box heights against the sum of widths.


2. PATTERN ANALYSIS

2.1 Divider Detection

Every row in which all cells equal 6 is a horizontal divider. Every column in
which all cells equal 6 is a vertical divider. The dividers partition the grid
into a set of non-overlapping rectangular sections.

2.2 Cell Count Sorting

The non-background, non-6 cells in each section are counted. Sections are placed
in ascending order of this count. In the 1-dimensional cases (all dividers in
one axis), this ordering is unambiguous. In the 2-dimensional case, ties are
broken by reverse reading order: a section with a higher original index (further
down-right in reading order) is placed before a section with the same count.

2.3 Direction Reversal

For a vertical-stack input (horizontal dividers, no vertical dividers), the
sections are output as a horizontal row (separated by single-cell 6-columns). For
a horizontal-row input (vertical dividers, no horizontal dividers), the sections
are output as a vertical stack (separated by single-cell 6-rows). For a
2-dimensional input (both types of dividers present), the output direction is
determined at runtime: if the sum of section bounding-box heights exceeds the sum
of their widths, the sections are stacked vertically; otherwise horizontally. The
2-dimensional case also reverses the overall sort order relative to the
vertical-stack case.

2.4 6-Divider Preservation

6-divider rows or columns are re-inserted as single rows or columns of 6 between
every pair of adjacent sections in the output. The dividers are exactly one cell
wide regardless of how many divider rows or columns appeared between sections in
the input.

Pair 1 (vertical stack, 3 sections): sections sorted ascending; output is a
  horizontal row of 3 sections separated by 6-columns.

Pair 2 (horizontal row, 3 sections): sections sorted ascending; output is a
  vertical stack of 3 sections separated by 6-rows.

Pair 3 (2D, 4 sections in 2x2): bounding-box height sum vs width sum determines
  output direction; sections sorted ascending with tie-breaking by reverse
  reading index.

Test (2D, similar to Pair 3): same 2D logic applied; all four cases confirmed
  by Python simulation and Prolog verification.


3. IMPLEMENTATION

The rule is implemented in arc_benchmark_2.pl (Mentova repository) as nine
predicates all prefixed arc2_ss_:

arc2_ss_divs_ finds all horizontal divider rows (every cell = 6) and all vertical
divider columns (every cell = 6) using include/3 with partial applications of
arc2_ss_hd_ and arc2_ss_vd_.

arc2_ss_segs_ converts a sorted list of divider positions into (Start, End) pairs
for the segments between them. It prepends -1 and appends N to the divider list,
then uses maplist with a lambda to compute A+1 and B-1 for each adjacent pair.

arc2_ss_kind_ classifies the arrangement: h (horizontal row, vertical dividers
only), v (vertical stack, horizontal dividers only), or td (2-dimensional, both).

arc2_ss_extract_ builds a list of sec(Grid, Index, Count) terms. It uses findall
over all (RI, CI) row-segment and column-segment index combinations to enumerate
sections in reading order, extract each section's sub-grid via arc2_ss_subgrid_,
and count non-background, non-6 cells via arc2_ss_ncount_.

arc2_ss_order_ dispatches on Kind to produce the sorted section list. For v
(vertical stack), it reverses the section list. For h (horizontal row), it
preserves the original order (already ascending by construction). For td
(2-dimensional), it uses keysort on compound keys (N, -Index) to sort ascending
by count with reverse-reading-order tie-breaking, then extracts values with
pairs_values/2.

arc2_ss_outdir_ dispatches on Kind. Vertical stack maps to horizontal output; horizontal
row maps to vertical output. For 2-dimensional input, arc2_ss_bbox_ computes the
bounding box of non-background, non-6 cells in each section; the sum of heights is
compared against the sum of widths to choose the output direction.

arc2_ss_build_ assembles the output. For vertical output (v), it stacks all
section grids with single 6-divider rows inserted between them using
arc2_ss_vstack_. For horizontal output (h), it uses a maplist over row indices to
concatenate corresponding rows from each section grid with single 6-divider
columns using arc2_ss_hstack_.
