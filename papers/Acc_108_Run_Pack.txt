Accomplishment 108: Run-Length Encoding of Grid Sequences - The Run Pack

Author: D. R. Dison, Founder of AIU, Creator and Owner of PrologAI and Mentova, ORCID 0009-0001-9246-5758, linkedin.com/in/d-r-dison/

Date: 2026-06-25

Abstract

Mentova's cognitive substrate gains its 86th work package: the run pack (rn_* predicate family, Layer 67). This pack provides 14 predicates for run-length encoding and decoding of value sequences and grid rows and columns. Run-length encoding transforms a list like [1,1,2,2,2,1] into a compact list of Value-Count pairs [(1-2), (2-3), (1-1)] that explicitly represents repeated patterns. The run pack gives Mentova a new vocabulary for reasoning about sequences: it can now measure the length of repeating runs, find the longest run of a given color, count how many times a color appears as a distinct run, test whether an entire row or column is uniform, trim background noise from sequences, and enumerate the exact positions of every cell with a given value.

Introduction

Many visual reasoning tasks require understanding sequences, not just individual cells. A row of [1,1,1,0,1,0,0] is more meaningfully described as "three 1s, then a 0, then a 1, then two 0s" than as a flat list. The run-length encoding representation makes this structure explicit and queryable.

Specifically, run-length encoding is useful for:
- Detecting stripe patterns (a row or column consisting of runs of uniform color)
- Measuring the width of color bands (the run count)
- Identifying whether a row is a simple two-color alternating pattern
- Comparing row patterns by their run structure
- Finding the position of a specific color within a sequence without scanning cell by cell

The run pack provides these capabilities as first-class Prolog predicates.

Predicates

The run pack exports 14 predicates under the rn_* prefix.

Encoding and decoding:
rn_encode/2 run-length encodes a flat list to Value-Count pairs.
rn_decode/2 decodes Value-Count pairs back to a flat list.

Grid integration:
rn_row_encode/3 encodes a single row of a grid.
rn_col_encode/3 encodes a single column of a grid.
rn_grid_rows/2 encodes all rows of a grid.
rn_grid_cols/2 encodes all columns of a grid.

Structural analysis:
rn_length/2 computes the total element count of a run list.
rn_at/3 looks up the value at a 0-indexed position within the encoded sequence.
rn_max_run/3 finds the length of the longest run of a given value (0 if absent).
rn_count_runs/3 counts the number of distinct runs of a given value.
rn_uniform/1 tests whether all runs in the list share the same value.

Structural operations:
rn_trim/3 removes leading and trailing occurrences of a background value from a flat list.
rn_repeat/3 repeats the encoded sequence N times, merging adjacent same-value runs at repetition boundaries.
rn_positions/3 returns the sorted list of 0-indexed positions where a given value appears.

Implementation

The core encoding predicate rn_encode uses rn_take_same_, which collects consecutive equal values via an if-then-else structure on integer comparison. This structure eliminates choicepoints that would otherwise arise from separate matching and non-matching clauses. The same if-then-else pattern is used in rn_at (to select between the current run and the tail) and in rn_merge_adjacent_ (used by rn_repeat to merge boundary runs).

rn_uniform uses a direct clause-head pattern: rn_uniform([V-_, V-N2|Rest]) requires the first two runs to share value V, then recurses. This is more direct than a forall-based check.

rn_positions uses a recursive accumulator that tracks the current offset and collects positions range by range using numlist/3.

Significance

The run pack gives Mentova a compact, analyzable representation of repeated patterns in grid rows and columns. Combined with the pattern pack (periodicity analysis) and the induction pack (learning from examples), Mentova can now reason about striped grids, detect uniform regions, and measure the width of color bands as part of a multi-step visual reasoning process.
