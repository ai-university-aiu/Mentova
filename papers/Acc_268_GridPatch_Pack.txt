Abstract

PrologAI Work Package 248 adds the gridpatch pack (gpt_*, Layer 227): fourteen
predicates for extracting, placing, overlaying, searching, tiling, scattering,
and inpainting sub-grid regions (patches). All operations are non-destructive.
All 44 acceptance tests pass.


1. Motivation

Many spatial reasoning tasks require working with sub-grid patterns: finding a
stamp in a scene, placing it at a new location, or tiling a small pattern to fill
a larger area. The gridpatch pack provides these operations as clean, named
predicates. gpt_find_all/3 locates all occurrences of a pattern in a grid;
gpt_scatter/4 stamps it at multiple positions; gpt_tile_fill/4 tiles it; and
gpt_overlay/6 allows transparent compositing where background cells in the stamp
do not overwrite the target.


2. Architecture

Layer 227 sits immediately above gridcrop (Layer 226) in the Data Layer.
Uses library(lists): nth0/3, member/2, append/3. No cross-pack dependencies.

Key design: a private gpt_dims_/3 helper reads grid dimensions. All placement
predicates (gpt_place, gpt_overlay) use findall with nested findall for
row-major reconstruction. gpt_match_at/4 uses negation-as-failure (\+) for
efficient all-or-nothing matching. gpt_scatter/4 uses deterministic recursion
with a cut on the base case.


3. Predicates

Extraction and placement:

gpt_extract/6       - copy sub-grid at rows R0..R1, cols C0..C1
gpt_place/5         - overwrite-paste Patch at (R0,C0)
gpt_overlay/6       - transparent-paste Patch (TranspColor cells skipped)

Searching:

gpt_match_at/4      - check if Patch matches at (R0,C0)
gpt_find_all/3      - all positions where Patch matches (row-major)
gpt_count/3         - count of matching positions
gpt_replace_first/4 - replace first matching occurrence

Tiling and scattering:

gpt_tile_fill/4     - build H x W grid by repeating Patch
gpt_scatter/4       - place Patch at each listed position

Dimensions and utility:

gpt_h/2             - row count of patch
gpt_w/2             - column count of patch
gpt_size/3          - row and column count
gpt_eq/2            - patch equality check
gpt_inpaint/7       - fill rectangle [R0..R1] x [C0..C1] with Color


4. Example Session

:- use_module(pack(gridpatch/prolog/gridpatch)).

Checker = [[x,b,x,b],[b,x,b,x],[x,b,x,b],[b,x,b,x]].
Tile = [[x,b],[b,x]].

gpt_find_all(Checker, Tile, Pos).    % Pos = [0-0, 0-2, 1-1, 2-0, 2-2]
gpt_count(Checker, Tile, N).         % N = 5
gpt_tile_fill(Tile, 4, 4, G).        % G = Checker
gpt_replace_first(Checker, Tile, [[z,z],[z,z]], R).  % replaces at (0,0)

Base = [[b,b,b,b],[b,b,b,b],[b,b,b,b],[b,b,b,b]].
gpt_scatter(Base, [[m]], [0-0, 1-1, 2-2, 3-3], R2).   % diagonal of m
gpt_inpaint(Checker, 1, 1, 2, 2, b, R3).              % clear inner 2x2

7. Conclusion

The gridpatch pack provides fourteen gpt_* predicates for full patch lifecycle:
extraction, placement, overlay, pattern matching, tiling, scattering, and region
clearing. The gpt_overlay predicate enables transparent compositing. The gpt_find_all
and gpt_replace_first predicates enable pattern search-and-replace within grids.
