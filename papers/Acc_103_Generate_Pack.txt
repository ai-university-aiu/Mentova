Accomplishment 103: Grid Construction from Visual Patterns - The Generate Pack

Author: D. R. Dison, Founder of AIU, Creator and Owner of PrologAI and Mentova, ORCID 0009-0001-9246-5758, linkedin.com/in/d-r-dison/

Date: 2026-06-25

Abstract

Mentova's cognitive substrate gains its 81st work package: the generate pack (ge_* predicate family, Layer 62). This pack provides 14 predicates for constructing new grids from scratch using common visual patterns: uniform color fills, gradients, checkerboards, stripes, bordered rectangles, diagonal and anti-diagonal lines, frame rings, crosses, identity matrix patterns, explicit cell-color maps, and tiled repetitions of a smaller pattern. The generate pack closes the construction layer: Mentova can now not only analyze and transform existing grids, but create candidate output grids to compare against observed outputs.

Introduction

ARC-AGI-2 reasoning involves two fundamental directions: analysis (what is this grid like?) and synthesis (what would a grid following this rule look like?). The packs built earlier - induction, compare, noise, spatial, gravity - are all analytical. The generate pack is synthetic: it builds candidate outputs from first principles.

The most direct application is hypothesis verification. If a system hypothesizes that the output for a new input should be a 3x3 checkerboard with colors 0 and 1, it can generate that grid with ge_checkerboard(3, 3, 0, 1, Expected) and compare it to the actual output with cp_grids_equal(Expected, Actual). If they match, the hypothesis is confirmed.

The generate pack also serves as a vocabulary of visual primitives. Many ARC-AGI-2 outputs are simple named patterns. Recognizing that an output is "a frame" or "a diagonal" or "a tiling of this tile" is a hypothesis that the generate pack makes testable.

Predicates

The generate pack exports 14 predicates under the ge_* prefix.

Uniform and gradient fills:
ge_uniform/4 creates a grid filled with one color.
ge_gradient_h/4 fills column C with color C mod NColors.
ge_gradient_v/4 fills row R with color R mod NColors.

Repeating patterns:
ge_checkerboard/5 alternates two colors based on (R+C) mod 2.
ge_stripes_h/4 assigns row R to Colors[R mod len(Colors)].
ge_stripes_v/4 assigns column C to Colors[C mod len(Colors)].

Geometric patterns:
ge_border_rect/5 fills the outermost ring with BorderColor and the interior with FillColor.
ge_frame/5 is equivalent, with BG as the interior color parameter.
ge_diagonal/4 marks cells where R = C with DiagColor and zeros the rest.
ge_antidiagonal/4 marks cells where R + C = Cols - 1.
ge_identity_grid/3 is ge_diagonal specialized to a square grid.
ge_cross/5 marks the middle row and middle column with CrossColor.

Flexible construction:
ge_from_map/3 builds a grid from an explicit list of r(R,C)-Color pairs.
ge_repeat_pattern/4 tiles a smaller pattern grid to fill a larger one.

Implementation

All predicates use a combination of numlist/3 to generate row and column indices, and maplist/3 to apply a per-cell or per-row predicate. Cell-level predicates compute the color from (R, C) coordinates using mod arithmetic or explicit conditionals. ge_from_map uses a recursive helper (foldl_map_) to set cells one at a time starting from a zero grid.

All 29 PLUnit acceptance tests pass with no warnings.

Significance

The generate pack completes Mentova's visual pattern vocabulary. Mentova can now:
- Generate any standard visual pattern as a grid
- Compare generated patterns to observed outputs
- Use generated patterns as masks in combination with ns_mask_apply
- Verify hypotheses of the form "output = pattern(type, params)"

Combined with the induction pack, the generation pack enables a full hypothesis cycle: induce a rule from examples, generate the expected output for the test input, and verify against the actual output.
