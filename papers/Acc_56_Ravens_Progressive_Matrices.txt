ABSTRACT

We report Mentova's Raven's Progressive Matrices demonstration, the nonverbal
fluid-intelligence flagship specified in Volume 6, Part 7 of the PrologAI
Demonstration and Proof-of-Concept Plan. Three Raven's-style matrices are solved
by abstract rule type induction: Mentova examines the first complete row of each
matrix, induces an attribute rule type (constant, increment, or cycle_3) for each
cell dimension (shape, size, count, colour), and applies the rule type to the
incomplete last row to predict the missing cell. All four acceptance criteria pass:
AC-PR56-001 (size progression matrix, correct cell predicted), AC-PR56-002 (colour
cycle matrix, correct cell predicted), AC-PR56-003 (count increment matrix, correct
cell predicted), AC-PR56-004 (all three rule types named explicitly glass-box).
The key insight is that rule types are abstracted from specific values, enabling
transfer across rows — the increment rule induced from a row of circles applies
correctly to a row of triangles.


INTRODUCTION

Volume 6, Part 7 of the PrologAI Demonstration and Proof-of-Concept Plan specifies:

    "Raven's Progressive Matrices — the nonverbal fluid-intelligence classic:
     induce the relational rule and show it."

Raven's Progressive Matrices (Raven 1936, 1941) are a test of fluid intelligence
originally designed to measure abstract reasoning ability while bypassing
differences in language, culture, and domain knowledge. Each item is a 3x3 (or 2x2)
grid of cells; the bottom-right cell is missing. The task is to find the pattern
and fill in the missing cell from a set of candidates.

The test has become a standard measure of "g" (general intelligence factor) in
psychometrics and is widely used in AI research as a challenge for non-verbal
abstract reasoning. Correctly solving a Raven's item requires:
1. Perceiving the regularities in the matrix (what changes, what stays the same).
2. Inducing an abstract rule (not tied to specific values).
3. Applying the rule to the incomplete row.
4. Selecting the completion that satisfies all rules simultaneously.

Mentova solves this through its inductive and analogical reasoning rungs, with
the rule named glass-box so the reasoning is fully auditable.


BACKGROUND AND RELATED WORK

RAVEN'S PROGRESSIVE MATRICES IN COGNITIVE SCIENCE

Raven (1936) introduced the Progressive Matrices as a tool for assessing
reasoning ability independently of acquired knowledge. The matrices are designed
to require relational thinking: the patterns involve systematic covariation of
attributes across rows and columns.

Carpenter, Just, and Shell (1990) analyzed how humans solve Raven's Matrices,
finding that successful performance requires:
- Identifying attributes (shape, size, colour, number, orientation).
- Finding the rule that governs each attribute (constant, monotone increase,
  pairwise identical, distribute three values, etc.).
- Composing attribute rules into a complete description of the missing cell.

This analysis maps directly onto PrologAI's attribute decomposition approach.

AI APPROACHES TO RAVEN'S MATRICES

Several AI systems have been applied to Raven's Matrices:
- Visual geometric approaches: extract visual features, match against candidates.
- Analogy-based approaches: use structure mapping over perceived relations.
- Neural network approaches: train on matrices and learn implicit patterns.
- Program synthesis: search for a program in a domain-specific language that
  maps each training pair and apply it to the test row.

PrologAI/Mentova uses an explicit rule type induction approach:
- Represent each cell as a structured term with named attributes.
- Induce the rule type for each attribute from the complete rows.
- Apply the rule types to predict the missing cell.
- Name the rule types explicitly for glass-box output.

The advantage over neural approaches is full transparency: the rule is a
named atom (constant, increment(size_sequence), etc.), not a hidden weight.
The advantage over visual approaches is that the attribute decomposition makes
the relational structure explicit from the start.

FLUID INTELLIGENCE AND SYMBOLIC AI

Raven's Matrices are considered a test of fluid intelligence (Gf) — the capacity
to reason about novel problems without relying on crystallized knowledge. Symbolic
AI has traditionally been stronger on crystallized, knowledge-intensive tasks
(expert systems, knowledge bases) and weaker on novel pattern induction.

Mentova's ability to induce abstract rule types from a single training row and
apply them correctly across rows represents a contribution toward fluid intelligence
in symbolic AI: the rule types (constant, increment, cycle_3) are domain-general
abstractions that transfer across different shapes, sizes, and colours.


METHOD

The demonstration is implemented in demos/ravens_demo.pl. It boots Mentova fully,
then solves three matrices in sequence.

CELL REPRESENTATION

Each cell is represented as: cell(Shape, Size, Count, Color).
- Shape: circle, square, triangle (constant across most matrices shown here).
- Size: small, medium, large (ordered in size_sequence).
- Count: 1-9 (ordered in count_sequence).
- Color: white, gray, black (ordered in color_sequence).

This explicit attribute decomposition makes the matrix pattern visible to Prolog's
unification and matching machinery.

RULE TYPE VOCABULARY

Three rule types are implemented:
- constant: all three values in a row are identical. V3 = V2.
- increment(Seq): each value is the next in sequence Seq. V3 = successor(V2) in Seq.
- cycle_3(Seq): the three values are all distinct members of Seq.

Rule types are abstracted from specific values. This is the critical design decision:
a row of circles has shape=constant, and a row of triangles has shape=constant,
so the same rule type transfers correctly to both rows.

RULE TYPE INDUCTION (rule_type/2)

From a complete row [C1, C2, C3], extract attributes and call rule_type/2:

    rule_type([V, V, V], constant) :- !.
    rule_type([V1, V2, V3], increment(Seq)) :- V1 \= V2, Seq in domains, ... .
    rule_type([V1, V2, V3], cycle_3(Seq)) :- all distinct, Seq contains all three.

The cut in the constant clause ensures mutual exclusivity: once constant is
confirmed, increment and cycle_3 are not tried.

RULE TYPE APPLICATION (apply_rule/3)

From [V1, V2] and a rule type, predict V3:

    apply_rule(constant,         [_V1, V2],  V2).
    apply_rule(increment(Seq),   [_V1, V2],  V3) :- V3 = successor(V2) in Seq.
    apply_rule(cycle_3(Seq),     [V1, V2],   V3) :- V3 in Seq, V3 \= V1, V3 \= V2.

The constant rule ignores V1 and copies V2 to V3 — it does not use the specific
value from the training row. This is what enables cross-row transfer.

SIGNIFICANCE

This accomplishment closes the Raven's Progressive Matrices flagship item from
Volume 6, Part 7 of the Demonstration Plan. The key result is not just that three
matrices were solved correctly, but that:

1. Rule types are abstract and transfer. The rule type induced from a row of
   circles applies correctly to a row of triangles. This is the essence of
   abstract reasoning: the pattern is extracted at a level that transcends the
   specific instances.

2. The reasoning is glass-box. The rule types are named atoms in Prolog, printed
   in the output. A human can read "size: increment(size_sequence)" and verify
   both that the rule is correct and that the prediction follows from it.

3. The attribute decomposition is explicit. By representing cells as structured
   terms with named dimensions, the matrix pattern is immediately accessible to
   Prolog's symbolic reasoning. This contrasts with visual approaches that must
   first extract features from pixel arrays.

4. Multi-attribute composition. The missing cell is predicted by composing four
   independent rule types simultaneously — one per attribute. This mirrors the
   Carpenter, Just, and Shell (1990) analysis of how humans solve Raven's matrices.


HONEST LIMITATIONS

The three matrices demonstrated here involve a single varying attribute per matrix.
Harder Raven's matrices involve multiple simultaneously varying attributes
(e.g., size increases AND colour shifts AND count increases in the same matrix).
The current system handles this correctly (each attribute is treated independently),
but the demonstration matrices were kept simple to focus on the rule type
induction mechanism.

The current rule type vocabulary (constant, increment, cycle_3) covers the most
common pattern types in standard Raven's tests. More complex rule types —
"distribute three values across a row" (where each of three values appears exactly
once in a row, but in any order), "pairwise identical" (columns 1 and 2 are
identical; column 3 is different), and "mirror symmetry" — would require additional
rule type clauses. Adding these is straightforward in Prolog.

The matrices here use symbolic cell representations, not visual pixel grids.
A complete implementation handling visual Raven's Matrices would require a
perceptual front-end to convert the visual grid to cell/4 terms before reasoning.
PrologAI's perception pack (PR 30) provides the architecture for this.
