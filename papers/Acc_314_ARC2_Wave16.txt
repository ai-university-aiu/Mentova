ABSTRACT

Mentova solves ARC-AGI-2 task dbff022c using a new glass-box named rule called
legend_fill. The rule identifies a compact rectangular "legend" table embedded
in the grid (a solid 2xN or Nx2 block of non-background cells) that maps
frame-border colors to fill colors. Each closed frame in the grid whose border
color appears as a key in the legend has its enclosed interior background cells
replaced with the corresponding value color. Open frames (with a gap in their
border) and frames whose colors are not legend keys are left unchanged. The rule
is verified on all three training pairs and the test case of task dbff022c,
raising the ARC-AGI-2 score from 15/120 = 12.50% to 16/120 = 13.33%.

1. PROBLEM STATEMENT

Task dbff022c presents grids containing two types of structures: a compact
legend table and several closed rectangular frames of varying shapes and colors.
The legend is a 2xN or Nx2 solid rectangle embedded somewhere in the grid.
Each closed frame has interior background cells that must be filled with a
specific color determined by consulting the legend. The challenge is to (a)
correctly locate and parse the legend, (b) identify which row or column of the
legend gives the key colors versus the fill colors, and (c) efficiently identify
enclosed interior cells for each keyed frame.

Training pair dimensions:
  Train 0: 12x12 grid, bg=0, 9 changes. Legend at rows 0-1, cols 7-11 (5 pairs).
  Train 1: 10x10 grid, bg=0, 6 changes. Legend at rows 0-3, cols 0-1 (4 pairs).
  Train 2:  4x15 grid, bg=0, 2 changes. Legend at rows 0-1, cols 1-2 (2 pairs).

Test case: 13x13 grid, bg=0, 11 changes. Legend at rows 11-12, cols 0-4 (5 pairs).

2. LEGEND DETECTION

The legend is the unique solid 2xN or Nx2 block in the grid where ALL cells in
the block are non-background. "Solid" means no background gaps in the rectangle.
"Non-identical" means the two rows (or columns) differ from each other - this
distinguishes the legend from a uniform frame border segment.

Detection algorithm:
  For all pairs of adjacent rows (r, r+1): find the LONGEST contiguous column
  range [c1..c2] where both row r and row r+1 are non-background at every
  column. Require c2-c1+1 >= 2 and that the two sub-rows are not identical.

  For all pairs of adjacent columns (c, c+1): find the LONGEST contiguous row
  range [r1..r2] where both column c and column c+1 are non-background at every
  row. Require r2-r1+1 >= 2 and that the two sub-columns are not identical.

  Among all valid 2xN and Nx2 candidates, choose the one with the greatest N
  (most key-value pairs). Ties favor row-format over column-format.

Training 0: legend is the 2x5 block at rows 0-1, cols 7-11.
  The 4-frame checkerboard at rows 0-1, cols 1-5 does NOT form a solid block
  because row 1 has background cells at cols 2 and 4. Only the legend region
  is solid across its full width of 5 columns.

Training 1: the best 2-row block is 2 columns wide (rows 0-1, cols 0-1, length 2).
  The best 2-col block is 4 rows tall (cols 0-1, rows 0-3, length 4).
  Since 4 > 2, the column-format legend is chosen: col 0 = keys, col 1 = values.

Training 2: both the real legend (rows 0-1, cols 1-2) and a 2x2 fragment of the
  2-frame border (rows 0-1, cols 12-13) have length 2. The frame fragment is
  [2,2]/[2,2] (both sub-rows identical). The legend [2,6]/[7,1] has distinct
  rows. The non-identical requirement filters the frame fragment, leaving the
  legend as the only candidate.

3. KEY-VALUE DIRECTION

Given a candidate block [Row1, Row2] (or [Col1, Col2]), the KEY row/col is
whichever contains more colors that appear as frame-border colors in the grid.

Frame-border color detection: a color C is a frame-border color if there exists
at least one background cell in the grid that is completely enclosed by C-cells
(not reachable from the grid boundary via 4-connected background traversal).

  Row1 matches 3,4 (2 matches). Row2 matches 3,7 (2 matches). Tie: Row1 is key.
  Mapping: 3->3, 4->6, 6->7, 9->2, 5->9.

4. INTERIOR DETECTION AND FILLING

Enclosed background cells are found by flood-filling outward from all boundary
background cells (4-connected, background only). Any background cell NOT reached
by this flood fill is "enclosed."

For a key color K with value V: among all enclosed background cells, retain
only those where every adjacent non-background cell has color K. These are the
cells that belong exclusively to the K-frame's interior. Fill them with V.

5. IMPLEMENTATION

Predicate structure:
  arc2_transform/3 with rule legend_fill:
    flatten+msort+arc2_bs_mode_ to find background.
    lf_legend_/4: find best solid block via lf_best_block_/4.
    pairs_keys_values/3 to build K-V pair list.
    foldl/4 over pairs, calling lf_apply_fill_/5 for each key.

  lf_best_block_/4:
    findall(N-row(A,B), lf_row_block_/5, RowCands).
    findall(N-col(A,B), lf_col_block_/5, ColCands).
    max_member to pick longest block.

  lf_row_block_/5 and lf_col_block_/5:
    Iterate over adjacent row (or col) pairs via between/3.
    For each pair: findall over lf_solid_run_/7 for all starting positions.
    Filter runs with length >= 2 and distinct sub-rows (SubA \= SubB).
    Pick maximum-length run via max_member/2.

  lf_solid_run_/7:
    Non-deterministic: enumerate all S in [Lo..Hi] where both lists are non-Bg.
    Extend each run with lf_solid_end_/6 to compute run length.

  lf_orient_kv_/6:
    lf_enclosed_all_/3: flood fill from boundary, subtract from all-Bg set.
    lf_frame_colors_/4: findall non-Bg adjacent colors of enclosed cells.
    include/3 to count Key/Val matches. Higher count = key.

  lf_apply_fill_/5:
    lf_enclosed_for_color_/4: enclosed cells where lf_unique_frame_color_ holds.
    lf_unique_frame_color_/5: findall adjacent non-Bg colors; require sort = [K].
    lf_fill_grid_/4: maplist over rows x cols; memberchk to detect cells to fill.

  lf_flood_/7:
    BFS via recursive list processing (queue of R-C pairs).
    findall 4-connected Bg neighbors not yet in visited set.
    subtract/3 to deduplicate new cells. append/3 to extend visited and queue.

All predicates use the lf_ prefix to avoid name conflicts with other rules.
