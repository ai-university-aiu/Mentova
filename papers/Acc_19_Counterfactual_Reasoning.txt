ABSTRACT

We report Rung 19 of Mentova's 48-rung reasoning ladder: counterfactual reasoning.
Mentova answers "what if this were different" by comparing the actual state of the
world with the nearest possible world in which the antecedent holds. The actual
world has ground_wet=no (it did not rain); the counterfactual "what if it had rained
today" yields ground_wet=yes, via the causal relation causes(rain, wet_ground). The
counterfactual answer differs correctly from the actual answer. Pass criterion met.


INTRODUCTION

Counterfactual reasoning asks about worlds that differ from the actual world in a
specific respect. "If it had rained yesterday, the ground would be wet today." This
statement is about a non-actual world — a world where yesterday's weather was
different. The conclusion (wet ground) is derived by propagating the antecedent
through causal relations in that alternative world.

Rung 19 implements counterfactual reasoning using the Lewis (1973) closest-possible-
world approach: substitute the antecedent, propagate via causal mechanisms, and
compare the result to the actual world.


BACKGROUND

Counterfactual conditionals were analysed by David Lewis (1973) in terms of
possible worlds semantics. The nearest possible world to the actual world in
which the antecedent A holds is used to evaluate the consequent C. If C holds
in that world, the counterfactual "if A then C" is true.

Pearl (2000) connects counterfactuals to the third rung of his causal hierarchy,
above interventions.


METHOD

A new module, src/mentova/counterfactual.pl, implements:

    mentova_counterfactual(+Query, -Result, -Justification)

The module stores five counterfactual knowledge facts
(counterfactual_result/3) covering rain, penguins, cats, sunlight, and exercise.
Actual-world facts are stored in actual/2.

The what_if(Antecedent, Query) form compares the counterfactual answer to the
actual answer and reports whether they differ.

Top-level clause:

    % Query Mentova for counterfactual reasoning about 'CFQuery'.
    mentova_query(counterfactual, CFQuery, answer(Result, Just)) :-
        % State the fact: mentova counterfactual(...).
        mentova_counterfactual(CFQuery, Result, Just).


DEMONSTRATION

Query 1 — what if it had rained today?

    % Query Mentova for counterfactual reasoning about 'what_if(rains_today_yes'.
    ?- mentova_query(counterfactual, what_if(rains_today_yes, ground_wet), R).
    R = answer(yes, just(counterfactual(
                          antecedent(rains_today_yes),
                          query(ground_wet),
                          actual(no),
                          counterfactual(yes),
                          mechanism(causes(rain, wet_ground)),
                          differs(yes))))

    Actual: ground_wet = no.
    Counterfactual: ground_wet = yes.
    Mechanism: causes(rain, wet_ground).
    Differs: yes. Correct.

Query 2 — what if the penguin were not flightless?

    answer(yes, just(counterfactual(antecedent(penguin_not_flightless),
                                     query(penguin_flies), actual(no),
                                     counterfactual(yes), ...)))

    If the penguin were not flightless, it would fly. Differs from actual (no).


RESULTS

Both queries return counterfactual answers that differ correctly from the actual
world. The mechanism is named in the justification.

Pass criterion met: counterfactual differs correctly from fact.


SIGNIFICANCE

Counterfactual reasoning is essential for causal attribution (if X had not happened,
would Y have occurred?), planning (if I took action A, what would happen?), and
moral responsibility (was X a necessary cause of Y?).

Rung 19 connects to Rung 6 (causal reasoning) above and prepares for Rung 38
(practical/deliberative reasoning), where counterfactuals are used to evaluate
candidate actions.


HONEST LIMITATIONS

The counterfactual results are stored explicitly for each antecedent-query pair.
A complete implementation would derive counterfactual conclusions algorithmically
from the causal graph using the structural equation framework.

The closest-world selection is implicit: only one counterfactual world is
considered per antecedent, not a ranked similarity space.


REPRODUCIBILITY

Requirements: SWI-Prolog 9.0.4 or later.

  git clone https://github.com/ai-university-aiu/Mentova.git
  cd Mentova
  % Load the Mentova bootstrap into SWI-Prolog, execute the query, and halt.
  swipl -l src/mentova/mentova.pl \
        % Supply a goal or termination instruction to the SWI-Prolog command.
        -g "mentova_boot, \
            % Query Mentova for counterfactual reasoning about 'what_if(rains_today_yes'.
            mentova_query(counterfactual, what_if(rains_today_yes,ground_wet), R1), \
            write(R1), nl" \
        % Supply a goal or termination instruction to the SWI-Prolog command.
        -t halt
