THE SYM_RESTORE RULE

Definition: for each non-background colour group, find the vertical axis
(represented as Ax2 = 2 * axis_column, enabling half-integer axes) that
minimises the number of orphan cells.  An orphan is a cell rc(R,C) whose
column-mirror rc(R, Ax2-C) is absent from the group.  Remove all orphans.

Algorithm:
1. Find background (modal value of all cells).
2. Group non-background cells by colour.
3. For each colour group:
   a. Let MinC..MaxC be the column range.
   b. For each Ax2 in [2*MinC .. 2*MaxC]:
      - Count cells rc(R,C) where rc(R,Ax2-C) is absent.
   c. Use the Ax2 with the smallest orphan count.
   d. Collect those orphan cells.
4. Replace each orphan cell with background.

Key insight: the axis is integer when the symmetric core has an odd width
(a central column), and half-integer when the width is even (no central column).
Both cases are handled uniformly by scanning doubled-axis integers.

Key observation: the rule always uses vertical (column) symmetry, never
horizontal (row) symmetry.  Among vertical axes, the minimum-orphan axis
uniquely identifies the intended axis in all training pairs and test case.
