ABSTRACT

Mentova solves ARC-AGI-2 task 3dc255db using a new glass-box named rule called
tip_escape. Each grid contains one or more "shape" components (8-connected
non-background color regions) that enclose "marker" components (a different
non-background color, strictly smaller, with bounding box contained inside the
shape's bounding box). The shape has a "tip": the unique single cell at an
extreme row or column direction. In the output all marker cells are removed and
min(N_markers, dist_to_grid_edge) new marker-color cells are placed just past
the tip, filling outward toward the nearest grid boundary. The escape direction
is determined by apex detection: the tip qualifies as an apex if perpendicular
shape-cell arms diverge (one above and one below for a left/right tip; one left
and one right for an up/down tip). If no apex is found, a projection method
chooses the tip with the greatest projection onto the vector from the marker
centroid to the shape centroid. The rule is verified on all three training pairs
and the test case, raising the ARC-AGI-2 score from 17/120 = 14.17% to
18/120 = 15.00%.

1. PROBLEM STATEMENT

Task 3dc255db presents 13x13 grids containing shapes made of 8-connected
non-background cells. Each shape has a cluster of "marker" cells of a different
color tucked inside its concavity or bounding box. In the output the markers
disappear from their original positions and reappear just past the shape's pointed
tip, extending outward from the tip toward the grid boundary. The count of placed
markers is capped at the number of available grid cells in that direction.

2. RULE DERIVATION

Step 1 - 8-connected component detection.
All non-background regions are identified using BFS with 8-adjacency (diagonal
neighbors included). This is essential for chain-like and hook-shaped components
where cells connect only diagonally.

Step 2 - Shape and marker identification.
For every ordered pair of components (S, M): if S and M have different colors,
|M.cells| < |S.cells|, and M's bounding box is fully contained inside S's
bounding box, then M is a marker for shape S.

Step 3 - Escape direction: apex detection (primary method).
For each direction (left, right, up, down): find the extreme row or column
(MinRow, MaxRow, MinCol, MaxCol). If exactly one shape cell is at that extreme,
it is a candidate tip. For each candidate tip, collect all 8-adjacent shape-cell
neighbors (cells within 1 row and 1 column distance, excluding the tip itself).
The tip is an apex if:
- For left/right escape: the neighborhood contains at least one cell with row
  strictly less than TipRow AND one cell with row strictly greater than TipRow.
- For up/down escape: the neighborhood contains at least one cell with column
  strictly less than TipCol AND one cell with column strictly greater than TipCol.
The first apex found is chosen as the escape direction.

Apex examples from the task:
- Pair 1, 6-shape top tip (3,7): neighbors (4,6) and (4,8) have columns 6 and 8.
  6 < 7 and 8 > 7. Apex confirmed. Escape UP.
- Pair 2, 6-frame top tip (3,5): neighbors (4,4), (4,5), (4,6). Columns 4 < 5
  and 6 > 5. Apex confirmed. Escape UP.
- Test, 2-shape right tip (3,10): neighbors (2,9) and (4,9). Rows 2 < 3 and
  4 > 3. Apex confirmed. Escape RIGHT.
- Test, 8-shape right tip (9,11): neighbors (8,10) and (10,10). Rows 8 < 9 and
  10 > 9. Apex confirmed. Escape RIGHT.

Step 4 - Escape direction: projection method (fallback when no apex).
Compute the shape centroid (SR, SC) and the marker centroid (MR, MC). The
escape vector is (SR - MR, SC - MC), pointing from the markers toward the
opposite side of the shape. For each candidate tip at (TipR, TipC) compute:
  Proj = (TipR - SR) * (SR - MR) + (TipC - SC) * (SC - MC)
The tip with the greatest projection value is chosen.

Projection examples:
- Pair 0, 3-shape (two tips: left at (10,3) and right at (7,9)):
  Shape centroid = (8.93, 5.79). Marker centroid = (8.5, 6.5).
  Escape vector = (0.43, -0.71).
  Left tip proj = 1.07*0.43 + (-2.79)*(-0.71) = 0.46 + 1.98 = 2.44.
  Right tip proj = (-1.93)*0.43 + (3.21)*(-0.71) = -0.83 - 2.28 = -3.11.
  Maximum: left tip. Escape LEFT.
- Pair 1, 3-shape (three tips: up, down, left):
  Shape centroid = (5.5, 2.5). Marker centroid = (6.0, 1.67).
  Escape vector = (-0.5, 0.833).
  Up tip (3,2) proj = 0.8335. Down tip (8,4) proj = 0.0. Left tip (6,0) = -2.33.
  Maximum: up tip. Escape UP.

Step 5 - Escape zone construction and marker placement.
From the tip, list all grid cells in the escape direction: from TipR+1 downward
for DOWN, from TipR-1 upward for UP, from TipC+1 rightward for RIGHT, from
TipC-1 leftward for LEFT, stopping at the grid boundary. This is the escape zone.
Remove all marker cells (set to 0). Place min(N_markers, zone_length) marker-color
cells starting at the position nearest the tip.

3. IMPLEMENTATION

All predicates use the te_ prefix. The BFS (te_bfs8_) uses an explicit frontier
queue: te_bfs8_(Grid, Color, NR, NC, Queue, Visited, Component). Each step
dequeues the head, finds all 8-adjacent in-bounds same-color unvisited cells,
appends them to the queue and visited set, then recurses.

Bounding box containment (te_is_marker_) uses pairs_keys_values/3 to split
R-C lists into row and column lists, then min_list/max_list for extremes.

Apex detection (te_is_apex_/4) has four clauses: right, left, up, down. Each
calls te_apex_nbrs_/4 to collect 8-adjacent shape cells of the tip, then checks
for divergence using member/2 with arithmetic comparisons.

Projection (te_projection_tip_/7) computes floating-point centroids
(AvgR is SumR / float(N)) to avoid integer truncation, then uses findall/3 to
collect all (Proj-Dir-TR-TC) terms and max_member/2 to select the maximum.

Escape zone (te_escape_zone_/6) uses numlist/3 and reverse/2 to produce the
ordered cell list, and maplist/3 with a lambda to pair each row or column index
with the fixed tip coordinate.

6. TECHNICAL NOTES

8-connectivity is required for this task. Using 4-connectivity would split the
bent-chain shape (8-shape in the test case) into separate components and would
misidentify its marker.

The bounding-box containment criterion correctly handles all cases: fully-enclosed
markers (inside a frame), concavity-enclosed markers (inside a hook or arrow), and
externally adjacent marker clusters, while correctly excluding stray same-color
cells that lie outside all shape bounding boxes.
