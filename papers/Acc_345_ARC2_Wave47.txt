ABSTRACT

Wave 47 adds the shape_count rule (WP-305, Layer 280) to PrologAI's ARC-AGI-2
benchmark solver, solving task 58490d8a and raising the score from 44/120 =
36.67% to 45/120 = 37.50%.

The rule recognizes a two-region layout: a rectangular template of 0-cells
with color markers at odd rows, and a game board containing multiple identical
instances of small geometric shapes per color. The output reproduces the
template with each color marker repeated as many times as there are instances
of that color-shape in the game board.


TASK DESCRIPTION

Task ID: 58490d8a

Input structure: a grid with a non-zero background. One rectangular sub-region
(the template) is filled entirely with 0-cells except for a single non-zero
value at column TC0+1 of every odd template row; that value names a color to
count. The rest of the grid (the game board) contains identical small geometric
shape instances for each named color.

Output: a grid matching the template dimensions. Each odd row is filled with
Count copies of the named color at columns 1, 3, 5, ..., 2*Count-1. All other
cells are 0. The output dimensions equal the template dimensions
(TR1-TR0+1 rows, TC1-TC0+1 cols).


THE SHAPE_COUNT RULE

Step 1 - Locate the template: find the bounding box (TR0, TR1, TC0, TC1) of
all 0-cells in the input grid.

Step 2 - Extract color assignments: for each odd relative template row TR
(1, 3, 5, ...), scan the template columns (TC0 to TC1) for the first non-zero
cell; that cell value is the color to count for that row.

Step 3 - Count instances: for each (TR, Color) pair, collect all Color cells
outside the template region. Sort them top-left first. Extract instances one
by one using iterative bounding-box expansion:

  EXPAND(Seed, AllCells):
    Cur = {Seed}
    LOOP:
      (R0,R1,C0,C1) = tight_bbox(Cur)
      Expanded = {p in AllCells : R0-1 <= p.r <= R1+1, C0-1 <= p.c <= C1+1}
      IF Expanded == Cur: RETURN Cur   (instance complete)
      Cur = Expanded

  Remove the extracted instance from the pool. Increment count. Repeat
  until the pool is empty. The final count is the number of instances.

Step 4 - Assemble output: start with a zero-filled NTR x NTC grid. For each
(TR, Color) pair with Count instances, write Color at output positions
(TR, 1), (TR, 3), ..., (TR, 2*Count-1).


WHY BBOX EXPANSION INSTEAD OF FLOOD-FILL

Some shape instances contain cells that are only diagonally adjacent, not
4-connected. For example, the 5-cell diamond-cross pattern (C _ C / _ C _ /
C _ C) has no pair of adjacent cells in the 4-connectivity sense. Standard
connected-component flood-fill would split one instance into multiple
components and produce an incorrect count.

The bbox-expansion algorithm is connectivity-agnostic: it groups all cells
that fit within a 1-cell-padded bounding box, regardless of whether they
touch orthogonally or diagonally. The iteration terminates as soon as no new
cells are pulled in, producing exactly one instance per shape occurrence.

SIGNIFICANCE

shape_count demonstrates PrologAI's ability to count occurrences of abstract
visual patterns using a purely geometric technique that is robust to internal
disconnectedness. The iterative bounding-box expansion algorithm is a
general-purpose instance-extraction procedure that works for any shape whose
cells are spatially separated by at most 1 unit but belong to the same
geometric motif.

The two-region layout (template + game board) represents a new pattern class
in the ARC-AGI-2 corpus where the template acts as a structured query: each
marked row asks "how many of this color appear in the game board?" The output
is the answer encoded as repeated color tokens at standardized column offsets.
