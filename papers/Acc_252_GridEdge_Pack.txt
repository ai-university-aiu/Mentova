Abstract

PrologAI Work Package 232 adds the gridedge pack (ge_*, Layer 211): fourteen
predicates for detecting edges, boundaries, and structural features of symbolic
grids. The pack covers 4- and 8-connected edge cell detection, single-cell edge
testing, directional boundary detection, inner and outer border extraction, edge
grid construction, edge counting, neighbor difference analysis, corner and
endpoint detection, smooth cell detection, per-cell transition count maps, and
color-specific edge counting. All 44 acceptance tests pass.


1. Motivation

Structural analysis of symbolic grids requires more than color statistics:
it requires understanding where color regions meet, where corners occur,
where lines begin and end, and how many cells on each side of a boundary are
affected. The gridedge pack provides the complete toolkit for all of these
boundary-awareness operations, serving as the structural companion to gridcolor
(color statistics) and gridflood (region connectivity).


2. Architecture

Layer 211 sits immediately above gridstitch (Layer 210). Uses library(lists)
only (nth0/3, member/2). No cross-pack dependencies.

Private helpers:
- ge_dims_/3: grid height and width
- ge_cell_/4: cell value at row R, column C (0-indexed); fails out of bounds
- ge_build_/4: build H x W grid using lambda Goal(R, C, V)
- ge_nbr4_/4: generate one 4-connected neighbor on backtracking (4 choices)
- ge_nbr8_/4: generate one 8-connected neighbor on backtracking (8 choices)
- ge_boundary_to_nonbg_/3: BgColor cells adjacent to any non-BgColor cell

Critical design invariant: predicates that ask "does this cell have at least
one neighbor satisfying property P?" must wrap the neighbor check in once/1
inside findall. Without once, a cell adjacent to k qualifying neighbors would
appear k times in the output list.


3. Predicates

Edge detection:

ge_edge_cells/3 - FgColor cells with at least one 4-connected differently-colored
  in-bounds neighbor; each cell appears exactly once.
ge_edge_cells8/3 - same with 8-connected (Moore) neighborhood.
ge_is_edge/3 - test single cell (R,C); returns yes or no.

Boundary detection:

ge_boundary/4 - Color1 cells adjacent to at least one Color2 cell (4-connected).
ge_inner_border/3 - BgColor cells adjacent to at least one non-BgColor cell.
ge_outer_border/3 - non-FgColor cells adjacent to at least one FgColor cell.

Grid construction:

ge_edge_grid/4 - grid with FgColor edge cells replaced by EdgeColor; other cells
  retain original color.

Counting:

ge_edge_count/3 - count of 4-connected edge cells for FgColor.
ge_edge_color_count/4 - count of FgColor cells adjacent to at least one BgColor cell.

Structural features:

ge_neighbors_diff/4 - list of 4-connected neighbor positions with different color.
ge_corners/3 - FgColor cells with 2 or more differently-colored 4-neighbors.
ge_endpoints/3 - FgColor cells with exactly 1 same-color 4-neighbor.
ge_smooth_cells/3 - FgColor cells with no differently-colored in-bounds 4-neighbors.
ge_transition_map/2 - grid where each cell value = count of differently-colored
  4-connected neighbors (integer in [0..4]).


4. Example Session

:- use_module(pack(gridedge/prolog/gridedge)).

% Edge cells of r in a 3x3 grid:
G = [[r,r,x],[r,r,x],[x,x,x]].
ge_edge_cells(G, r, Cells).
% Cells = [0-1, 1-0, 1-1]
% Note: (0,0) is NOT an edge cell; its only neighbors are (0,1)=r and (1,0)=r.

% Test a single cell:
ge_is_edge(G, 0-1, B).
% B = yes
ge_is_edge(G, 0-0, B2).
% B2 = no

% Directional boundary: r cells touching x:
ge_boundary(G, r, x, Cells).
% Cells = [0-1, 1-0, 1-1]

% Outer border (x cells touching r):
ge_outer_border(G, r, OB).
% OB = [0-2, 1-2, 2-0, 2-1]

% Transition map:
ge_transition_map(G, TM).
% TM = [[0,1,1],[1,2,1],[1,1,0]]

% Corner cells of r:
ge_corners(G, r, Corners).
% Corners = [1-1]   (only cell with 2+ differently-colored neighbors)

% Smooth cells of r:
ge_smooth_cells(G, r, Smooth).
% Smooth = [0-0]   (only cell with all-r in-bounds neighbors)

% Endpoints of r in a cross pattern:
ge_endpoints([[x,r,x],[r,r,r],[x,r,x]], r, Pts).
% Pts = [0-1, 1-0, 1-2, 2-1]   (the 4 arm tips)

- ge_neighbors_diff: corner with all-r neighbors (empty), edge cell (1 diff), cross center (empty)
- ge_corners: rb [1-1], uniform empty
