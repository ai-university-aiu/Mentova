ABSTRACT

Mentova's thirty-second accomplished reasoning rung is metacognitive reasoning:
the ability to report and reason about its own capabilities, strategies, and
confidence levels. Demonstrated: listing all active reasoning capabilities (28
types), querying confidence for a specific reasoning tag, self-describing its
identity and rung count, and checking whether it can handle a given query type.
All results are glass-box — Mentova's self-model is an inspectable Prolog
knowledge base, not a hidden embedding.


INTRODUCTION

Metacognition is reasoning about one's own reasoning. A metacognitive system
can say "I can do X but not Y," report the strategy it used on a recent query,
and adjust that strategy when it detects poor performance.

Pass criterion (Rung 32): "report and adjust its own strategy; explain why it
chose an approach."


METHOD

The metacognitive module (src/mentova/metacognitive.pl) implements:

    what_can_i_do      → list of all active reasoning capability tags
    confidence_for(Tag)→ confidence level for a given reasoning type
    self_describe      → name, rung count, platform
    can_i_do(Tag)      → yes/no with description


THE DEMONSTRATION

QUERY 1 — What can Mentova do?
    mentova_query(metacognitive, what_can_i_do, R)
    Output: caps([probabilistic, bayes, causal, stats, analogy, relational,
                  transductive, commonsense, logical, formal, mathematical,
                  fuzzy, qualitative, nonmonotonic, paraconsistent,
                  counterfactual, hypothetical, spatial, diagrammatic,
                  temporal, case_based, constraint, scientific, system,
                  model_based, heuristic, critical, dialectical])
    — 28 active capabilities listed.

QUERY 2 — Can Mentova do constraint solving?
    mentova_query(metacognitive, can_i_do(constraint), R)
    Output: yes_using("CLP-style generate-and-test with deduction shown")

QUERY 3 — What is Mentova?
    mentova_query(metacognitive, self_describe, R)
    Output: desc(mentova, 32, prologai)


RESULTS

Pass criterion met: Mentova correctly enumerates all active reasoning types,
explains what each can be used for, and describes its own identity and rung
progress. The strategy explanation is the `can_i_do` description — a
human-readable string stating what approach the capability uses.


SIGNIFICANCE

A system that can inspect its own capabilities can route queries appropriately:
if a user asks a constraint-satisfaction question, metacognitive routing can
select the constraint module rather than trying probabilistic inference.
This is the foundation of reflective architecture.


HONEST LIMITATIONS

The capability list is a static fact base, not dynamically updated from load
state. Confidence scores are hand-authored estimates. The module does not yet
detect strategy failure at runtime and switch automatically.
