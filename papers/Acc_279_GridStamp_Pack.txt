Abstract

PrologAI Work Package 259 adds the gridstamp pack (gst_*, Layer 238): fourteen
predicates for placing patterns on grids, finding pattern matches, resizing
grids, and creating blank canvases. The pack covers stamp operations with
transparent backgrounds, opaque cell replacement, multi-position scatter,
exhaustive pattern match finding, padding and cropping, horizontal and vertical
replication, border addition, centering, subgrid extraction, and canvas
creation. All 44 acceptance tests pass.


1. Motivation

Grid reasoning tasks frequently require compositing patterns: placing a
template at one or more positions, finding all positions where a motif occurs,
or building a canvas from scratch and populating it with elements. The
gridstamp pack provides a clean predicate for each of these operations rather
than requiring callers to re-implement cell-by-cell traversal. The transparency
model of gst_stamp (background cells in a stamp do not overwrite the underlying
grid) is the key design choice: it enables layered compositing without an
explicit masking step.


2. Architecture

Layer 238 sits immediately above gridgrav (Layer 237) in the Data Layer.
Uses library(lists): member/2. All other operations (nth0/3, length/2,
append/3, between/3, findall/3) are SWI-Prolog built-ins.

Key design decisions:

Transparency vs opacity: gst_stamp collects only non-bg cells from Stamp as
modifications, so bg cells in Stamp show through to the underlying Grid.
gst_scatter and gst_replace are fully opaque: every target cell is overwritten
regardless of its current color.

gst_matches_at_/5: uses \+ with a negation-as-failure body that succeeds if
any Pattern cell differs from the corresponding Grid cell. This short-circuits
on the first mismatch, making rejection fast.

gst_center: uses direct cell bounds arithmetic (SR = R - R0, SC = C - C0)
rather than a nested gst_stamp call. This handles the case where Grid contains
bg content that should be preserved, which a stamp-based approach would
incorrectly overwrite.

gst_replicate_v: findall(Row, (between(1,N,_), member(Row,Grid)), Result).
Idiomatic repeat via backtracking over member, avoiding explicit list concat.

gst_border: T=0 returns Grid unchanged because findall over between(1,0,_)
generates zero rows and zero padding cells.

gst_unpad: computes the tight bounding box by collecting all non-bg row and
column indices via findall, then finding the min and max of each set. An
all-bg grid yields [].


3. Predicates

Canvas:

gst_canvas/4        - create an H x W grid filled entirely with Bg

Stamping:

gst_stamp/6         - paste Stamp at (R,C) with transparent background
gst_stamp_all/5     - stamp at each position in a list (left to right)
gst_scatter/5       - place Color at each position in a list (opaque)

Finding:

gst_find_matches/4  - all top-left positions where Pattern matches Grid exactly
gst_stamp_count/4   - count of pattern occurrences (wraps gst_find_matches)

Resizing:

gst_pad/5           - pad to at least H x W, anchored top-left
gst_unpad/3         - crop to tight bounding box of non-Bg content
gst_replicate_h/4   - repeat N times side by side horizontally
gst_replicate_v/4   - stack N times vertically

Decoration:

gst_border/5        - surround with colored border of thickness T
gst_center/5        - center on H x W canvas using integer division

Extraction and replacement:

gst_extract/6       - extract H x W subgrid at (R,C)
gst_replace/5       - overwrite rectangle with Patch (opaque, clips at bounds)


4. Example Session

:- use_module(pack(gridstamp/prolog/gridstamp)).

% Build a blank canvas and stamp a pattern.
gst_canvas(4, 4, b, C),
gst_stamp(C, [[r,g],[y,p]], 1, 1, b, Stamped).
% Stamped = [[b,b,b,b],[b,r,g,b],[b,y,p,b],[b,b,b,b]].

% Find all positions of a motif.
gst_find_matches([[r,b,r],[b,r,b]], [[r]], b, M).
% M = [r(0,0),r(0,2),r(1,1)].

% Transparent stamp: bg in stamp does not overwrite.
gst_stamp([[r,g],[y,p]], [[b,x],[b,b]], 0, 0, b, R).
% R = [[r,x],[y,p]].

% Unpad trims all-bg border.
gst_unpad([[b,b,b],[b,r,b],[b,b,b]], b, U).
% U = [[r]].

% Add border of thickness 1.
gst_border([[r,g],[y,p]], x, 1, b, B).
% B = [[x,x,x,x],[x,r,g,x],[x,y,p,x],[x,x,x,x]].

% Center on a 3x3 canvas.
gst_center([[r]], 3, 3, b, Cen).
% Cen = [[b,b,b],[b,r,b],[b,b,b]].

7. Conclusion

The gridstamp pack provides fourteen gst_* predicates for compositing,
resizing, and querying patterns on symbolic grids. The transparency model
of gst_stamp enables layered compositing; gst_find_matches provides exhaustive
motif search; and the canvas/border/center/replicate family covers the full
range of grid construction and decoration operations needed for systematic
grid-task reasoning.
