Title: ARC-AGI-2 Wave 34: Rail-Fill - 0-Bordered Boxes with 9-Rail Arms Fill 7-Cells from the Arm Side and Shift Toward the Outer Rail

Author: D. R. Dison, Founder of AIU (Artificial Intelligence University). Creator and Owner of PrologAI and Mentova. Open Researcher and Contributor ID (ORCID): 0009-0001-9246-5758. (https://www.linkedin.com/in/d-r-dison/)

ABSTRACT

PrologAI solves ARC-AGI-2 task 271d71e2 by discovering the rail_fill rule: each input sub-object is a 0-bordered rectangular box with a 5- or 7-valued interior and two parallel 9-rails (an arm) on one side. The arm gap - the distance between the inner rail (closer to the box) and the outer rail - determines how many new 7-cells the box gains. The box shifts toward the outer rail by the same step count. The inner rail merges into the new box position. The fill follows a direction-specific column-or-row-major traversal that produces a diagonal staircase of 7-cells starting from the arm-side near corner. The rule is implemented as a pure Prolog glass-box named rule and passes all 3 training pairs plus the test case. Score advances from 33/120 = 27.50% to 34/120 = 28.33%.

1. INTRODUCTION

ARC-AGI-2 task 271d71e2 presents grids containing sub-objects, each consisting of a rectangular 0-bordered box with colored interior cells (values 5 and 7) and two parallel 9-valued rails adjacent to one side of the box. The rails act as an arm, and the transformation of each box is determined by the gap between the two rails, the current count of 7-cells in the interior, and the arm direction.

PrologAI approaches this as a glass-box pattern discovery problem: derive the exact named rule from training pairs and apply it to the test case.

2. THE RAIL-FILL RULE

2.1 Structure of Each Sub-Object

Each sub-object consists of:

- A 0-bordered rectangular box at rows R1..R2, cols C1..C2. All border cells are 0. Interior cells (rows R1+1..R2-1, cols C1+1..C2-1) are 5 or 7.

- Two parallel 9-rails on one side of the box. If the arm is on the left, both rails are vertical columns of 9-values spanning R1..R2. The inner rail is adjacent to the box (at column C1-1 or the nearest 9-column to the left). The outer rail is further in the same direction.

2.2 Transformation Parameters

For each box with arm direction Dir, inner rail at position Inner, outer rail at Outer:

  arm_gap = |Inner - Outer|
  N7in = count of 7-cells in interior
  free_cells = (R2-R1-1)*(C2-C1-1) - N7in
  N7gained = min(arm_gap, free_cells)
  N7new = N7in + N7gained

2.3 Fill Order

The new 7-cells are placed by a direction-specific traversal of the interior:

  arm_left:   column by column left to right; within each column, bottom to top.
  arm_right:  column by column right to left; within each column, top to bottom.
  arm_top:    row by row top to bottom; within each row, left to right.
  arm_bottom: row by row bottom to top; within each row, right to left.

The key invariant: the existing N7in 7-cells in the input already occupy exactly the first N7in positions in this traversal order. Therefore the output interior is determined solely by the new total N7new - the first N7new positions in the order become 7, the rest become 5.

2.4 Box and Rail Movement

After computing N7new, the box shifts by N7gained steps toward the outer rail:

  arm_left:   NewC1 = C1 - N7gained, NewC2 = C2 - N7gained (rows unchanged)
  arm_right:  NewC1 = C1 + N7gained, NewC2 = C2 + N7gained (rows unchanged)
  arm_top:    NewR1 = R1 - N7gained, NewR2 = R2 - N7gained (cols unchanged)
  arm_bottom: NewR1 = R1 + N7gained, NewR2 = R2 + N7gained (cols unchanged)

The outer rail remains at its original position. The new arm gap is arm_gap - N7gained. If the new gap is 0, only the outer rail appears in the output (the inner rail has merged into the box). If the new gap is positive, a new inner rail appears at Outer + new_gap * sign(Inner - Outer).

3. IMPLEMENTATION

The rule is implemented as arc2_transform(rail_fill, Grid, Out) in the PrologAI/Mentova benchmark file arc_benchmark_2.pl.

Key predicates:

- arc2_rf_zero_border_: verifies all four edges of a rectangle are 0-valued.
- arc2_rf_valid_interior_: verifies all interior cells are 5 or 7.
- arc2_rf_arm_: scans each cardinal direction to find the inner and outer 9-rail.
- arc2_rf_fill_order_: generates the (Row-Col) fill sequence for the arm direction.
- arc2_rf_make_interior_: builds the new interior grid from the fill sequence and N7new.
- arc2_rf_box_transform_: orchestrates all sub-computations into a plan term.
- arc2_rf_cell_in_plan_: resolves the output value at any (R,C) from a plan.

Box detection uses a ZeroCells index (all 0-valued positions) to avoid scanning all possible (R1,C1,R2,C2) combinations. Only pairs drawn from ZeroCells are tested as potential box corners, keeping detection tractable for 26-by-26 grids.

4. RESULTS

Task 271d71e2 - 3 training pairs - all pass.

Pair 1 (8 x 8):   Left box gains 1 seven, moves up 1. Right box gains 1 seven, moves up 1.
Pair 2 (17 x 17): Three boxes. Box A (arm left, gap 5): 3 new sevens. Box B (arm right, gap 7): 7 new sevens. Box C (arm left, gap 4): 4 new sevens. All move correctly.
Pair 3 (12 x 12): Top box (arm right, gap 5): 5 new sevens, moves right 5. Bottom box (all 7s, arm left, gap 2): no free cells, unchanged.

Test case (26 x 26): transform produced without error.

Score: 33/120 = 27.50% to 34/120 = 28.33%.

5. CONCLUSION

Wave 34 demonstrates that PrologAI can recover a multi-component spatial rule - arm gap, fill order, staircase invariant, box translation, and rail update - purely from 3 training examples using glass-box named-rule induction. The fill-order staircase invariant (existing 7s always occupy the first N7in fill positions) reduces the transformation to a single integer N7new and a fixed traversal, making the rule both compact and verifiable.
