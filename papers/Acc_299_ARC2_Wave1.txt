SUMMARY

Wave 1 of the ARC-AGI-2 climb. Downloads the ARC-AGI-2 public evaluation
task files, builds the Prolog converter, loads 120 evaluation tasks, runs
the first benchmark (0/120 = 0.00% baseline), and adds the plus-shape
recolor rule to solve the first task. Final Wave 1 score: 1/120 = 0.83%.


WHAT WAS BUILT

Task data pipeline:

The ARC-AGI-2 public repository (github.com/arcprize/ARC-AGI-2) was cloned
into /home/ccaitwo/Mentova/data/arc_agi_2/arc_agi2_repo/. This provides
1000 training tasks and 120 public evaluation tasks as JSON files.

The converter tools/arc_agi2_to_prolog.py was written and run. It reads
each evaluation JSON from data/arc_agi_2/arc_agi2_repo/data/evaluation/
and produces a Prolog fact file data/arc_agi_2/arc_tasks_2.pl containing
120 arc2_task/4 facts.

Format: arc2_task(TaskId, TrainingPairs, TestInput, TestOutput).
TrainingPairs = list of pair(InputGrid, OutputGrid).
Grids = list of rows; each row = list of color integers 0-9.

Baseline run:

The six initial geometric transforms (reverse_rows, vertical_flip,
transpose, rotate_90_cw, rotate_90_ccw, rotate_180) and the color
bijection fallback solved 0 of 120 tasks. This is the expected starting
point. None of the evaluation tasks are simple geometric transforms.

Analysis methodology:

A systematic Python analysis classified all 120 tasks by their change
signature (what color changes to what, in how many cells, across all
training pairs). Key findings:

- 120 tasks total.
- 81 tasks preserve input/output dimensions.
- 39 tasks change grid dimensions.
- 4 tasks have only a single color change type (A->B exclusively).
- 3 tasks have only a color-swap pattern (A<->B).
- None are solvable by gravity, flood-fill of enclosed regions, or simple
  color bijection.

Rule added: recolor_plus(A, B)

Description: Find all 5-cell cross/plus shapes of source color A in the
input grid. A plus shape is defined by a center cell (R, C) and its four
cardinal neighbors (R-1, C), (R+1, C), (R, C-1), (R, C+1), all having
color A. Recolor every cell in each plus shape to target color B.

Implementation: arc2_apply_plus_recolor/4 in arc_benchmark_2.pl.

Learning mechanism: arc2_learn_single_recolor_/3 extracts A and B from
training pairs where all changed cells follow the single rule A->B.
arc2_induce_rule/2 accepts recolor_plus(A, B) if the rule reproduces all
training outputs from their training inputs.

Task solved: 1818057f

Rule: recolor_plus(4, 8)
Description: All 5-cell plus shapes of color 4 are recolored to color 8.
Verification across 3 training pairs:
- Train 0: 1 plus shape (5 cells), all correctly recolored.
- Train 1: 9 plus shapes (45 cells), all correctly recolored.
- Train 2: 2 plus shapes (10 cells), all correctly recolored.
Test input: 6 plus centers found; predicted output matches exactly.

Bug fixed:

arc_benchmark_2.pl imported msort/2 from library(lists), but msort/2 is a
system predicate not exported by that library. Removed from explicit import
list. No functional impact; eliminates the harmless warning on benchmark runs.

SIGNIFICANCE

The plus-shape recolor rule is the simplest meaningful inductive rule
found in the evaluation set: every changed cell in every training example
belongs to a 5-cell plus-shaped cluster of the same color.

The systematic change-type analysis identified six additional candidate
