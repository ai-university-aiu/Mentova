ABSTRACT

We report Rung 40 of Mentova's 48-rung reasoning ladder: strategic reasoning.
Mentova uses minimax search in a game tree and computes Nash equilibria and
dominant strategies in matrix games. In a three-branch game tree, minimax selects
move=a (value=3). In the Prisoner's Dilemma, the Nash equilibrium is
[defect-defect] and the dominant strategy for the row player is defect.
Pass criterion met: optimal move found with minimax value.


INTRODUCTION

Strategic reasoning concerns decision-making in the presence of other agents with
potentially opposing interests. Game theory (von Neumann and Morgenstern, 1944)
formalises optimal strategies. Rung 40 implements minimax and Nash equilibrium.


BACKGROUND

Minimax (Shannon, 1950) is the core algorithm for two-player zero-sum games.
Nash equilibrium (Nash, 1950) identifies stable strategy profiles in general games.


METHOD

A new module, src/mentova/strategic.pl, implements:

    mentova_strategic(+Query, -Result, -Justification)

Games: three-branch minimax tree, Prisoner's Dilemma, Matching Pennies.
Queries: minimax(Position), nash(Game), dominant(Game, Player), payoff(Game, Row, Col).


DEMONSTRATION

    % Query Mentova for strategic reasoning about 'minimax(root)'.
    ?- mentova_query(strategic, minimax(root), R).
    R = answer(move(a, value(3)),
               just(strategic(minimax_search(root), best_move(a), value(3))))

    % Query Mentova for strategic reasoning about 'nash(prisoners_dilemma)'.
    ?- mentova_query(strategic, nash(prisoners_dilemma), R).
    R = answer(equilibria(prisoners_dilemma, [defect-defect]), ...)

    % Query Mentova for strategic reasoning about 'dominant(prisoners_dilemma'.
    ?- mentova_query(strategic, dominant(prisoners_dilemma, row), R).
    R = answer(dominant(prisoners_dilemma, row, defect), ...)


RESULTS

Minimax selects a=3. Nash equilibrium: defect-defect. Dominant strategy: defect.
All results align with classical game theory. Pass criterion met.


SIGNIFICANCE

Strategic reasoning is the foundation of competitive AI (chess, Go, poker),
auction design, mechanism design, and multi-agent coordination.


HONEST LIMITATIONS

The game tree is static and small. Real game AI uses alpha-beta pruning, Monte
Carlo tree search, or neural evaluation functions for large state spaces.
