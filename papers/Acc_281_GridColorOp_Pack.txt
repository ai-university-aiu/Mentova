Abstract

PrologAI Work Package 261 adds the gridcolorop pack (gco_*, Layer 240): fourteen
predicates for querying and transforming the color distribution of symbolic grids.
The pack covers color counting, distinct-color listing, Nth-color selection,
swapping, replacement, map-based substitution, keep-only masking, color removal,
palette cycling, frequency-rank replacement, palette application, most/least
color query, and binary inversion. All 44 acceptance tests pass.


1. Motivation

Grid reasoning tasks frequently require reasoning about which colors are present,
how frequent they are, and how to reassign or suppress colors according to rules.
Frequency rank is particularly important: many ARC-AGI-2 tasks encode abstract
relationships through relative color frequency (the most common color plays one
role, the least common another). The gridcolorop pack provides a dedicated,
composable vocabulary for all such color-distribution operations without requiring
callers to implement cell iteration from scratch.


2. Architecture

Layer 240 sits immediately above gridalign (Layer 239) in the Data Layer.
Depends only on library(lists): member/2. No cross-pack dependencies.

Key design decisions:

gco_sort_by_count_desc_ uses Neg is -N rather than -N as a Prolog term. In
SWI-Prolog, -N in a findall template creates the compound term -(N), not the
integer -N. When msort compares compound terms, it compares by functor name then
by arguments: -(3) < -(2) < -(1), so lower-count items sort first - the wrong
order. Arithmetic evaluation Neg is -N produces integers, which compare
numerically: -3 < -2 < -1, so msort gives ascending negatives = descending
originals. This is the critical correctness condition for all frequency-based
predicates (gco_nth_color, gco_rank_grid, gco_apply_palette, gco_most_least).

gco_cycle uses ((Idx + N) mod PLen + PLen) mod PLen to handle negative N
correctly. SWI-Prolog mod can return negative values for negative N, so the
double-mod idiom ensures the result is always in [0, PLen).

gco_apply_palette delegates to gco_apply_map after computing the replacement map
from (ByFreq[K] -> Palette[K]) pairs. This ensures consistent semantics between
the palette application and the general map application.

gco_invert is a binary operation: all non-bg cells become Bg, all Bg cells
become FgColor. It does not preserve distinctions among different non-bg colors.


3. Predicates

Color queries:

gco_color_counts/3        - cc(Color,Count) pairs sorted by count descending
gco_distinct_colors/3     - distinct non-bg colors sorted by count descending
gco_nth_color/4           - Nth most frequent non-bg color (0-indexed)
gco_count_distinct/3      - count of distinct non-bg colors
gco_most_least/4          - most and least frequent non-bg colors

Color transformations:

gco_swap/5                - exchange Color1 and Color2 throughout grid
gco_replace/5             - replace every OldColor cell with NewColor
gco_apply_map/4           - apply list of OldColor-NewColor substitutions
gco_keep_only/4           - keep Color cells; bg-ify all others
gco_remove_color/4        - replace Color cells with Bg
gco_invert/4              - binary invert: non-bg->Bg, Bg->FgColor

Frequency-based operations:

gco_cycle/5               - advance each palette-member cell N steps (wrapping)
gco_rank_grid/3           - replace non-bg cells with 0-indexed frequency rank
gco_apply_palette/4       - replace Kth most frequent color with Kth palette entry


4. Example Session

:- use_module(pack(gridcolorop/prolog/gridcolorop)).

% Count non-bg colors.
gco_color_counts([[r,r,g],[b,b,b]], b, Counts).
% Counts = [cc(r,2), cc(g,1)].

% Rank by frequency.
gco_rank_grid([[r,r,r],[g,g,b]], b, R).
% R = [[0,0,0],[1,1,b]].

% Apply palette: most frequent gets first palette color.
gco_apply_palette([[r,r,r],[g,g,b]], b, [x,y], R).
% R = [[x,x,x],[y,y,b]].

% Binary invert.
gco_invert([[r,b],[b,g]], b, x, R).
% R = [[b,x],[x,b]].

% Cycle palette forward.
gco_cycle([[r,g],[b,b]], [r,g,y], 1, b, R).
% R = [[g,y],[b,b]].

7. Conclusion

The gridcolorop pack provides fourteen gco_* predicates for color distribution
analysis and transformation on symbolic grids. The frequency-based operations
(rank grid, apply palette, most/least) enable tasks that require reasoning about
which colors are most or least prominent. Together with the earlier grid packs,
gridcolorop completes a comprehensive color reasoning vocabulary at Layer 240.
