ABSTRACT

We report the completion of the shared game-as-a-body harness specified in Volume 6,
Part 7 of the PrologAI Demonstration and Proof-of-Concept Plan. The harness enrolls
interactive game environments as bodies following the PrologAI Mind-Body pattern (PR 10),
routes game frames as visual perception signals, and dispatches Mentova's action decisions
as actuator commands through the tool faculty. Built once and pointed at four targets,
it demonstrates the perceive-reason-act cycle on three working game drivers - the
Abstraction and Reasoning Corpus (ARC-AGI), Raven's Progressive Matrices, and Baba Is You
- plus a documented stub for Pokémon Red and Emerald. All three ARC tasks pass; both
Raven's matrices are solved; both Baba levels produce glass-box plans. Every answer
includes a human-readable justification naming the specific inference steps used.


INTRODUCTION

Volume 6, Part 7 of the PrologAI Demonstration Plan specifies a shared infrastructure
that enrolls interactive game environments as bodies and drives Mentova through a
perceive-reason-act cycle. The guiding design principle is explicit: build it once, and
point it at several targets. The targets named in the specification are Pokémon Red and
Emerald (the crowd flagship), ARC-AGI-1 and ARC-AGI-2 (the credibility flagship),
Raven's Progressive Matrices (the nonverbal fluid-intelligence classic), and Baba Is You
(the jewel, where the player wins by rewriting the game rules as word-blocks).

The harness also implements the continual refinement thread from Part 3: the game step
log records every perceive-reason-act cycle as node_facts, giving the self-improvement
pack (PR 17) the history it needs to rewrite the harness as Mentova gains experience.

This accomplishment record documents the harness implementation, the four drivers, and the
ten demonstration queries that exercise the full cycle.


BACKGROUND AND RELATED WORK

PrologAI PR 10 (Mind-Body Interface) defines the body enrollment and percept-routing
protocol. Any body - a physical robot, a virtual robot, or a game environment - is
enrolled via manifest_body/3. Sensors arrive as perception_signal/3 terms; actuators
receive command terms via dispatch_command/2. The game harness follows this pattern
exactly, replacing hardware sensors with game frame observations and hardware actuators
with in-game action dispatches.

ARC-AGI (Chollet, 2019) is a benchmark that tests the ability to induce transformation
rules from small sets of examples and apply them to novel instances, with no pretraining
on the corpus. Raven's Progressive Matrices (Raven, 1936) is a nonverbal test of fluid
intelligence requiring induction of relational patterns from a partial matrix. Baba Is You
(Hempuli, 2019) is a puzzle game where the player wins by manipulating word-blocks that
encode the game rules, making it the most direct analogue of a mind rewriting its own
node_facts.

The continual refinement harness (PR 17) is the PrologAI mechanism through which Mentova
rewrites its own configuration from experience. The game step log feeds this harness.

Volume 6, Part 7 notes: "An honest note on all flagships: a first-version Mentova will
not top frontier agents on any of these; the win is the distinctive behavior shown on a
recognized task." This accomplishment is that first version.


METHOD

The game-as-a-body harness adds nine new files to the Mentova repository.

HARNESS CORE (src/mentova/game_body.pl)

  game_enroll/3    — enroll any game as a body with a named driver and capabilities.
                     Records a game_body_record fact; initializes the step counter.

  game_observe/3   — pull current game state as a perception_signal(visual, Frame, T).
                     Dispatches to the driver's observe predicate via module-qualified call.
                     Falls back to a placeholder frame if the driver has no state.

  game_reason/5    — apply Mentova's reasoning to a game percept to produce an action.
                     Dispatches to the driver's reason predicate with the query type.
                     Falls back to a heuristic action if the driver has no specific reasoning.

  game_act/4       — dispatch an action to the game body and record the confirmation.
                     Dispatches to the driver's act predicate. Increments the step counter.
                     Logs every cycle as a game_step_log node_fact for self-improvement.

  game_loop/5      — run N steps of the full perceive-reason-act cycle.
                     Supports stopping conditions: always_stop, goal_reached, max_steps.

  game_history/2   — retrieve the full step log for a game instance.

ARC-AGI DRIVER (src/mentova/games/arc.pl)

  Three toy ARC tasks are encoded as arc_task/3 node_facts:
    recolor_to_red   — replace all non-black cells with colour 2 (red).
                       Induced rule: recolor_nonblack(2). 3-pair training set.
    horizontal_flip  — mirror the grid left-to-right. Induced rule: horizontal_flip.
    fill_border      — set all border cells to colour 5 (grey). Rule: fill_border(5).

  arc_induce_rule/3 searches candidate rules in order of complexity and verifies each
  against all training pairs before accepting it. The induced rule is then applied to
  the test input. arc_verify/3 compares the proposed output against the expected output
  and returns pass/fail with the expected output shown on failure.

RAVEN'S PROGRESSIVE MATRICES DRIVER (src/mentova/games/ravens.pl)

  Two toy RPM tasks are encoded as ravens_task/3 node_facts:
    size_progression   — size increases left-to-right across each row.
                         Correct answer: option 1 (large grey circle).
    count_progression  — count increases left-to-right across each row.
                         Correct answer: option 3 (3 small grey squares).

  ravens_induce_rule/3 verifies each candidate rule against the first two rows.
  ravens_predict_missing/3 applies the induced rule to compute the missing 9th cell.
  ravens_select_option/3 finds which of the six answer options matches the prediction.

BABA IS YOU DRIVER (src/mentova/games/baba.pl)

  Two toy Baba levels are encoded as baba_level/3 node_facts:
    simple_win     — BABA IS YOU and FLAG IS WIN are active. Direct path to flag.
    rule_rewrite   — ROCK IS YOU (not BABA IS YOU). Need to reason about the current
                     player noun and plan accordingly.

  baba_plan_to_win/3 reads the active rule set, checks which noun is YOU and which
  is WIN, and plans the appropriate sequence of steps. The plan type is either
  direct_path_to_win or rule_rewrite_then_win. The justification cites teleological
  and procedural (or symbolic) reasoning types.

POKEMON STUB (src/mentova/games/pokemon.pl)

  A fully documented stub driver. All three interface predicates succeed with stub
  results. The stub frame represents an early game state in Pokemon Red: Pallet Town,
  a Charmander at level 5, no active battle, objective go_north_to_route_1. The stub
  explicitly documents the full implementation path: mGBA emulator with Lua scripting
  socket, Pokemon Red RAM map addresses, Python bridge process, SWI-Prolog socket
  library, and the mapping onto the Mind-Body herald pattern.

INTEGRATION

  The mentova_query/3 top-level interface was extended with a game clause that calls
  game_observe and game_reason in sequence, making game reasoning accessible through
  the same interface as all 48 reasoning rungs and Track A.

RESULTS

Pass criterion met (Volume 6, Part 7): the harness enrolls at least one game as a body,
runs one perceive-reason-act cycle, and returns the action with a human-readable
justification. The full demonstration shows this for three working drivers and one stub.

ARC-AGI results:
  recolor_to_red   — correct. Induced rule: recolor_nonblack(2). Rule verified on 3
                     training pairs by search_and_verify. Applied to test input, produced
                     the expected output grid exactly.

  horizontal_flip  — correct. Induced rule: horizontal_flip. Each training pair verified
                     by checking that reversing every row reproduces the output. Applied
                     to the test input, produced [[0,2,1],[3,0,0],[0,0,4]] exactly.

  fill_border      — correct. Induced rule: fill_border(5). Each training pair verified
                     by checking that border cells become 5 and interior cells are
                     unchanged. Applied to test input [[0,0,0],[0,2,0],[0,0,0]], produced
                     [[5,5,5],[5,2,5],[5,5,5]] with the interior cell 2 preserved.

Raven's Progressive Matrices results:
  size_progression — correct. Induced rule: size_increases_across_row. Verified on rows 1
                     and 2 of the training matrix. Predicted missing cell:
                     figure(shape(circle), size(large), colour(grey), count(1)). Matched
                     option 1 exactly. The justification names the induced rule, the
                     predicted figure, and the option selected.

  count_progression — correct. Induced rule: count_increases_across_row. Predicted missing
                      cell: figure(shape(square), size(small), colour(grey), count(3)).
                      Matched option 3 exactly.

Baba Is You results:
  simple_win level — correct plan. BABA IS YOU and FLAG IS WIN are active. Plan:
                     5 steps ending with touch_win_object(flag). Justification cites
                     direct_path_to_win with teleological and procedural reasoning.

  rule_rewrite level — correct reasoning about the current player noun (ROCK IS YOU, not
                       BABA IS YOU). Plan adapts: locate_player(rock) instead of
                       locate_player(baba). Justification correctly identifies the active
                       rule set and produces the appropriate plan type.

Pokemon stub — the perceive-reason-act cycle runs end-to-end on the stub driver. The
               stub frame represents a realistic game state (Pallet Town, Charmander
               level 5). The heuristic action is press(up) at step 0. The stub
               confirmation explicitly documents the next implementation step.


SIGNIFICANCE

The harness establishes three things.

First, it proves the universality of the body metaphor. An ARC grid, a Raven's matrix,
a Baba level, and a Pokemon world are structurally different environments, but all four
enroll through the same three predicates - game_enroll, game_observe, game_act - and
drive the same perceive-reason-act cycle. The Mind-Body pattern generalizes.

Second, it proves that induction is the right reasoning type for pattern-based games.
ARC and Raven's both require inducing a rule from examples and applying it. Mentova's
inductive reasoning module (PR 37), accessed through the game harness, solves all five
tasks correctly. The justification names the induced rule, the number of training pairs
it was verified against, and the method used.

Third, Baba Is You demonstrates that game rules are node_facts. When BABA IS YOU and
FLAG IS WIN are active rules, the plan is direct. When ROCK IS YOU instead, the plan
adapts - because the active rule set is read from node_facts, and the plan generator
reads those facts. This is not a hard-coded case; it is genuinely rule-driven reasoning.

The self-improvement thread is also in place: every perceive-reason-act cycle is logged
as a game_step_log node_fact. The continual refinement harness (PR 17) can read this
history to improve Mentova's strategy over time - the developmental story that Volume 6
describes for the Pokemon flagship.

The Baba Is You jewel - where Mentova would rewrite game rules by pushing word-blocks
to change its own node_facts - is the natural next step in the Baba driver. The
infrastructure is in place.


HONEST LIMITATIONS

The ARC tasks are toy-sized. The real ARC-AGI benchmark contains 400 training tasks and
400 evaluation tasks with grids up to 30x30 and transformations far more complex than
the three demonstrated here. Extending the driver to handle the full ARC-AGI dataset
requires a richer primitive library covering rotation, reflection, tiling, object
counting, gravity, connectivity, and many more transformations.

The Raven's matrices are toy-sized. The real RPM has 60 items with increasing difficulty.
The rule induction here uses a simple search over candidate rules; real RPM solving
requires richer relational representations and potentially multi-dimensional rule
induction.

The Baba Is You driver does not yet execute physical moves. The plan is computed and
returned as a sequence of steps, but the baba_apply_move predicate (which moves entity
positions on the grid) is defined but not fully connected to the level win-check. Winning
by touching the flag requires checking whether the player entity's position overlaps with
the flag entity's position after each move.

The Pokemon driver requires a live emulator bridge that does not yet exist. The stub
documents the full path but cannot play the game.


REPRODUCIBILITY

Requirements: SWI-Prolog 9.0.4 or later.

  % Clone the Mentova repository.
  git clone https://github.com/ai-university-aiu/Mentova.git
  % Change to the Mentova directory.
  cd Mentova
  % Run the full game harness demonstration.
  swipl -l demos/game_harness_demo.pl -g "run_game_harness" -t halt

Individual ARC queries can be run interactively:

  % Load the ARC driver module.
  swipl -l src/mentova/games/arc.pl
  % Solve the recolor_to_red task.
  ?- arc_solve(recolor_to_red, Grid, Just).
  % Solve the fill_border task.
  ?- arc_solve(fill_border, Grid, Just).

Individual Raven's queries:

  % Load the Ravens driver module.
  swipl -l src/mentova/games/ravens.pl
  % Solve the size progression matrix.
  ?- ravens_solve(size_progression, Option, Just).

Through the top-level mentova_query/3 interface:

  % Load Mentova and run ARC via the standard query interface.
  swipl -l src/mentova/mentova.pl
  % Boot Mentova then run a game reasoning query.
  ?- mentova_boot,
     game_enroll(g1, arc, [perceive,induce,act]),
     arc_set_task(g1, recolor_to_red),
     mentova_query(game, game_reason(g1, inductive), R).
