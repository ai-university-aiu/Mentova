ABSTRACT

We report Rung 23 of Mentova's 48-rung reasoning ladder: temporal reasoning.
Mentova answers ordering and duration questions over a timeline of daily events.
Breakfast (tick 7) is before lunch (tick 12); the gap from wake_up (tick 6) to
dinner (tick 19) is 13 hours; lunch is between commute and dinner in the sequence.
All before-and-after answers are correct. Pass criterion met.


INTRODUCTION

Temporal reasoning requires representing when events happen and answering questions
about their order, duration, and relative timing. Rung 23 equips Mentova with a
timeline of named daily events and a set of temporal predicates.


BACKGROUND

Allen's interval algebra (1983) provides a formal framework for temporal reasoning
over intervals. Mentova uses a simpler point-based timeline (events at timestamps)
sufficient for the ordered-sequence pass criterion.


METHOD

A new module, src/mentova/temporal.pl, implements:

    mentova_temporal(+Query, -Result, -Justification)

Eight daily events with timestamps (6-22) and five with durations.

Supported queries: before, after, between, gap, duration, when, sequence_order, window.


DEMONSTRATION

    % Query Mentova for temporal reasoning about 'before(breakfast'.
    ?- mentova_query(temporal, before(breakfast, lunch), R).
    R = answer(yes, just(temporal(breakfast, before, lunch, t1(7), t2(12), result(yes))))

    Breakfast at tick 7 is before lunch at tick 12.

    % Query Mentova for temporal reasoning about 'gap(wake_up'.
    ?- mentova_query(temporal, gap(wake_up, dinner), R).
    R = answer(13, just(temporal_gap(wake_up, dinner, hours(13))))

    13 hours from wake_up to dinner.


RESULTS

All temporal queries return correct answers with timestamps named in justifications.
Pass criterion met: before-and-after is correct.


SIGNIFICANCE

Temporal reasoning is prerequisite for narrative reasoning (Rung 41), procedural
reasoning (Rung 36), and planning (Rung 38). Events must be ordered before
narratives can be followed or plans evaluated.


HONEST LIMITATIONS

Point-based timeline does not handle intervals, overlapping events, or temporal
uncertainty. Allen's full interval algebra would be needed for richer temporal queries.


REPRODUCIBILITY

Requirements: SWI-Prolog 9.0.4 or later.

  git clone https://github.com/ai-university-aiu/Mentova.git
  cd Mentova
  % Load the Mentova bootstrap into SWI-Prolog, execute the query, and halt.
  swipl -l src/mentova/mentova.pl \
        % Supply a goal or termination instruction to the SWI-Prolog command.
        -g "mentova_boot, \
            % Query Mentova for temporal reasoning about 'before(breakfast'.
            mentova_query(temporal, before(breakfast,lunch), R1), write(R1), nl, \
            % Query Mentova for temporal reasoning about 'gap(wake_up'.
            mentova_query(temporal, gap(wake_up,dinner), R2), write(R2), nl" \
        % Supply a goal or termination instruction to the SWI-Prolog command.
        -t halt
