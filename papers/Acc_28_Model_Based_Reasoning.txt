ABSTRACT

We report Rung 28 of Mentova's 48-rung reasoning ladder: model-based reasoning.
Mentova maintains internal models — linear, threshold, and state-transition — and
uses them to predict outputs and trace state evolution. For a temperature sensor
with linear model slope=2/intercept=5, input 10 yields predicted output 25. For a
thermostat state machine in state dry, input rain causes a transition to state wet.
Pass criterion met: prediction from internal model returned with justification.


INTRODUCTION

Model-based reasoning allows an agent to consult an internal model of a system —
rather than direct observations — to make predictions and trace behavior. Rung 28
equips Mentova with three model types: linear (y=mx+b), threshold (if x≥t then
class A else class B), and state-transition machines.


BACKGROUND

Model-based reasoning (Davis, 1984; de Kleer and Williams, 1987) is foundational
to diagnosis, planning, and simulation in AI. A model captures regularities that
allow prediction beyond observed cases.


METHOD

A new module, src/mentova/model_based.pl, implements:

    mentova_model(+Query, -Result, -Justification)

Model types: linear (predict_linear), threshold (predict_threshold),
state-transition (predict_transition). State machines: weather_fsm
(dry/wet/humid transitions) and traffic_fsm (free/slow/stopped transitions).


DEMONSTRATION

    % Query Mentova for model reasoning about 'predict_linear(temp_sensor'.
    ?- mentova_query(model, predict_linear(temp_sensor, 10), R).
    R = answer(25, just(linear_model(temp_sensor, slope(2), intercept(5), input(10), output(25))))

    Linear model: y = 2*10 + 5 = 25.

    % Query Mentova for model reasoning about 'predict_transition(dry'.
    ?- mentova_query(model, predict_transition(dry, rain), R).
    R = answer(wet, just(fsm_transition(weather_fsm, dry, rain, wet)))

    State machine: dry + rain → wet.


RESULTS

Linear prediction: input 10 → output 25. Threshold prediction: input 6 → class high.
State transition: dry + rain → wet. All with justifications.
Pass criterion met: prediction from internal model returned with justification.


SIGNIFICANCE

Model-based reasoning allows Mentova to predict outcomes in new situations by
consulting its internal models rather than searching for matching past cases.
It is complementary to case-based reasoning (Rung 24).


HONEST LIMITATIONS

Models are manually specified with fixed parameters. A learning system would
fit models automatically from data. No uncertainty is propagated through predictions.


REPRODUCIBILITY

        -t halt
