Abstract

PrologAI gains Work Package 130: the signal pack, a Layer 109 symbolic
reasoning module providing 14 predicates for 1D signal analysis of integer
lists and grid profiles. The pack finds local extrema, tests monotone order,
computes consecutive differences and slope signs, locates the index of the
maximum or minimum value, partitions values by threshold, and extracts row,
column, and cross-shaped value profiles from grids. All 42 PLUnit acceptance
tests pass.


1. Motivation

Many grid reasoning tasks reduce to 1D analysis: does this row contain a peak?
Is this column strictly increasing? What fraction of column values exceed a
threshold? Which row index has the highest sum? These questions arise when
detecting symmetry, identifying boundaries, comparing profiles across frames,
and scanning for regularities that a purely spatial predicate set would handle
only awkwardly.

The signal pack makes 1D signal analysis a first-class operation in PrologAI,
bridging the gap between raw grid data (rows and columns as integer lists) and
higher-level structural reasoning (peaks mark boundaries, valleys mark centers,
increasing profiles indicate gradients, threshold partitions separate foreground
from background).


2. Predicate Inventory (14 predicates, sg_* prefix)

2.1 Local Extrema

sg_peaks(+List, -PeakIdxs)
PeakIdxs is the sorted list of 0-based indices of strict interior local
maxima. Index I is a peak if List[I] > List[I-1] and List[I] > List[I+1] and
I is not the first or last element. Returns [] for lists shorter than 3 or
when no interior maxima exist.

sg_valleys(+List, -ValleyIdxs)
ValleyIdxs is the sorted list of 0-based indices of strict interior local
minima by the symmetric condition.

Examples:
sg_peaks([1,3,1,3,1], P).   % P = [1,3]
sg_valleys([5,1,5,1,5], V). % V = [1,3]


2.2 Position of Extrema

sg_argmax(+List, -Idx)
Idx is the 0-based index of the maximum value. First occurrence wins on ties.
Implemented with max_list/2 to find the maximum, then nth0/3 plus cut for
determinism.

sg_argmin(+List, -Idx)
Idx is the 0-based index of the minimum value.

Examples:
sg_argmax([0,0,5], Idx).  % Idx = 2
sg_argmin([3,1,2], Idx).  % Idx = 1
sg_argmax([3,3,3], Idx).  % Idx = 0  (first occurrence)


2.3 Differences and Slope Signs

sg_diffs(+List, -Diffs)
Diffs is the list of consecutive element-wise differences: Diffs[i] =
List[i+1] - List[i]. Length(Diffs) = Length(List) - 1. Returns [] for lists
of length 0 or 1.

sg_slope_signs(+List, -Signs)
Signs maps each consecutive difference to its sign: 1 (positive), -1
(negative), or 0 (equal). Uses sg_diffs and maplist over the helper sg_sign_.

Examples:
sg_diffs([1,3,6], D).          % D = [2,3]
sg_slope_signs([1,3,3,1], S).  % S = [1,0,-1]


2.4 Monotone Order Tests

Four predicates test ordering properties. All are vacuously true for lists of
length 0 or 1.

sg_is_increasing(+List)   - strictly increasing (A < B for each pair)
sg_is_decreasing(+List)   - strictly decreasing (A > B)
sg_is_nondecreasing(+List) - A =< B (equal elements allowed)
sg_is_nonincreasing(+List) - A >= B

These predicates are recursive with base cases for the empty list and the
single-element list, using cut to avoid choicepoints.

Examples:
sg_is_nondecreasing([1,1,2,3]).  % true
sg_is_increasing([1,1,2]).       % false (equal pair at positions 0-1)


2.5 Threshold Partitioning

sg_threshold(+List, +T, -AboveIdxs, -BelowOrEqIdxs)
AboveIdxs is the sorted list of 0-based indices where List[I] > T.
BelowOrEqIdxs is the sorted list where List[I] =< T.
Implemented with two findall calls over between/3 and nth0/3.

Example:
sg_threshold([1,5,3,7,2], 4, Above, Below).
% Above = [1,3],  Below = [0,2,4]


2.6 Grid Profiles

sg_row_profile(+Grid, +R, -Values)
Values is the list of cell values in row R (0-based) of Grid.
Equivalent to nth0(R, Grid, Values).

sg_col_profile(+Grid, +C, -Values)
Values is the list of cell values in column C (0-based) of Grid across all
rows. Implemented with maplist and a YALL (Yet Another Lambda Library) lambda
to capture the outer column index C inside maplist:
  maplist([Row, V]>>(nth0(C, Row, V)), Grid, Values).

sg_cross_profile(+Grid, +R, +C, -RowVals, -ColVals)
RowVals is row R; ColVals is column C. Together they form the cross-shaped
value profile of cell (R, C) in the grid.

Examples:
sg_col_profile([[1,2,3],[4,5,6],[7,8,9]], 1, V).  % V = [2,5,8]
sg_cross_profile([[1,2,3],[4,5,6],[7,8,9]], 1, 1, R, C).
% R = [4,5,6], C = [2,5,8]


3. Implementation Notes

The four monotone order predicates (sg_is_increasing, sg_is_decreasing,
sg_is_nondecreasing, sg_is_nonincreasing) share a common recursive structure:
base cases for [] and [_], then a two-element head check followed by recursive
application to the tail starting from the second element.

sg_diffs uses a three-clause recursive structure: empty and single-element
base cases with cut, then a two-element head clause that computes D is B - A
and recurses with [B | Rest].

sg_col_profile uses a YALL lambda because the outer predicate argument C is a
free variable that must be visible inside the maplist goal. The lambda syntax
[Row, V]>>(nth0(C, Row, V)) creates a closure that captures C from the
enclosing scope.

Imports: library(lists) for nth0/3, max_list/2, min_list/2; library(apply)
for maplist/3.


4. Test Results

42 PLUnit acceptance tests pass with no warnings. Three tests per predicate
cover the primary case, an edge case, and a boundary or failure case.

Predicates tested: all 14 sg_* predicates.
Test file: packs/signal/test/test_signal.pl.


5. Significance

The signal pack makes row and column profiles first-class objects for analysis.
Any reasoning that detects gradients, monotone regions, local extrema, or
threshold-based partitions in a grid now has a clean predicate-level interface
rather than requiring ad-hoc list traversal in each solver.

Grid profiles extracted by sg_row_profile and sg_col_profile can be passed
directly to sg_peaks, sg_argmax, sg_threshold, or the monotone tests, creating
a composable pipeline from raw grid data to structural observations.
