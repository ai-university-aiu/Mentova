ABSTRACT

We report Rung 20 of Mentova's 48-rung reasoning ladder: hypothetical reasoning.
Mentova explores the consequences of a supposition without asserting it as true
in the knowledge base. Suppose fire: the system derives smoke as a consequence
(via causes(fire, smoke)) without adding fire or smoke to the main KB. The
derivation is correctly attributed to the supposition, and the KB remains
unmodified. Pass criterion met: results derived without polluting KB.


INTRODUCTION

Hypothetical reasoning explores the consequences of a supposition without
committing to its truth. "Suppose we start a fire — what would follow?" This
differs from counterfactual reasoning in that the supposition may be entirely
novel, not a modification of a known actual state. And it differs from assertion
in that the supposition is not added to the knowledge base; it exists only for
the duration of the hypothetical query.

Rung 20 implements hypothetical reasoning as a temporary context extension:
the supposition is prepended to a local context list, consequences are derived
within that context by propagating via causes/2, and the main KB is never touched.


BACKGROUND

Hypothetical reasoning has been studied in AI as assumption-based truth maintenance
(ATMS, de Kleer 1986) and in non-monotonic reasoning as conditional reasoning.
In planning, hypothetical reasoning is used to evaluate action consequences before
committing to an action.

The key property is isolation: the supposition's consequences must not pollute
the persistent knowledge base. Mentova achieves this by passing the supposition
as a parameter rather than asserting it.


METHOD

A new module, src/mentova/hypothetical.pl, implements:

    mentova_hypothetical(+Query, -Result, -Justification)

The suppose(P) query:
  1. Prepends P to an empty context list.
  2. Calls hypothetical_derive(P, [], Derived) to propagate via causes/2.
  3. Returns the derived consequences.
  4. The note in the justification states: 'KB unchanged; supposition is temporary'.

The main KB is never modified. No assert/retract is called.

Query forms:

    suppose(P)           consequences of supposing P
    suppose(P, Context)  consequences in a richer context
    two_step(P)          two-hop causal propagation
    verify_kb_intact(P)  confirm P was not asserted into KB

Top-level clause:

    % Query Mentova for hypothetical reasoning about 'HypQuery'.
    mentova_query(hypothetical, HypQuery, answer(Result, Just)) :-
        % State the fact: mentova hypothetical(...).
        mentova_hypothetical(HypQuery, Result, Just).


DEMONSTRATION

Query 1 — suppose(fire):

    % Query Mentova for hypothetical reasoning about 'suppose(fire)'.
    ?- mentova_query(hypothetical, suppose(fire), R).
    R = answer(consequents(fire, [smoke]),
               just(hypothetical(suppose(fire),
                                 derived_without_kb_pollution([smoke]),
                                 note('KB unchanged; supposition is temporary'))))

    Supposition: fire is true.
    Derived consequence: smoke (via causes(fire, smoke)).
    Note: KB unchanged.

Query 2 — two_step(sunlight):

    suppose sunlight → derived: plant_growth (step 1)
    continue from plant_growth → (no further cause in KB, so stops)

Query 3 — verify_kb_intact(magic_fact):

    Returns kb_intact because magic_fact was never asserted.


RESULTS

The suppose(fire) query derives [smoke] without polluting the KB. The justification
names the derivation and notes that the KB is unchanged.

Pass criterion met: results derived without polluting KB.


SIGNIFICANCE

Hypothetical reasoning is essential for safe deliberation. A planning agent that
must evaluate candidate actions cannot assert each candidate into the KB — it
must reason about it hypothetically and commit only to the chosen action.

Rung 20's hypothetical engine is the substrate for Rung 38 (practical/deliberative
reasoning) and Rung 26 (scientific hypothesis testing), where hypotheses must be
evaluated without being asserted as facts.


HONEST LIMITATIONS

The current implementation only propagates one or two hops via causes/2. A general
hypothetical reasoner would propagate through any applicable rules in the knowledge base.

The context-passing approach is correct but requires explicit management. In a larger
system, a hypothetical context stack would be managed by the inference engine.


REPRODUCIBILITY

Requirements: SWI-Prolog 9.0.4 or later.

  git clone https://github.com/ai-university-aiu/Mentova.git
  cd Mentova
  % Load the Mentova bootstrap into SWI-Prolog, execute the query, and halt.
  swipl -l src/mentova/mentova.pl \
        % Supply a goal or termination instruction to the SWI-Prolog command.
        -g "mentova_boot, \
            % Query Mentova for hypothetical reasoning about 'suppose(fire)'.
            mentova_query(hypothetical, suppose(fire), R1), write(R1), nl" \
        % Supply a goal or termination instruction to the SWI-Prolog command.
        -t halt
