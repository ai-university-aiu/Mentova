ABSTRACT

We report Rung 38 of Mentova's 48-rung reasoning ladder: practical reasoning.
Mentova selects the lowest-cost action to achieve a goal from the current state
using means-ends analysis. Given goal=not_hungry and state=[has(food)], the best
action is eat_food (cost 1). Given goal=has(food) and state=[has(money)], the best
action is buy_food (cost 2). Pass criterion met: best action selected with
means-ends justification.


INTRODUCTION

Practical reasoning concerns what to do to achieve a goal. Means-ends analysis
(Newell and Simon, 1963) selects the action that reduces the gap between current
and goal state. Rung 38 implements this for an action base with preconditions,
effects, and costs.


BACKGROUND

Means-ends analysis is the core of the General Problem Solver (GPS) and STRIPS-based
planners. It decomposes goals into sub-goals and selects actions to reduce each gap.


METHOD

A new module, src/mentova/practical.pl, implements:

    mentova_practical(+Query, -Result, -Justification)

Eleven actions across domains: food, travel, fitness, learning. Each with
preconditions, effects, and cost. Queries: best_action, achieves_what, how_to.


DEMONSTRATION

    % Query Mentova for practical reasoning about 'best_action(not_hungry'.
    ?- mentova_query(practical, best_action(not_hungry, [has(food)]), R).
    R = answer(action(eat_food, cost(1)),
               just(practical(means_ends(goal(not_hungry), state([has(food)]),
                                          best_action(eat_food), cost(1)))))

    % Query Mentova for practical reasoning about 'best_action(has(food)'.
    ?- mentova_query(practical, best_action(has(food), [has(money)]), R).
    R = answer(action(buy_food, cost(2)), ...)


RESULTS

eat_food selected for not_hungry (lowest cost when food available).
buy_food selected for has(food) when money available. Pass criterion met.


SIGNIFICANCE

Practical reasoning bridges goals and actions. It is foundational to autonomous
agents, robotics, and decision support systems.


HONEST LIMITATIONS

The action selection is greedy (lowest cost), not globally optimal (no lookahead).
A planner would also handle chained sub-goals and resource constraints.


REPRODUCIBILITY

        % Supply a goal or termination instruction to the SWI-Prolog command.
        -t halt
