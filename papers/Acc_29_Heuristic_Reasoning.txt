ABSTRACT

We report Rung 29 of Mentova's 48-rung reasoning ladder: heuristic reasoning.
Mentova uses heuristic-guided search (greedy best-first) to find paths in a
five-node weighted graph. Given start=a, goal=e, heuristic estimates, greedy
search finds path [a, c, e] with total cost 10. The full shortest-path algorithm
(Dijkstra) confirms the optimal path cost. Pass criterion met: path found with
heuristic-guided search.


INTRODUCTION

Heuristic reasoning uses approximation and domain-specific knowledge to guide
search toward a goal efficiently. Rung 29 equips Mentova with a weighted graph,
heuristic estimates, greedy best-first search, and Dijkstra's shortest-path algorithm.


BACKGROUND

Heuristic search (Pearl, 1984) is one of the core methods in AI problem solving.
Greedy best-first search expands the node with the lowest heuristic estimate.
A* combines heuristic estimate with path cost so far for optimal search.


METHOD

A new module, src/mentova/heuristic.pl, implements:

    mentova_heuristic(+Query, -Result, -Justification)

Graph: nodes {a, b, c, d, e}, edges with costs. Heuristic: h_estimate/3 table.
Algorithms: greedy_search/5 (best-first expansion) and shortest_path/4 (Dijkstra).
Supported queries: greedy(Start,Goal,Cost), shortest(Start,Goal).


DEMONSTRATION

    % Query Mentova for heuristic reasoning about 'greedy(a'.
    ?- mentova_query(heuristic, greedy(a, e, Cost), R).
    R = answer(path([a, c, e], cost(10)),
               just(greedy_search(a, e, path([a,c,e]), cost(10))))

    Greedy search finds [a, c, e] with cost 10.

    % Query Mentova for heuristic reasoning about 'shortest(a'.
    ?- mentova_query(heuristic, shortest(a, e), R).
    R = answer(path([a,c,e], cost(10)), just(dijkstra(a, e, path([a,c,e]), cost(10))))


RESULTS

Greedy path: [a, c, e], cost 10. Dijkstra confirms optimal cost 10.
Pass criterion met: path found with heuristic-guided search.


SIGNIFICANCE

Heuristic reasoning is the foundation of planning, routing, and resource allocation
under time or computation constraints. Greedy search is practical for large spaces.


HONEST LIMITATIONS

Greedy best-first search is not guaranteed to find the optimal path (though it does
so here). A* would guarantee optimality with an admissible heuristic. The graph is
small and fixed; a larger dynamic graph would require more sophisticated data structures.
