Acc_254: gridconv Pack - Sliding Window, Pattern Matching, Density Maps, and Square Morphology (gcv_*, Layer 213)

Date: 2026-06-26


Abstract

PrologAI Work Package 234 adds the gridconv pack (gcv_*, Layer 213): fourteen
predicates for grid convolution operations. The pack covers sliding window
extraction with out-of-bounds fill, windowed color counting, density map
construction, majority color computation, majority-map generation, window
uniformity testing, subgrid pattern matching, hot spot detection, and square
structuring element morphology (dilation and erosion). All 46 acceptance
tests pass.


1. Motivation

ARC-AGI-2 tasks frequently involve local neighborhood reasoning: detecting
dense color regions, finding repeated patterns, applying local voting rules,
and expanding or shrinking regions using square rather than cross-shaped
structuring elements. The gridconv pack provides the complete symbolic toolkit
for these convolution-based operations. Window density maps expose spatial
concentration structure. Pattern matching enables rule detection and subgrid
search. Square structuring element morphology complements the cross-shaped
morphology of gridmorph (Layer 212) with full-square neighborhoods.


2. Architecture

Layer 213 sits immediately above gridmorph (Layer 212). Uses library(lists)
only (nth0/3, member/2, append/3). msort/2 is a SWI-Prolog built-in.
No cross-pack dependencies.

Private helpers:
- gcv_dims_/3: grid height and width
- gcv_cell_/4: cell value at row R, column C; fails out of bounds
- gcv_cell_fill_/5: cell value with fallback Fill for out-of-bounds
- gcv_build_/4: build H x W grid using lambda Goal(R, C, V)
- gcv_inbounds_vals_/6: collect all in-bounds values in (2N+1)x(2N+1) window
- gcv_mode_/2: most frequent element in list; ties by term order
- gcv_max_run_/7: recursive accumulator for longest run in sorted list
- gcv_all_eq_/1: test whether all list elements are equal
- gcv_count_inbounds_/5: count in-bounds cells in window (any color)
- gcv_match_at_/4: test whether Pattern exactly matches Grid at (R,C)

Window radius convention: radius N gives a (2N+1) x (2N+1) neighborhood.
gcv_count_in ignores out-of-bounds (no phantom fill). gcv_window fills.
gcv_erode_sq correctness: compares Color count to total in-bounds count
(not to (2N+1)^2) so boundary cells behave correctly.


3. Predicates

Window operations:

gcv_window/6 - extract (2N+1)x(2N+1) window; Fill used for out-of-bounds
gcv_count_in/6 - count Color in in-bounds window cells only
gcv_density_map/4 - grid of Color counts per (2N+1)x(2N+1) window

Window statistics:

gcv_majority/5 - most frequent color among in-bounds window cells
gcv_majority_map/3 - each cell replaced by local majority color (radius N)
gcv_uniform_at/5 - yes/no: all in-bounds window cells are the same color
gcv_uniform_cells/3 - list of (R,C) where in-bounds window is uniform

Pattern matching:

gcv_find_pattern/3 - all (R,C) where Pattern exactly matches as subgrid
gcv_count_pattern/3 - count matching subgrid positions
gcv_has_pattern/2 - test if pattern occurs at least once

Density threshold:

gcv_hot_spots/5 - cells where Color count in window >= Threshold

Square SE morphology:

gcv_dilate_sq/4 - cell becomes FgColor if any (2N+1)x(2N+1) window cell is FgColor
gcv_erode_sq/4 - FgColor cell loses color if any in-bounds window cell is not FgColor
gcv_replace_pattern/4 - replace first (row-major) Pattern match with Replacement


4. Example Session

:- use_module(pack(gridconv/prolog/gridconv)).

% Density map for a single r dot:
Dot = [[x,x,x,x,x],[x,x,x,x,x],[x,x,r,x,x],[x,x,x,x,x],[x,x,x,x,x]].
gcv_density_map(Dot, 1, r, DMap).
% DMap[R][C] = 1 for the 9 cells in the 3x3 region around (2,2); 0 elsewhere

% Window fill at corner:
G = [[r,r,x],[r,r,x],[x,x,x]].
gcv_window(G, 0, 0, 1, z, W).
% W = [[z,z,z],[z,r,r],[z,r,r]]

% Pattern matching:
Grid = [[x,x,x,x,x],[x,r,r,x,x],[x,r,r,x,x],[x,x,x,r,r],[x,x,x,r,r]].
gcv_find_pattern(Grid, [[r,r],[r,r]], Positions).
% Positions = [1-1, 3-3]

% Square dilation vs cross dilation:
gcv_dilate_sq(Dot, r, 1, Sq).
% Sq has 9 r cells (full 3x3 square)
% gmo_dilate(Dot, r, 1, Cross) would give 5 r cells (cross shape)

% Majority map smoothing:
Mixed = [[r,x,r],[x,r,x],[r,x,r]].
gcv_majority_map(Mixed, 1, Smoothed).
% Center (1,1): 4 r vs 5 x (including corners) -> x? depends on in-bounds count
% Edge cells: partial windows -> majority varies


5. Test Coverage

All 46 acceptance criteria (AC-GCV-001 through AC-GCV-046) exercised in
packs/gridconv/test/test_gridconv.pl. Tests cover:
- gcv_window: center (no fill needed), corner fill, radius-0
- gcv_count_in: center (4), corner (4 in-bounds), zero, radius-0 (1)
- gcv_density_map: dot center (1), all-x (0), all-r N=0 (1)
- gcv_majority: center rb (x), corner rb (r), uniform (r)
- gcv_majority_map: uniform unchanged, dot center (x)
- gcv_uniform_at: all-r yes, mixed no, corner yes, N=0 single cell yes
- gcv_uniform_cells: all-r N=0 (9), all-r N=1 (9), rb N=1 (1 corner)
- gcv_find_pattern: identity, 1x1, not found, 2x2 block (2 positions)
- gcv_count_pattern: one match, zero matches
- gcv_has_pattern: found, not found
- gcv_hot_spots: dot (9 cells), all-x (0), high threshold (1)
- gcv_dilate_sq: N=0 identity, dot fills (9 cells), uniform unchanged
- gcv_erode_sq: N=0 identity, block to center (1 cell), uniform unchanged
- gcv_replace_pattern: first match replaced, no match unchanged
- Combined: find/count consistent, density matches count_in, dilate increases, erode decreases, hot_spots 9 cells
