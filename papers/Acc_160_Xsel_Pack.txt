Abstract

PrologAI gains Work Package 140: the xsel pack, a Layer 119 symbolic
reasoning module providing 14 predicates for selecting, locating, and
transforming grid cells based on numerical value comparisons. Predicates
cover threshold-based cell selection, closed-interval range queries,
inequality selection, maximum and minimum cell location, binary threshold
mapping, conditional value replacement, and ordinal value ranking. All 42
PLUnit acceptance tests pass with no warnings.


1. Motivation

Many visual reasoning tasks require asking "which cells have values above
a given level?", "where is the maximum?", "replace all low values with a
background color", or "which value is second-largest?". The arith pack
provides arithmetic on cells and finds global max/min values. The histogram
pack counts value frequencies and ranks by frequency. The count pack tests
equality. None of these provides order-based selection of individual cells
or ordinal ranking of values by their numerical magnitude.

The xsel pack fills this gap with a unified set of predicates organized
around the theme of value comparison: from simple less-than or greater-than
cell lists through to a full ordinal ranking of every cell in the grid.


2. Predicate Inventory (14 predicates, xs_* prefix)

2.1 Threshold Cell Selectors

xs_cells_lt(+Grid, +V, -Cells)
Returns the sorted list of R-C pairs for cells whose value is strictly less
than V. Uses findall with a Val < V arithmetic guard, then sort/2 to produce
canonical order. Empty list if no cell satisfies the condition.

xs_cells_gt(+Grid, +V, -Cells)
Returns the sorted list of R-C pairs for cells whose value is strictly greater
than V. Same pattern with Val > V.

xs_cells_le(+Grid, +V, -Cells)
Returns the sorted list of R-C pairs for cells with value less than or equal
to V. Guard: Val =< V.

xs_cells_ge(+Grid, +V, -Cells)
Returns the sorted list of R-C pairs for cells with value greater than or
equal to V. Guard: Val >= V.

Examples:
xs_cells_lt([[1,2],[3,4]], 3, Cs).   % Cs = [0-0, 0-1]  (values 1 and 2)
xs_cells_ge([[1,2],[3,4]], 3, Cs).   % Cs = [1-0, 1-1]  (values 3 and 4)


2.2 Range and Inequality Selectors

xs_cells_between(+Grid, +Lo, +Hi, -Cells)
Returns sorted R-C pairs for cells with Lo =< value =< Hi (closed interval).
Uses a combined arithmetic guard: Val >= Lo, Val =< Hi.

xs_cells_ne(+Grid, +V, -Cells)
Returns sorted R-C pairs for cells whose value is not structurally equal to V.
Uses Val \== V which handles ground integer comparisons cleanly.

Examples:
xs_cells_between([[0,1],[2,3]], 1, 2, Cs).  % Cs = [0-1, 1-0]
xs_cells_ne([[0,1],[1,0]], 0, Cs).          % Cs = [0-1, 1-0]
xs_cells_ne([[1,1],[1,1]], 1, Cs).          % Cs = []


2.3 Extremum Locators

xs_max_cells(+Grid, -Cells)
Returns the sorted list of R-C pairs of all cells containing the maximum
value in Grid. Two-pass: first collects all cell values via nested member/2
and applies max_list/2; then collects positions where the cell equals MaxV.
Returns multiple cells when the maximum appears more than once.

xs_min_cells(+Grid, -Cells)
Same two-pass approach using min_list/2 for the minimum.

Examples:
xs_max_cells([[1,2],[3,4]], Cs).        % Cs = [1-1]  (4 is the max)
xs_max_cells([[5,0],[0,5]], Cs).        % Cs = [0-0, 1-1]  (two cells)
xs_min_cells([[3,3],[3,3]], Cs).        % Cs = [0-0, 0-1, 1-0, 1-1]


2.4 Conditional Replacement

xs_threshold(+Grid, +T, +OnV, +OffV, -Grid2)
Produces Grid2 by replacing each cell V with OnV if V >= T, otherwise with
OffV. This binary threshold is useful for converting a grayscale-style grid
to a two-color mask. Applies the mapping via nested maplist (row then cell)
with an if-then-else in the private helper xs_thresh_val_.

xs_replace_lt(+Grid, +V, +New, -Grid2)
Produces Grid2 by replacing every cell with value strictly less than V with
New, leaving other cells unchanged. Useful for clamping low outliers to a
background value.

xs_replace_gt(+Grid, +V, +New, -Grid2)
Produces Grid2 by replacing every cell with value strictly greater than V
with New. Useful for clamping high outliers.

Examples:
xs_threshold([[1,3],[5,2]], 3, 1, 0, G).  % G = [[0,1],[1,0]]
xs_replace_lt([[1,3],[2,4]], 3, 0, G).    % G = [[0,3],[0,4]]
xs_replace_gt([[1,2],[3,4]], 0, 0, G).    % G = [[0,0],[0,0]] (all > 0)


2.5 Value Ranking

xs_rank_vals(+Grid, -Vals)
Returns the sorted list of unique values appearing in Grid in ascending order.
The position of each value in this list is its 0-based ordinal rank (position
0 = smallest, position N-1 = largest). Uses findall with nested member/2 to
collect all cell values, then sort/2 which simultaneously deduplicates and
orders them. Computing the rank ladder is essentially free once the grid is
enumerated.

xs_val_rank(+Grid, +V, -Rank)
Returns the 0-based ordinal rank of value V among all unique cell values in
Grid. Delegates to xs_rank_vals to get the sorted unique list, then uses
nth0(Rank, Vals, V) with a cut to commit to the single matching position.
Fails if V does not appear in Grid.

xs_rank_grid(+Grid, -RankGrid)
Produces a new grid where each cell contains the ordinal rank of its original
value. Calls xs_rank_vals once to build the rank ladder, then applies
xs_rank_val_ (nth0 + cut) to every cell via nested maplist. Converts any
integer-valued grid into a compact representation using ranks 0..N-1.

Examples:
xs_rank_vals([[3,1],[2,1]], Vs).         % Vs = [1, 2, 3]
xs_val_rank([[3,1],[2,1]], 3, R).        % R = 2
xs_rank_grid([[3,1],[2,1]], G).          % G = [[2,0],[1,0]]
xs_rank_grid([[5,3],[1,7]], G).          % G = [[2,1],[0,3]]


3. Implementation Notes

3.1 Choicepoint Safety

xs_val_rank and the private helper xs_rank_val_ both use nth0(Rank, Vals, V)
with Rank unbound. Because nth0 generates positions lazily, it leaves a
choicepoint after finding the first match. Since Vals is a sorted list of
unique values, each value appears exactly once, so the first match is the
only match. A cut after nth0 commits to that match and eliminates the
choicepoint, producing clean deterministic behavior and no PLUnit warnings.

3.2 Two-Pass Extremum Pattern

xs_max_cells and xs_min_cells use a two-pass design: first gather all values
(O(HxW) work), find the extremum once, then gather positions matching that
extremum (another O(HxW) pass). The alternative - sorting all R-C-value
triples - would be heavier and less readable. The two-pass approach is simple
and sufficient.

3.3 Nested maplist for Grid Transformation

xs_threshold, xs_replace_lt, and xs_replace_gt all apply transformations to
every cell. The pattern is: outer maplist processes rows, inner maplist
processes cells within a row, and a private helper predicate applies the
if-then-else. Private helpers are named xs_thresh_row_/xs_thresh_val_,
xs_repl_row_lt_/xs_repl_lt_, and xs_repl_row_gt_/xs_repl_gt_. This keeps
each predicate clause short while maintaining full ERC comment coverage.

3.4 Complementary Packs

xs_rank_vals produces a sorted unique value list identical in structure to
hi_sorted_by_freq from the histogram pack, but ordered by value rather than
by frequency. xs_max_cells returns positions rather than the maximum value
itself (ar_cell_max does that). xs_cells_le and xs_cells_ge together cover
the same ground as xs_cells_between, but the dedicated xs_cells_between
is more direct and does not require a post-hoc intersection step.

Library imports: library(lists): member/2, nth0/3, max_list/2, min_list/2.
library(apply): maplist/3.


4. Test Results

42 PLUnit acceptance tests pass with no warnings.

Three tests per predicate cover a standard case, a boundary/repeated-value
case, and an empty-result case.

Key correctness tests:
  xs_cells_lt([[3,3],[3,3]], 3, Cs) => Cs=[]
  xs_max_cells([[3,3],[3,3]], Cs) => Cs=[0-0, 0-1, 1-0, 1-1]
  xs_threshold([[1,3],[5,2]], 3, 1, 0, G) => G=[[0,1],[1,0]]
  xs_replace_lt([[1,2],[3,4]], 5, 0, G) => G=[[0,0],[0,0]] (all replaced)
  xs_rank_grid([[5,3],[1,7]], G) => G=[[2,1],[0,3]]
  xs_val_rank([[3,1],[2,1]], 3, R) => R=2


5. Significance

The xsel pack is the primary general-purpose value-comparison layer in
PrologAI. xs_cells_lt through xs_cells_ne cover six of the seven basic
comparison relations for numeric cell values. xs_max_cells and xs_min_cells
answer a different form of "where is the extreme?" that complements the
"what is the extreme?" answer from arith.

xs_rank_grid performs a compact relabeling of grid values that is useful when
the task cares only about relative order and not absolute values. A grid
where cells take values 0, 3, 7, 9 becomes a rank grid where they take
values 0, 1, 2, 3, simplifying subsequent comparisons and transformations.

xs_threshold provides the single-threshold binary mapping, which is one of
the most common preprocessing steps in visual recognition tasks: converting
a multi-value grid into a foreground/background mask at a given cutoff.
