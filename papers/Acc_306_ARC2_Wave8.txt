Wave 8 of the ARC-AGI-2 benchmark climb adds the bar_sort named glass-box rule,
solving task 31f7f899 and bringing the running total to 8 out of 120 tasks (6.67%).

The bar_sort rule identifies a solid horizontal divider row in the grid, locates
vertical color bars that cross it both above and below, measures each bar's height
on both sides of the divider, and reorders the bars so that their above-heights
are sorted in ascending order from left to right.  The divider row itself and all
bar positions are preserved; only the color assignment to each bar slot changes.


TASK ANALYSIS: 31f7f899

Grid dimensions: varies across training pairs (13x13, 15x15, 7x7).
Training pairs: 3.
Test case: 1 (17x17).

The input contains:
  - A background color (8 = the most common cell value).
  - A single horizontal divider row: the unique row containing no background cells.
    The divider is filled with a single DIV color (e.g. 6 in train 0 and 1).
  - Several vertical color bars.  Each bar is a contiguous run of a single
    non-background, non-DIV color appearing in the divider row and extending
    some number of rows upward (height HA) and downward (height HB) from the
    divider.

The rule:
  1. Identify background BG = mode(all cells).
  2. Find the divider row: first row with no BG cell.  Record DIV = mode(divider row).
  3. Parse the divider row into bar descriptors bar(StartCol, EndCol, Color).
  4. For each bar, measure HA (rows above divider with that color in bar column)
     and HB (rows below divider with that color in bar column).
  5. Sort the (HA, HB) pairs by HA ascending: SortedPairs.
  6. Reconstruct the grid: the divider row is unchanged; a cell (R, C) inside
     bar slot I receives the color of bar(I) if its distance from the divider
     is within the sorted HA or HB for that slot.  Cells outside all bar spans
     revert to BG.

In plain terms: the bars are reordered left-to-right by their above-heights,
shortest first.  The bar widths and the divider are preserved exactly.

Example (train 0, 13x13):
  Divider at row 6 (all 6s with bar values).
  Bars at columns: 1 (color 5, HA=4, HB=4), 6 (color 2, HA=3, HB=3),
                   7 (color 7, HA=2, HB=2), 8 (color 1, HA=1, HB=1).
  Sorted by HA ascending: 1, 2, 3, 4.
  Output: column 1 gets HA=1 (color 1), column 6 gets HA=2 (color 7),
          column 7 gets HA=3 (color 2), column 8 gets HA=4 (color 5).

The above-heights after sorting match 1 <= 2 <= 3 <= 4 in left-to-right order.


RULE IMPLEMENTATION

Rule name: bar_sort
Prolog predicate: arc2_transform(bar_sort, Grid, Result)

Algorithm:
  1. Flatten Grid; compute BG = mode of all cells.
  2. Find DivIdx: the first row index with no BG cell.
     Compute DIV = mode of the divider row.
  3. Parse the divider row into Bars = [bar(SC,EC,Color), ...].
  4. For each bar compute Heights = [HA-HB, ...].
  5. SortedPairs = msort(Heights).
  6. Rebuild Grid: row DivIdx is copied unchanged; every other cell is
     assigned by arc2_bs_bar_covers_ which selects the bar slot by column
     and checks the sorted height against the row distance from DivIdx.

Helper predicates:
  arc2_bs_mode_/2          mode of a sorted list.
  arc2_bs_divider_/4       locate the divider row.
  arc2_bs_bars_/5          parse the divider row into bar descriptors.
  arc2_bs_heights_/4       measure above/below heights for each bar.
  arc2_bs_count_up_/6      count matching rows above the divider.
  arc2_bs_count_dn_/7      count matching rows below the divider.
  arc2_bs_cell_/8          assign cell color in the output.
  arc2_bs_bar_covers_/6    check if a bar slot covers a given (row, col).

Verification: arc2_transform(bar_sort, In, Out) succeeds for all 3 training
pairs and the test case of task 31f7f899.

LESSONS LEARNED

The bar_sort pattern is an abstract sorting-by-measure problem.  The key insight
is that the divider row is not just decoration: it encodes the bar positions and
colors, while the bar bodies above and below the divider encode numeric quantities
(heights) that determine the sorted order.  Recognizing the divider as the
structural anchor for the entire transformation was the main conceptual step.

Tasks of this type share a common structure: a 2D "key" embedded in the grid
encodes a permutation, and the transformation reorders visual elements to realize
that permutation.  The same family of ideas applies to any task where a marker
row or column defines a ranking.
