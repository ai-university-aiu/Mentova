Abstract

PrologAI gains Work Package 115: the permute pack, a Layer 94 symbolic reasoning
module providing 14 predicates for arbitrary reordering of grid rows and
columns. The pack covers index-list permutation, pairwise swap, cyclic
shifting, permutation discovery, lexicographic sorting, and structural
insertion and deletion. This accomplishment is verified by 42 PLUnit
acceptance tests, all passing.


1. Motivation

Many visual transformation tasks involve rearranging rows or columns of a
grid: placing them in a new order, shifting them cyclically, or discovering
which reordering maps one grid to another. Previous PrologAI packs addressed
fixed orientations - the flip pack mirrors in two directions, the rotate pack
turns by multiples of 90 degrees, and the tile pack splits into equal bands.
None of these can express a general index permutation or find the permutation
between two grids. The permute pack fills that gap.


2. Predicate Inventory (14 predicates, pm_* prefix)

2.1 Direct permutation

pm_permute_rows(+Grid, +Perm, -Result)

Reorders rows by an explicit index list. Result row i equals Grid row Perm[i].

pm_permute_cols(+Grid, +Perm, -Result)

Reorders columns by an explicit index list. Result column j equals Grid column
Perm[j].

Example:

pm_permute_rows([[1,2],[3,4],[5,6]], [2,0,1], R).
% R = [[5,6],[1,2],[3,4]]

pm_permute_cols([[1,2,3],[4,5,6]], [2,0,1], R).
% R = [[3,1,2],[6,4,5]]


2.2 Pairwise swap

pm_swap_rows(+Grid, +R1, +R2, -Result)

Exchanges rows R1 and R2. All other rows are unchanged.

pm_swap_cols(+Grid, +C1, +C2, -Result)

Exchanges columns C1 and C2. Implemented by computing a swap permutation and
delegating to pm_permute_cols.

Example:

pm_swap_rows([[1,2],[3,4],[5,6]], 0, 2, R).
% R = [[5,6],[3,4],[1,2]]


2.3 Cyclic shifting

pm_cycle_rows(+Grid, +N, -Result)

Moves the last N rows to the front. N is reduced modulo the row count.
N equal to zero returns the grid unchanged.

pm_cycle_cols(+Grid, +N, -Result)

Moves the last N columns to the left. N is reduced modulo the column count.

Example:

pm_cycle_rows([[1,2],[3,4],[5,6]], 1, R).
% R = [[5,6],[1,2],[3,4]]

pm_cycle_cols([[1,2,3],[4,5,6]], 1, R).
% R = [[3,1,2],[6,4,5]]


2.4 Permutation discovery

pm_find_row_perm(+Grid1, +Grid2, -Perm)

Finds the permutation such that Grid2[i] equals Grid1[Perm[i]] for every row.
Fails if any row of Grid2 does not appear in Grid1.

pm_find_col_perm(+Grid1, +Grid2, -Perm)

Finds the permutation such that column j of Grid2 equals column Perm[j] of
Grid1. Fails if any column of Grid2 does not appear in Grid1.

Example:

pm_find_row_perm([[1,2],[3,4],[5,6]], [[5,6],[1,2],[3,4]], P).
% P = [2,0,1]


2.5 Lexicographic sorting

pm_sort_rows(+Grid, -Sorted)

Sorts rows in ascending lexicographic order. Duplicate rows are preserved
(msort, not sort).

pm_sort_cols(+Grid, -Sorted)

Sorts columns in ascending lexicographic order. Extracts column lists, sorts
them, then reconstructs the grid.

Example:

pm_sort_cols([[3,1,2],[6,4,5]], R).
% R = [[1,2,3],[4,5,6]]


2.6 Structural editing

pm_insert_row(+Grid, +R, +NewRow, -Result)

Inserts NewRow before position R. Existing rows at and beyond R shift down by
one.

pm_delete_row(+Grid, +R, -Result)

Removes row R. Rows beyond R shift up by one.

pm_insert_col(+Grid, +C, +NewCol, -Result)

Inserts a column of values before position C in every row. NewCol is a list
with one value per row.

pm_delete_col(+Grid, +C, -Result)

Removes column C from every row.

Example:

pm_insert_row([[1,2],[3,4]], 1, [9,9], R).
% R = [[1,2],[9,9],[3,4]]

pm_delete_col([[1,2,3],[4,5,6]], 1, R).
% R = [[1,3],[4,6]]


3. Implementation Notes

All predicates use only standard library imports: nth0/3 and nth0/4 from
library(lists) for indexed access and list removal; numlist/3, append/2, and
append/3 from library(lists) for range generation and list splitting; and
maplist/2, maplist/3, and maplist/4 from library(apply) for bulk row and
column processing.

Three internal helpers support the pack: pm_col_ extracts a column as a list
using maplist, pm_first_row_idx_ finds the first index of a matching row using
nth0 with cut, and pm_first_col_idx_ finds the first matching column index by
iterating with between and cut.

Cyclic shift uses append/3 to split then rejoin. Structural editing uses
append/3 for rows and nth0/4 in its removal mode for columns.

The pm_swap_cols predicate avoids a separate code path by computing a swap
permutation and delegating to pm_permute_cols.


4. Test Results

42 PLUnit (Prolog Unit Testing) acceptance tests pass. Three tests per
predicate cover the primary case, an edge case, and a boundary case for each
of the 14 predicates.


5. Relationship to Existing Packs

The flip pack (Layer 20) reflects in left-right or top-bottom directions only.

The rotate pack (Layer 21) rotates by 90-degree increments only.

The tile pack (Layer 81) splits into equal-sized bands.

The region pack (Layer 92) splits at content-defined separator lines.

The permute pack is the only pack that supports arbitrary index permutations,
cyclic shifting, permutation discovery, and structural row/column editing.
These are complementary capabilities; none replaces another.


6. Significance

Permutation operations are fundamental to any task where the correct answer
involves rearranging the structure of a grid. A solver that can propose
permutations, test them, and discover which permutation maps input to output
has a qualitatively different reach than one limited to fixed transforms.
