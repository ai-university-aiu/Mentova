Abstract

PrologAI gains Work Package 131: the histogram pack, a Layer 110 symbolic
reasoning module providing 14 predicates for value frequency analysis. The
pack counts occurrences, computes sorted frequency tables, finds the most and
least common values, tests for value presence and uniformity, selects the top-N
most frequent values, ranks values by frequency, and computes frequency tables
and modal colors from 2D integer grids. All 42 PLUnit acceptance tests pass.


1. Motivation

Color counting is fundamental to visual pattern recognition. A solver that
cannot answer "how many times does color 3 appear?" or "what is the most
common color in the output grid?" cannot address the majority of ARC-AGI-2
tasks, where outputs are frequently determined by the dominant, rarest, or
second-most-frequent color in the input.

The existing PrologAI pack library provides scene-level object extraction,
region detection, and spatial reasoning, but lacked a dedicated frequency
analysis layer. The histogram pack fills this gap, making value counting and
frequency ranking first-class operations at Layer 110.


2. Predicate Inventory (14 predicates, hi_* prefix)

2.1 Basic Counting

hi_count(+List, +V, -N)
N is the number of times value V appears in List. Uses include/3 with ==(V)
to collect matching elements, then length/2 to count them. Returns 0 when V
is not present.

hi_freq(+List, -Pairs)
Pairs is the sorted list of V-Count pairs for every distinct value in List,
sorted by value ascending. Computed by sort/2 to get unique values, then
hi_count for each.

Examples:
hi_count([1,2,1,3,1], 1, N).   % N = 3
hi_count([1,2,3], 4, N).       % N = 0
hi_freq([3,1,2,1,3,3], P).     % P = [1-2, 2-1, 3-3]
hi_freq([], P).                  % P = []


2.2 Modal Values

hi_mode(+List, -V, -N)
V is the most frequent value; N is its count. On ties, the smallest value
wins. Implemented by taking the head of hi_sorted_by_freq.

hi_rare(+List, -V, -N)
V is the least frequent value; N is its count. On ties, the largest value
wins. Implemented by taking the last element of hi_sorted_by_freq.

Examples:
hi_mode([1,2,2,3,3,3], V, N).  % V=3, N=3
hi_rare([1,2,2,3,3,3], V, N).  % V=1, N=1
hi_mode([1,1,2,2], V, N).      % V=1, N=2 (tie: smallest wins)
hi_rare([1,1,2,2], V, N).      % V=2, N=2 (tie: largest wins)


2.3 Distinct Values

hi_unique_vals(+List, -Vals)
Vals is the sorted list of distinct values in List. Equivalent to sort/2.

hi_unique_count(+List, -N)
N is the number of distinct values in List.

Examples:
hi_unique_vals([3,1,2,1,3], Vals).  % Vals = [1,2,3]
hi_unique_count([1,1,2,3,3], N).    % N = 3


2.4 Sorted by Frequency

hi_sorted_by_freq(+List, -Pairs)
Pairs is the list of V-Count pairs sorted by Count descending. Ties in count
are broken by value ascending. Implementation: negate counts, apply keysort
(stable, ascending), recover the original counts.

The stable sort preserves the value-ascending order from hi_freq for equal
counts, ensuring deterministic tie-breaking without additional comparison.

Examples:
hi_sorted_by_freq([1,2,2,3,3,3], P).  % P = [3-3, 2-2, 1-1]
hi_sorted_by_freq([1,1,2,2], P).      % P = [1-2, 2-2]  (tie: V asc)


2.5 Top-N Selection

hi_top_n(+List, +N, -TopN)
TopN is the list of the N most frequent V-Count pairs from List. Uses
length/2 to create an N-element list, then append/3 to extract it as a
prefix of hi_sorted_by_freq. Fails if the list has fewer than N distinct
values.

Examples:
hi_top_n([1,2,2,3,3,3], 2, T).  % T = [3-3, 2-2]
hi_top_n([1,2,3], 1, T).        % T = [1-1]


2.6 Presence and Uniformity

hi_has_value(+List, +V)
Succeed if V appears at least once in List. Uses memberchk/2 for
determinism (no choicepoint).

hi_all_equal(+List, -V)
Succeed if every element of List equals V. V is unified with the first
element; forall(member(X, Tail), X =:= V) verifies the rest. Fails for
the empty list.

Examples:
hi_has_value([1,2,3], 2).        % true
hi_all_equal([3,3,3], V).        % V = 3
hi_all_equal([3,3,4], _).        % fail


2.7 Grid-Level Predicates

hi_grid_freq(+Grid, -Pairs)
Frequency table of all cell values in a 2D grid. Flattens the grid one level
with append/2, then calls hi_freq.

hi_grid_mode(+Grid, -Color, -Count)
Most frequent cell color in Grid. Flattens and calls hi_mode.

hi_grid_rare(+Grid, -Color, -Count)
Least frequent cell color in Grid. Flattens and calls hi_rare.

Examples:
hi_grid_freq([[1,2],[2,3]], P).   % P = [1-1, 2-2, 3-1]
hi_grid_mode([[1,2],[2,2]], C, N). % C=2, N=3
hi_grid_rare([[1,2,2,2]], C, N).  % C=1, N=1


2.8 Ranking

hi_rank(+List, +V, -Rank)
Rank is the 1-based rank of V by frequency (Rank 1 = most frequent). Uses
hi_sorted_by_freq and nth1/3 with a cut after the first match for determinism.
Fails if V does not appear in List.

Examples:
hi_rank([1,2,2,3,3,3], 3, R).  % R = 1 (most common)
hi_rank([1,2,2,3,3,3], 1, R).  % R = 3 (least common)


3. Implementation Notes

hi_sorted_by_freq uses negated-count keysort for efficiency and determinism.
The three-step algorithm:
  (1) hi_freq gives V-N pairs sorted by V.
  (2) findall(NegN-V, ...) negates N to get (-N)-V pairs in V-ascending order.
  (3) keysort sorts by (-N) ascending = N descending; stability preserves V order.
  (4) findall recovers V-N pairs.

hi_all_equal uses arithmetic equality (=:=) because grid values are always
integers. Structural equality (==) would also work, but =:= is the established
convention in PrologAI packs for integer cell comparison.

Imports: library(lists) for member/2, memberchk/2, append/2, nth1/3, last/2;
library(apply) for include/3.


4. Test Results

42 PLUnit acceptance tests pass with no warnings. Three tests per predicate
cover the primary case, a tie-breaking case, and a boundary or edge case.

Key edge cases tested:
  hi_freq([], P) => P = [] (empty list)
  hi_unique_count([], N) => N = 0 (empty list)
  hi_count([1,2,3], 4, N) => N = 0 (absent value)
  hi_mode([1,1,2,2], V, N) => V=1, N=2 (count tie)
  hi_rare([1,1,2,2], V, N) => V=2, N=2 (count tie)


5. Significance

Value frequency analysis is a prerequisite for a large class of visual
reasoning tasks where the answer depends on the dominant, rarest, or
specific-rank color in the input. The histogram pack makes all frequency
operations first-class predicates rather than incidental ad-hoc counts.

Combined with the signal pack (sg_row_profile, sg_col_profile) and scene pack
(object extraction), the histogram pack completes a frequency analysis pipeline:
  (1) Extract a row or column as a list.
  (2) Call hi_mode or hi_freq to identify the dominant value.
  (3) Call hi_grid_mode on the full grid for the global dominant color.
  (4) Call hi_rank to find which frequency position a given color occupies.
