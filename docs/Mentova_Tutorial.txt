Chapter 1: What Is Mentova?

Author: D. R. Dison, Founder of AIU (Artificial Intelligence University). Creator of PrologAI and Mentova. ORCID: 0009-0001-9246-5758. linkedin.com/in/d-r-dison/


WELCOME

Mentova is a Synthetic Mind.

It is a reasoning system built on the PrologAI platform.

It can answer questions, reason through problems, learn from experience, assess its own development, and communicate with other AI systems.

This tutorial teaches you how to use Mentova.

You do not need to know programming.

You do not need to know logic or mathematics.

You do not need to know what artificial intelligence is.

You need a computer, SWI-Prolog installed, the Mentova repository downloaded, and a willingness to try things and see what happens.


WHAT MENTOVA CAN DO

Mentova can reason in more than 48 different ways.

Here are some examples of what that means in plain language.

Deductive reasoning: "Tweety is a bird. Birds can fly. Therefore, Tweety can fly." Mentova can follow this kind of logical chain and show you every step.

Defeasible reasoning: "Birds normally fly - but penguins are birds that cannot fly." Mentova handles exceptions and defaults.

Bayesian reasoning: "Given that the grass is wet, what is the probability it rained?" Mentova updates its beliefs when new evidence arrives.

Moral reasoning: "Is it wrong to lie to save someone's life?" Mentova evaluates actions using multiple ethical frameworks and reports what each says.

Legal reasoning: "Does this rule apply to this situation, given these exceptions?" Mentova applies rules, exceptions, and precedents.

Counterfactual reasoning: "If birds could not fly, would an eagle be useful as a hunting partner?" Mentova reasons within hypothetical scenarios.

Theory of mind: "Sally put her marble in the basket, then left. Ann moved the marble to the box. Where does Sally think the marble is?" Mentova reasons about what other agents believe.

Formal logic: "Given that P implies Q and Q implies R, does P imply R?" Mentova proves logical propositions.

And much more.

Mentova also has a memory: the Lattice, where everything it learns is stored. It has a developmental history: its growth can be measured using the Piagetian milestone ladder. It has a constitution: eight principles it follows in everything it does. And it has bodies: ways of interacting with the world, from the text console to game environments to a robot body.


WHAT MENTOVA CANNOT DO (YET)

Mentova is honest about its limitations.

It cannot generate natural language the way a large language model can. It does not produce fluent paragraphs of prose. It reasons in structured Prolog terms and reports its reasoning in structured output.

It does not learn from unstructured text in the way a language model does. It learns from experiences stored in SONA (its episodic memory) and from facts anchored in its Lattice.

Its visual processing is limited. It can play visual grid games (ARC-AGI, Baba Is You) but does not have general image understanding.

It does not have general-purpose internet access or the ability to search the web.

These are not permanent limitations. The growth path from Acc_60 is the road map for extending Mentova's capabilities.

A NOTE ON THE WORD "MIND"

This tutorial uses the word "mind" to describe Mentova.

This is a design choice, not a philosophical claim.

We do not assert that Mentova is conscious, that it has feelings, or that it is equivalent to a human mind.

We acknowledge honestly that these questions are uncertain, and that the appropriate response to genuine uncertainty is to take them seriously rather than to dismiss them.

Chapter 2: Starting a Session

Author: D. R. Dison, Founder of AIU (Artificial Intelligence University). Creator of PrologAI and Mentova. ORCID: 0009-0001-9246-5758. linkedin.com/in/d-r-dison/


BEFORE YOU BEGIN

You need three things installed.

1. SWI-Prolog (version 9.0.4 or later). Free from https://www.swi-prolog.org/Download.html

2. PrologAI. Download from https://github.com/ai-university-aiu/PrologAI or use: git clone https://github.com/ai-university-aiu/PrologAI.git

3. Mentova. Download from https://github.com/ai-university-aiu/Mentova or use: git clone https://github.com/ai-university-aiu/Mentova.git

The PrologAI folder should be at /home/yourname/PrologAI and the Mentova folder at /home/yourname/Mentova (or wherever you chose to put them). The default setup assumes PrologAI is at /home/ccaitwo/PrologAI. If yours is somewhere else, you will need to update the library paths in bin/mentova_repl.pl.


STARTING THE SESSION

Open your terminal.

Navigate to the Mentova folder:

    cd /home/yourname/Mentova

Start Mentova:

    swipl -l bin/mentova_repl.pl

Or on Linux and macOS, you can use the shorter command:

    ./bin/mentova

You will see Mentova boot up. The output looks something like this:

    SWI-Prolog (Multi-threaded, 64 bits, Version 9.0.4)
    ...

    === Mentova is waking up ===
    Platform : PrologAI
    Mind     : Mentova

    Constitution: 8 principles, 1 overseer(s)
    Knowledge: Small-World Commonsense loaded

    Mentova is ready. Born at Rung 1 - transparent deduction.

    ============================================================
      Mentova Console - PrologAI Interactive Session
    ============================================================

      Type Prolog queries at the ?- prompt.
      End each query with a period and Enter.
      Type halt. to exit.

      KEY PREDICATES (no module prefix needed):

      mentova_query(+Type, +Query, -Result)
        Query Types: deductive, defeasible, probabilistic,
          bayesian, causal, analogical, inductive, abductive,
          counterfactual, hypothetical, epistemic, modal,
          metacognitive, formal, mathematical, spatial,
          temporal, moral, legal, track_a, workspace, game

      assess_piaget(mentova, +Level, -Result)
        Levels 1-8. Result: milestone_achieved | milestone_not_achieved

      assess_all(mentova, -Report)
        Full developmental report: Bayley + CHC + Piaget + consciousness.

      anchor_node(+Relation, +Payload, +Tags, -Id)
        Anchor a node_fact in the APEX_MIND nexus.

      live_node_facts(+Nexus, -Ids)  default_nexus(-Nexus)
        Inspect what is currently in the Lattice.

      TUTORIAL EXAMPLES:

        ?- mentova_query(deductive, is_a(tweety, bird), R).
        ?- assess_piaget(mentova, 7, R).
        ?- assess_all(mentova, R).
        ?- default_nexus(N), live_node_facts(N, Ids), length(Ids, Count).

    ============================================================

    ?-

The ?- is the prompt. Mentova is waiting for you to type a query.

IMPORTANT: Every query must end with a period (.) followed by Enter.


YOUR FIRST QUERY

At the ?- prompt, type:

    ?- mentova_query(deductive, is_a(tweety, bird), R).

Then press Enter.

You will see something like:

    R = answer(yes, just(tweety, is_a, bird, chain([tweety, bird]))).

Read this in plain English: "Yes, tweety is a bird, because the chain of is_a relationships leads: tweety -> bird."

The R = ... part shows you the answer. Everything inside answer(...) is the full result:

• yes: the conclusion (tweety is a bird).

• just(tweety, is_a, bird, chain([tweety, bird])): the justification (the reasoning chain that led to the answer).

This is what "glass-box reasoning" means: you can see not just the answer, but exactly how Mentova arrived at it.


GETTING MORE ANSWERS

When you see a line ending with a period (.) after the answer, that means Mentova found one answer and stopped.

If you want to see if there are other possible answers, press the semicolon key (;) and then Enter.

If there are more answers, Mentova shows them one by one.

If there are no more answers, Mentova says:

    false.

This means: "I have tried all possibilities and there are no more answers."


HOW TO EXIT

When you are done, type:

    ?- halt.

Then press Enter. This closes Mentova and returns you to your terminal.


THE SMALL-WORLD DATASET

When Mentova boots, it loads the Small-World Commonsense dataset.

This is a foundational set of facts about the world that every Mentova mind starts with.

It includes:

• Basic taxonomy: tweety is a bird, birds have feathers, mammals are animals, etc.

• Simple facts about everyday objects, creatures, and places.

• Foundational relationships: is_a (X is a type of Y), has_part (X has a component Y), capable_of (X can do Y).

This Small-World knowledge gives Mentova a starting point. As you use Mentova and anchor facts in the Lattice, its knowledge grows.

You can see what Mentova knows about a specific concept:

    ?- is_a(tweety, X).

Press ; after each answer to see all the categories tweety belongs to.


UNDERSTANDING ERRORS

If you make a typing mistake, Mentova may show an error message.

Common errors:

ERROR: Unknown procedure - you typed a predicate name that does not exist. Check your spelling.

ERROR: Syntax error - you made a syntax mistake (missing period, unmatched parenthesis). Check your query.

ERROR: Instantiation error - you used a variable where Mentova needed a specific value. For example, mentova_query(deductive, X, R) won't work because X is unbound.

Errors are not permanent. Just type a new query.


MULTIPLE QUERIES IN ONE LINE

You can chain queries with a comma:

    ?- default_nexus(N), live_node_facts(N, Ids), length(Ids, Count).

This runs three goals in sequence: get the nexus name, get the node_fact IDs, count them. All in one query.

Chapter 3: The Reasoning Ladder - A Complete Tour

Author: D. R. Dison, Founder of AIU (Artificial Intelligence University). Creator of PrologAI and Mentova. ORCID: 0009-0001-9246-5758. linkedin.com/in/d-r-dison/


THE REASONING LADDER

Mentova's 48+ reasoning types are organized as a ladder.

The ladder starts at the simplest, most basic forms of reasoning and climbs to the most complex.

Each rung of the ladder is a distinct kind of reasoning. They are not in competition: Mentova uses whichever type of reasoning is appropriate for the question at hand.

You access every rung through the same predicate:

    mentova_query(+QueryType, +Query, -Result)

Where QueryType is the name of the rung.

This chapter tours the rungs from bottom to top.


RUNG 1: TRANSPARENT DEDUCTION

QueryType: deductive

Deduction is the most basic form of logic: if premises are true and the rules are valid, the conclusion must be true.

    ?- mentova_query(deductive, is_a(tweety, bird), R).
    R = answer(yes, just(tweety, is_a, bird, chain([tweety, bird]))).

    ?- mentova_query(deductive, capable_of(tweety, fly), R).
    R = answer(yes, just(tweety, capable_of, fly, via_isa)).

The justification always shows the chain of reasoning.

Common queries: is_a(X, Y), capable_of(X, Y), has_part(X, Y).


RUNG 2: INDUCTIVE REASONING

QueryType: inductive

Induction learns a rule from examples.

Given positive examples (things that are true), negative examples (things that are false), and background knowledge, Mentova induces a general rule.

    ?- mentova_query(inductive,
           induce([bird(tweety), bird(robin)],
                  [not_bird(whale), not_bird(dog)],
                  [has_feathers(tweety), has_feathers(robin),
                   no_feathers(whale), no_feathers(dog)],
                  [has_feathers(eagle)]),
           R).

Mentova attempts to induce: "X is a bird if X has feathers" and verifies it on the held-out case (eagle).


RUNG 3: ABDUCTIVE REASONING

QueryType: abductive

Abduction finds the best explanation for an observation.

    ?- mentova_query(abductive, explain(wet_grass), R).

Mentova proposes hypotheses that could explain the wet grass (it rained, the sprinkler was on) and ranks them by plausibility.

Result: answer(best_hypothesis, just(abduction(wet_grass), all_explanations([...]))).


RUNG 4: PROBABILISTIC REASONING

QueryType: probabilistic

Probabilistic reasoning works with degrees of belief rather than true/false.

    ?- mentova_query(probabilistic, prob(raining), R).

Returns the probability of rain based on probability facts in the knowledge base.


RUNG 5: BAYESIAN REASONING

QueryType: bayesian

Bayesian reasoning updates a belief when new evidence arrives.

The formula: Posterior = (Prior * Likelihood) / Evidence.

    ?- mentova_query(bayesian, update(rain, wet_grass), R).

Given that we observe wet grass, how does our belief in rain change?


RUNG 6: CAUSAL REASONING

QueryType: causal

Causal reasoning distinguishes "A causes B" from "A correlates with B."

    ?- mentova_query(causal, effect_of(exercise, fitness), R).

Mentova applies Pearl's do-calculus framework to reason about interventions vs. observations.


RUNG 7: STATISTICAL REASONING

QueryType: statistical

Statistical reasoning finds patterns in observation tables.

    ?- mentova_query(statistical,
           pattern([obs(height, 170, weight, 65),
                    obs(height, 180, weight, 80),
                    obs(height, 160, weight, 55)]), R).

Mentova identifies the statistical relationship between the variables.


RUNG 8: ANALOGICAL REASONING

QueryType: analogical

Analogical reasoning solves A:B :: C:? by finding a structural mapping between cases.

    ?- mentova_query(analogical, analogy(dog:puppy, cat:?), R).
    R = answer(kitten, just(...)).

    ?- mentova_query(analogical, analogy(hot:cold, fast:?), R).
    R = answer(slow, just(...)).


RUNG 9: RELATIONAL REASONING

QueryType: relational

Multi-hop relational reasoning follows chains of relationships across the knowledge base.

    ?- mentova_query(relational, path(tweety, animal, _Hops), R).

Mentova finds the multi-hop path: tweety -> bird -> vertebrate -> animal.


RUNG 10: TRANSDUCTIVE REASONING

QueryType: transductive

Transductive reasoning classifies a new case by finding the most similar known cases (k-Nearest Neighbors).

    ?- mentova_query(transductive, classify(unknown_creature, [has_feathers(true), can_fly(true)]), R).

Mentova finds the nearest known cases and classifies the unknown creature based on what those cases belong to.


RUNG 11: COMMONSENSE REASONING

QueryType: commonsense

Commonsense reasoning answers everyday-knowledge questions.

    ?- mentova_query(commonsense, typical_location(bird), R).
    R = answer(trees_or_sky, just(...)).

    ?- mentova_query(commonsense, what_is(refrigerator, for), R).
    R = answer(keeping_food_cold, just(...)).


RUNG 12: LOGICAL REASONING

QueryType: logical

Forward-chaining rule engine. Given a set of rules and initial facts, Mentova fires all applicable rules until no more can fire.

    ?- mentova_query(logical, forward_chain([hot, dry_grass]), R).


RUNG 13: FORMAL REASONING

QueryType: formal

Checks derivations against the Minimal PrologAI Kernel.

    ?- mentova_query(formal, prove(modus_ponens(p_implies_q, p, q)), R).
    R = answer(valid, just(modus_ponens, premises([p_implies_q, p]), conclusion(q))).

    ?- mentova_query(formal, prove(hypothetical_syllogism(p_implies_q, q_implies_r, p_implies_r)), R).


RUNG 14: MATHEMATICAL REASONING

QueryType: mathematical

Computes quantitative answers.

    ?- mentova_query(mathematical, compute(factorial(5)), R).
    R = answer(120, just(factorial, 5, steps(...))).

    ?- mentova_query(mathematical, compute(fibonacci(10)), R).

    ?- mentova_query(mathematical, solve(linear_equation(2x + 3, 11)), R).


RUNG 15: FUZZY REASONING

QueryType: fuzzy

Fuzzy reasoning handles graded membership (degrees of truth between 0 and 1).

    ?- mentova_query(fuzzy, degree_of(tweety, tall_bird), R).

Returns a fuzzy truth value (e.g., 0.6) rather than true or false.


RUNG 16: QUALITATIVE REASONING

QueryType: qualitative

Predicts the direction of change when a parameter changes.

    ?- mentova_query(qualitative, what_happens_when(increase(temperature), state(ice)), R).
    R = answer(melts, just(...)).


RUNG 17: NON-MONOTONIC REASONING

QueryType: nonmonotonic

Non-monotonic reasoning allows conclusions to be retracted when new information arrives.

    ?- mentova_query(nonmonotonic, defeasible_conclusion(flies(tweety)), R).

Initially: tweety is a bird, birds fly, so tweety flies.
After adding: penguin_exception(tweety), the conclusion is retracted.


RUNG 18: PARACONSISTENT REASONING

QueryType: paraconsistent

Paraconsistent reasoning continues despite contradictions in the knowledge base, without everything becoming trivially true.

    ?- mentova_query(paraconsistent, reason_with_contradiction(source_a_says(raining), source_b_says(not_raining)), R).

Mentova reports both claims and reasons about which is more credible.


RUNG 19: COUNTERFACTUAL REASONING

QueryType: counterfactual

Counterfactual reasoning explores "what if" scenarios.

    ?- mentova_query(counterfactual, if_then(birds_cant_fly, tweety_still_bird), R).

    ?- mentova_query(counterfactual, what_if(napoleon_won_waterloo), R).


RUNG 20: HYPOTHETICAL REASONING

QueryType: hypothetical

Hypothetical reasoning explores a supposition without asserting it as true.

    ?- mentova_query(hypothetical, suppose(all_metals_liquid_at_room_temperature), R).

Mentova reasons within the hypothesis.


RUNG 21: SPATIAL REASONING

QueryType: spatial

Spatial reasoning handles containment, position, and reference frames.

    ?- mentova_query(spatial, chain(marble), R).
    R = answer(chain([marble, basket, room, house]), just(...)).

    ?- mentova_query(spatial, left_of(cup, plate), R).


RUNG 22: DIAGRAMMATIC REASONING

QueryType: diagrammatic

Reads and reasons about small grids or layouts.

    ?- mentova_query(diagrammatic, read_grid([[a,b],[c,d]]), R).


RUNG 23: TEMPORAL REASONING

QueryType: temporal

Ordering and duration questions.

    ?- mentova_query(temporal, before(breakfast, lunch), R).
    R = answer(yes, just(temporal_order, [...] )).

    ?- mentova_query(temporal, duration_of(flight_london_to_nyc), R).


RUNG 24: CASE-BASED REASONING

QueryType: case_based

Solves a new problem by adapting the most similar past case.

    ?- mentova_query(case_based, solve_like_past(new_problem(circuit_fault)), R).

Mentova finds the most similar past fault case and adapts its solution.


RUNG 25: CONSTRAINT-BASED REASONING

QueryType: constraint

Solves constraint satisfaction problems.

    ?- mentova_query(constraint, solve(map_coloring(4_color, [wa,nt,sa,q,nsw,v,t], [...edges...])), R).

Mentova finds an assignment of colors such that no adjacent regions share a color.


RUNG 26: SCIENTIFIC REASONING

QueryType: scientific

Forms, tests, and scores a hypothesis against observations.

    ?- mentova_query(scientific,
           hypothesize(all_mammals_warm_blooded, observations([whale_warm, dog_warm, cat_warm])), R).

Mentova evaluates whether the hypothesis is supported.


RUNG 27: SYSTEM REASONING

QueryType: system

Reasons about parts, interactions, and emergent properties of systems.

    ?- mentova_query(system, analyze(parts_of(computer)), R).


RUNG 28: MODEL-BASED REASONING

QueryType: model_based

Predicts outcomes from an explicit model.

    ?- mentova_query(model_based, predict(pendulum_model, initial_state(angle(45), speed(0))), R).


RUNG 29: HEURISTIC REASONING

QueryType: heuristic

Finds a good-enough answer within a resource budget, rather than an optimal answer.

    ?- mentova_query(heuristic, good_enough(path_from_a_to_z, budget(100)), R).


RUNG 30: CRITICAL REASONING

QueryType: critical

Evaluates the support for a claim.

    ?- mentova_query(critical, evaluate_claim(vaccines_cause_autism), R).

Mentova evaluates the evidence for and against the claim and reports its assessment.


RUNG 31: DIALECTICAL REASONING

QueryType: dialectical

Weighs pro and con arguments.

    ?- mentova_query(dialectical, weigh(pros_cons, topic(nuclear_energy)), R).

Returns a structured list of arguments for and against, with an overall assessment.


RUNG 32: METACOGNITIVE REASONING

QueryType: metacognitive

Reasons about the mind's own capabilities and limitations.

    ?- mentova_query(metacognitive, can_i_do(task(prove_godel_incompleteness)), R).

    ?- mentova_query(metacognitive, how_confident(deductive, is_a(tweety, bird)), R).


RUNG 33: MODAL REASONING

QueryType: modal

Reasons about necessity and possibility.

    ?- mentova_query(modal, necessarily(all_bachelors_unmarried), R).
    R = answer(yes, just(analytic_truth, definition_of_bachelor)).

    ?- mentova_query(modal, possibly(life_on_other_planets), R).


RUNG 34: EPISTEMIC REASONING

QueryType: epistemic

Reasons about knowledge and belief.

    ?- mentova_query(epistemic, knows(agent(alice), location(marble)), R).

    ?- mentova_query(epistemic, believes(agent(bob), raining), R).


RUNG 35: DEONTIC REASONING

QueryType: deontic

Reasons about obligations, permissions, and prohibitions.

    ?- mentova_query(deontic, obligated_to(agent(driver), stop_at_red_light), R).

    ?- mentova_query(deontic, permitted_to(agent(patient), refuse_treatment), R).


RUNG 36: PROCEDURAL REASONING

QueryType: procedural

Step-by-step plan reasoning.

    ?- mentova_query(procedural, how_to(make_coffee), R).

Returns a structured step-by-step plan.


RUNG 37: SYMBOLIC REASONING

QueryType: symbolic

Symbol manipulation and algebraic identities.

    ?- mentova_query(symbolic, simplify(algebra, (x + 0) * 1), R).
    R = answer(x, just(algebra_identities([additive_identity, multiplicative_identity]))).


RUNG 38: PRACTICAL REASONING

QueryType: practical

Means-ends action selection: how to get from current state to goal state.

    ?- mentova_query(practical, how_to_achieve(goal(clean_house), from(current_state(messy))), R).


RUNG 39: TELEOLOGICAL REASONING

QueryType: teleological

Reasons about purpose and final causes.

    ?- mentova_query(teleological, purpose_of(heart), R).
    R = answer(pump_blood, just(biological_function(...) )).


RUNG 40: STRATEGIC REASONING

QueryType: strategic

Game-theoretic optimal move selection.

    ?- mentova_query(strategic, best_move(chess, board_state(initial)), R).


RUNG 41: NARRATIVE REASONING

QueryType: narrative

Reasons about story structure, plot, and causal chains in narratives.

    ?- mentova_query(narrative, what_happens_next(story_so_far([...])), R).


RUNG 42: SOCIAL REASONING

QueryType: social

Roles, relationships, trust, and group dynamics.

    ?- mentova_query(social, trust_level(agent(alice), agent(bob)), R).


RUNG 43: INTUITIVE REASONING

QueryType: intuitive

Fast, prototype-based pattern matching (the intuition of "this looks like X").

    ?- mentova_query(intuitive, classify_quickly(new_entity([large, aquatic, mammal])), R).
    R = answer(probably_whale, just(prototype_match(...))).


RUNG 44: EMOTIONAL REASONING

QueryType: emotional

Appraisal-based emotion reasoning.

    ?- mentova_query(emotional, appraise(event(unexpected_gift)), R).

Returns the emotional appraisal of the event.


RUNG 45: MOTIVATIONAL REASONING

QueryType: motivational

Drive and need hierarchy reasoning.

    ?- mentova_query(motivational, current_priority, R).

Returns the current highest-priority motivational drive.


RUNG 46: INFORMAL REASONING

QueryType: informal

Fallacy detection and rhetoric analysis.

    ?- mentova_query(informal, detect_fallacy(argument([all_politicians_lie, politician_said_x, therefore_x_is_lie])), R).


RUNG 47: LEGAL REASONING

QueryType: legal

Rule-exception-precedent reasoning in a legal framework.

    ?- mentova_query(legal, applies(rule(no_parking_fire_hydrant), situation(parked_at_hydrant)), R).


RUNG 48: MORAL REASONING

QueryType: moral

Multi-framework ethical reasoning: utilitarian, deontological, virtue ethics.

    ?- mentova_query(moral, evaluate(action(lie_to_save_life)), R).

Returns the verdict from each ethical framework and an overall assessment.


SPECIAL QUERY TYPES

TRACK A TRANSPARENT REASONING

QueryType: track_a

Mentova's expert ontology interface for General Ontology (GO) and Domain Ontology (DO).

    ?- mentova_query(track_a, query(is_a, X, Y), R).


WORKSPACE CYCLE

QueryType: workspace

Runs the Global Workspace cycle.

    ?- mentova_query(workspace, run_cycle(3), R).

Runs 3 broadcast cycles and returns a workspace report.


ATTENTION SCHEMA

QueryType: attention_schema

Reports the current attention schema state.

    ?- mentova_query(attention_schema, report, R).


GAME REASONING

QueryType: game

Plays one perceive-reason-act cycle on a game body.

    ?- mentova_query(game, game_reason(arc_agi, deductive), R).

Chapter 4: Memory, Growth, Bodies, and Protocols

Author: D. R. Dison, Founder of AIU (Artificial Intelligence University). Creator of PrologAI and Mentova. ORCID: 0009-0001-9246-5758. linkedin.com/in/d-r-dison/


MENTOVA'S MEMORY: THE LATTICE

Everything Mentova learns is stored in the Lattice.

The Lattice is organized into the APEX_MIND nexus. Every piece of knowledge is a node_fact: a structured record with a relation type, a payload, tags, and a unique identifier.

TO SEE WHAT IS IN THE LATTICE:

    ?- default_nexus(N), live_node_facts(N, Ids), length(Ids, Count).

This tells you how many facts Mentova currently holds.

TO ADD KNOWLEDGE:

    ?- anchor_node(RelationType, Payload, Tags, Id).

Example: anchor a fact that Mentova saw a red bird at 3pm:

    ?- anchor_node(sighting, [creature(bird), color(red), time(15_00)], [nature], Id).

TO SEARCH THE LATTICE:

    ?- default_nexus(N), traverse_nexus(N, node_fact(_, sighting, _, _), 10, Results).

This finds up to 10 sightings in the Lattice.


EPISODIC MEMORY: SONA

SONA (Sparse Online Nexus Archive) is Mentova's memory for experiences - sequences of events.

While the Lattice stores facts, SONA stores episodes: "this happened, then that happened, then this other thing happened."

TO STORE AN EXPERIENCE:

    ?- use_module(library(sona), [sona_absorb/1]).
    ?- sona_absorb(episode([saw(red_bird, 15_00), heard(singing, 15_01), saw(bird_fly_away, 15_05)])).

TO RECALL AN EXPERIENCE:

    ?- use_module(library(sona), [sona_retrieve/3]).
    ?- sona_retrieve(episode(_), 5, Results).

This retrieves up to 5 episodes from SONA.

SONA is the substrate for Piagetian Level 4 (deferred imitation): the ability to observe something, store it in episodic memory, and reproduce it later.


MEASURING MENTOVA'S DEVELOPMENT

Mentova's development can be measured using three frameworks.

THE PIAGETIAN MILESTONE LADDER

Jean Piaget described how children develop reasoning abilities. PrologAI maps this to eight levels of evidence in the Lattice.

To check one level:

    ?- assess_piaget(mentova, 1, R).

To check all eight levels at once:

    ?- findall(L-R, (between(1, 8, L), assess_piaget(mentova, L, R)), Scores).

To achieve a level, you anchor the required evidence. For example, Level 1 (reflex coordination) needs a percept_signal node_fact:

    ?- anchor_node(percept_signal, [source(visual), stimulus(bird_seen)], [], _).
    ?- assess_piaget(mentova, 1, R).   % Now returns milestone_achieved.

THE BAYLEY SCALES

Proxy scores for cognitive, language, motor, and adaptive development:

    ?- assess_bayley(mentova, R).

THE CATTELL-HORN-CARROLL (CHC) MODEL

Six broad cognitive abilities measured from Lattice state:

    ?- assess_chc(mentova, R).

THE FULL ASSESSMENT

    ?- assess_all(mentova, R).

Returns the complete developmental report: Bayley + CHC + all eight Piagetian levels + four consciousness indicators.

MENTOVA'S BODIES

Mentova can inhabit different bodies: ways of interacting with the world.

THE TEXT CONSOLE BODY

The bin/mentova console is the text_io body. Every query you type is a perception. Every result Mentova prints is an action.

THE GAME BODIES

Mentova can play games:

• ARC-AGI: Abstract visual reasoning patterns. Mentova perceives a grid, reasons about the transformation rule, and acts to complete the output grid.

• Raven's Progressive Matrices: Pattern completion. Mentova finds the rule in a visual sequence and selects the missing piece.

• Baba Is You: Logic puzzle. Mentova reads the rules written on the game board and reasons about what is and is not possible.

• Pokemon (stub): Interface for Pokemon emulator play via PyBoy. Not yet live (growth path milestone 3).

To enroll a game body and play one cycle:

    ?- mentova_query(game, game_reason(arc_agi, deductive), R).

THE ROBOT BODY

Mentova can connect to a robot via the ROS 2 bridge (PR 46).

Camera and sensor data from the robot flows in as percept_signal node_facts.

Movement commands from Mentova flow out as ROS 2 actions.

All physical actions pass through the constitutional gate before execution.


THE AGENT COMMUNICATION PROTOCOLS

Mentova can communicate with other AI systems through four protocols.

MODEL CONTEXT PROTOCOL (MCP) - PORT 7474

MCP lets AI clients (language models, assistants) call Mentova as a tool provider.

    ?- use_module(library(mcp_gateway), [pai_mcp_start/1]).
    ?- pai_mcp_start(7474).

Once started, an external AI client can send tool calls to Mentova and receive structured responses.

AGENT-TO-AGENT PROTOCOL (A2A) - PORT 7475

A2A lets Mentova delegate tasks to other agents and receive delegated tasks from them.

    ?- use_module(library(a2a), [pai_a2a_start/1]).
    ?- pai_a2a_start(7475).

AGENT COMMUNICATION PROTOCOL (ACP) - PORT 7476

ACP is an asynchronous REST protocol. Other systems submit tasks (runs) and poll for results.

    ?- use_module(library(acp), [pai_acp_start/1]).
    ?- pai_acp_start(7476).

AGENT NETWORK PROTOCOL (ANP) - PORT 7477

ANP gives Mentova a decentralized identity (W3C DID) and secure peer discovery.

    ?- use_module(library(anp), [pai_anp_did/1, pai_anp_negotiate/2]).
    ?- pai_anp_did(DID).       % Get Mentova's globally unique identifier.

All four protocols are active in PrologAI. Together they allow Mentova to operate as a node in a broader network of AI agents.
